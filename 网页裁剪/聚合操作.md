## 课程：聚合操作

**注意**：为了更好地理解本节中的概念，请先回顾 [[类与对象-Lambda表达式|Lambda 表达式]] 和[[类与对象-方法引用|方法引用]]两节。

你为什么使用集合？你不会只是把对象存储在集合中然后就让它们留在那里。在大多数情况下，你使用集合是为了检索其中存储的项目。

再考虑 [[类与对象-Lambda表达式|Lambda 表达式]]一节中描述的场景。假设你正在创建一个社交网络应用程序。你想创建一个功能，使管理员能够对满足特定条件的社交网络应用程序成员执行任何类型的操作，例如发送消息。

如前所述，假设此社交网络应用程序的成员由以下 [`Person`](https://docs.oracle.com/javase/tutorial/collections/streams/examples/Person.java) 类表示：

```java
public class Person {

    public enum Sex {
        MALE, FEMALE
    }

    String name;
    LocalDate birthday;
    Sex gender;
    String emailAddress;
    
    // ...

    public int getAge() {
        // ...
    }

    public String getName() {
        // ...
    }
}
```text

以下示例使用 for-each 循环打印集合 `roster` 中包含的所有成员的姓名：

```java
for (Person p : roster) {
    System.out.println(p.getName());
}
```

以下示例打印集合 `roster` 中包含的所有成员，但使用的是聚合操作 `forEach`：

```java
roster
    .stream()
    .forEach(e -> System.out.println(e.getName());

尽管在本示例中，使用聚合操作的版本比使用 for-each 循环的版本更长，但你将看到，对于更复杂的任务，使用批量数据操作的版本会更简洁。

### 管道和流

*管道*是一系列聚合操作。以下示例使用由聚合操作 `filter` 和 `forEach` 组成的管道打印集合 `roster` 中包含的男性成员：

```java
roster
    .stream()
    .filter(e -> e.getGender() == Person.Sex.MALE)
    .forEach(e -> System.out.println(e.getName()));

将此示例与以下使用 for-each 循环打印集合 `roster` 中包含的男性成员的示例进行比较：

```java
for (Person p : roster) {
    if (p.getGender() == Person.Sex.MALE) {
        System.out.println(p.getName());
    }
}

管道包含以下组件：

- 一个源：这可以是集合、数组、生成器函数或 I/O 通道。在此示例中，源是集合 `roster`。
- 零个或多个*中间操作*。中间操作（如 `filter`）产生一个新的流。
	*流*是一系列元素。与集合不同，它不是存储元素的数据结构。相反，流通过管道将值从源传递出去。此示例通过调用方法 `stream` 从集合 `roster` 创建一个流。
	`filter` 操作返回一个包含匹配其谓词（此操作的参数）的元素的新流。在此示例中，谓词是 Lambda 表达式 `e -> e.getGender() == Person.Sex.MALE`。如果对象 `e` 的 `gender` 字段值为 `Person.Sex.MALE`，则返回布尔值 `true`。因此，此示例中的 `filter` 操作返回一个流，它包含集合 `roster` 中的所有男性成员。
- 一个*终端操作*。终端操作（如 `forEach`）产生一个非流结果，例如一个原始值（如 double 值）、一个集合，或在 `forEach` 的情况下完全不产生值。在此示例中，`forEach` 操作的参数是 Lambda 表达式 `e -> System.out.println(e.getName())`，它在对象 `e` 上调用方法 `getName`。（Java 运行时和编译器推断对象 `e` 的类型为 `Person`。）

以下示例使用由聚合操作 `filter`、`mapToInt` 和 `average` 组成的管道，计算集合 `roster` 中包含的所有男性成员的平均年龄：

```java
double average = roster
    .stream()
    .filter(p -> p.getGender() == Person.Sex.MALE)
    .mapToInt(Person::getAge)
    .average()
    .getAsDouble();

`mapToInt` 操作返回一个 `IntStream` 类型的新流（它是一个只包含整数值的流）。该操作将其参数中指定的函数应用于特定流中的每个元素。在此示例中，函数是 `Person::getAge`，它是一个返回成员年龄的方法引用。（或者，你可以使用 Lambda 表达式 `e -> e.getAge()`。）因此，此示例中的 `mapToInt` 操作返回一个流，它包含集合 `roster` 中所有男性成员的年龄。

`average` 操作计算 `IntStream` 类型的流中包含的元素的平均值。它返回一个 `OptionalDouble` 类型的对象。如果流不包含任何元素，则 `average` 操作返回一个空的 `OptionalDouble` 实例，调用方法 `getAsDouble` 会抛出 `NoSuchElementException`。JDK 包含许多像 `average` 这样通过组合流的内容返回一个值的终端操作。这些操作称为*归约操作(reduction operations)*；更多信息请参阅[[聚合操作-归约|归约]]一节。

### 聚合操作和迭代器之间的区别

像 `forEach` 这样的聚合操作看起来像迭代器。然而，它们有几个根本区别：

- **它们使用内部迭代**：聚合操作不包含像 `next` 这样的方法来指示它们处理集合的下一个元素。使用*内部委托*，你的应用程序确定它迭代*什么*集合，但 JDK 确定*如何*迭代集合。使用*外部迭代*，你的应用程序同时确定迭代什么集合以及如何迭代它。然而，外部迭代只能按顺序遍历集合的元素。内部迭代没有此限制。它可以更容易地利用并行计算，并行计算涉及将一个问题划分为子问题，同时解决这些子问题，然后合并子问题的解决方案的结果。更多信息请参阅[[聚合操作-并行|并行]]一节。
- **它们从流处理元素**：聚合操作从流而不是直接从集合处理元素。因此，它们也被称为*流操作*。
- **它们支持将行为作为参数**：你可以为大多数聚合操作指定 [[类与对象-Lambda表达式|Lambda 表达式]]作为参数。这使你能自定义特定聚合操作的行为。
