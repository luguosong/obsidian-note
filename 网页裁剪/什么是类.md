## 什么是类？

在真实世界中，您常常会发现许多同一种类的独立对象。现实中可能存在成千上万辆自行车，全部属于同一品牌和型号。每辆自行车都是按照同一套图纸制造出来的，因此包含相同的组件。用面向对象的术语来说，您的自行车是被称为「自行车」的*对象类(class of objects)*的一个*实例(instance)*。*类(class)*就是创建各个独立对象的图纸。

下面的 [`Bicycle`](https://docs.oracle.com/javase/tutorial/java/concepts/examples/Bicycle.java) 类是自行车的一种可能实现：

```java
class Bicycle {

    int cadence = 0;
    int speed = 0;
    int gear = 1;

    void changeCadence(int newValue) {
         cadence = newValue;
    }

    void changeGear(int newValue) {
         gear = newValue;
    }

    void speedUp(int increment) {
         speed = speed + increment;   
    }

    void applyBrakes(int decrement) {
         speed = speed - decrement;
    }

    void printStates() {
         System.out.println("cadence:" +
             cadence + " speed:" + 
             speed + " gear:" + gear);
    }
}
```

Java 编程语言的语法对您来说可能是陌生的，但这个类的设计正是基于此前关于自行车对象的讨论。字段 `cadence`、`speed` 和 `gear` 表示对象的状态，而方法（`changeCadence`、`changeGear`、`speedUp` 等）定义了它与外部世界的交互方式。

您可能已经注意到，`Bicycle` 类中并不包含 `main` 方法。这是因为它不是一个完整的应用程序，它只是可能在应用程序中被*使用*的自行车的图纸。创建和使用新 `Bicycle` 对象的职责属于应用程序中的某个其他类。

下面这个 [`BicycleDemo`](https://docs.oracle.com/javase/tutorial/java/concepts/examples/BicycleDemo.java) 类创建了两个独立的 `Bicycle` 对象并调用它们的方法：

```java
class BicycleDemo {
    public static void main(String[] args) {

        // 创建两个不同的
        // Bicycle 对象
        Bicycle bike1 = new Bicycle();
        Bicycle bike2 = new Bicycle();

        // 对
        // 这些对象调用方法
        bike1.changeCadence(50);
        bike1.speedUp(10);
        bike1.changeGear(2);
        bike1.printStates();

        bike2.changeCadence(50);
        bike2.speedUp(10);
        bike2.changeGear(2);
        bike2.changeCadence(40);
        bike2.speedUp(10);
        bike2.changeGear(3);
        bike2.printStates();
    }
}
```

这个测试的输出打印了两辆自行车最终的踏频、速度和档位：

```
cadence:50 speed:10 gear:2
cadence:40 speed:20 gear:3
```
