## 为类提供构造方法

一个类包含构造方法(constructor)，它们被调用以根据类的蓝图创建对象。构造方法的声明与方法声明相似——只不过它们使用类的名称，并且没有返回类型。例如，`Bicycle` 有一个构造方法：

```text
public Bicycle(int startCadence, int startSpeed, int startGear) {
    gear = startGear;
    cadence = startCadence;
    speed = startSpeed;
}
```

要创建一个名为 `myBike` 的新 `Bicycle` 对象，需要由 `new` 运算符来调用一个构造方法：

```text
Bicycle myBike = new Bicycle(30, 0, 8);
```

`new Bicycle(30, 0, 8)` 在内存中为该对象分配空间，并初始化它的字段。

虽然 `Bicycle` 只有一个构造方法，但它可以有其它的构造方法，包括一个无参构造方法：

```text
public Bicycle() {
    gear = 1;
    cadence = 10;
    speed = 0;
}
```

`Bicycle yourBike = new Bicycle();` 调用无参构造方法来创建一个名为 `yourBike` 的新 `Bicycle` 对象。

这两个构造方法都可以在 `Bicycle` 中声明，因为它们有不同的参数列表。与方法一样，Java 平台根据参数列表中参数的数量和类型来区分构造方法。您不能为同一个类编写两个具有相同数量和类型参数的构造方法，因为平台将无法区分它们。这样做会导致编译时错误。

您不必为您的类提供任何构造方法，但在这样做时必须小心。对于任何没有构造方法的类，编译器会自动提供一个无参的默认构造方法(default constructor)。这个默认构造方法会调用超类的无参构造方法。在这种情况下，如果超类没有无参构造方法，编译器就会报错，因此您必须验证它确实有。如果您的类没有显式的超类，那么它有一个隐式的超类 `Object`，而 `Object` *确实*有无参构造方法。

您可以自行使用超类的构造方法。本课开头的 `MountainBike` 类正是这样做的。这将在稍后关于接口和继承的课程中讨论。

您可以在构造方法的声明中使用访问修饰符，以控制哪些其它类可以调用该构造方法。

---

**注意：** 如果另一个类无法调用 `MyClass` 的构造方法，它就无法直接创建 `MyClass` 对象。
