## 课程：接口

*核心集合接口*封装了不同类型的集合，如下图所示。这些接口允许独立于其表示的细节来操作集合。核心集合接口是 Java 集合框架的基础。正如下图所示，核心集合接口形成了一个层次结构。

![[collections-interfaces-colls-coreInterfaces.gif]]

核心集合接口。

`Set` 是一种特殊的 `Collection`，`SortedSet` 是一种特殊的 `Set`，依此类推。还要注意，该层次结构由两棵不同的树组成——`Map` 不是真正的 `Collection`。

注意，所有核心集合接口都是泛型的。例如，这是 `Collection` 接口的声明。

```java
public interface Collection<E>...
```

`<E>` 语法告诉你该接口是泛型的。当你声明 `Collection` 实例时，你可以*并且应该*指定集合中包含的对象类型。指定类型允许编译器（在编译时）验证你放入集合的对象类型是否正确，从而减少运行时错误。有关泛型类型的信息，请参阅[[学习Java语言-泛型|泛型（更新版）]]课程。

当你了解如何使用这些接口时，你就掌握了 Java 集合框架的大部分知识。本章讨论有效使用接口的一般准则，包括何时使用哪个接口。你还将学习每个接口的编程惯用法，帮助你充分利用它。

为了使核心集合接口的数量可控，Java 平台不为每种集合类型的每个变体提供单独的接口。（此类变体可能包括不可变、固定大小和仅追加。）相反，每个接口中的修改操作被指定为*可选的*——给定的实现可以选择不支持所有操作。如果调用了不支持的操作，集合会抛出 [`UnsupportedOperationException`](https://docs.oracle.com/javase/8/docs/api/java/lang/UnsupportedOperationException.html)。实现负责记录它们支持哪些可选操作。Java 平台的所有通用实现都支持所有可选操作。

以下列表描述了核心集合接口：

- `Collection` —— 集合层次结构的根。集合表示一组称为其*元素*的对象。`Collection` 接口是所有集合实现的最小公分母，用于在需要最大通用性时传递和操作集合。某些类型的集合允许重复元素，而其他则不允许。某些是有序的，其他是无序的。Java 平台不提供此接口的任何直接实现，但提供更特定子接口（如 `Set` 和 `List`）的实现。另请参阅 [[集合-接口-Collection接口|Collection 接口]]一节。
- `Set` —— 不能包含重复元素的集合。此接口对数学集合抽象进行建模，用于表示集合，例如构成一手扑克牌的卡片、组成学生课程的进程或在机器上运行的进程。另请参阅 [[集合-接口-Set接口|Set 接口]]一节。
- `List` —— 有序集合（有时称为*序列*）。`List` 可以包含重复元素。`List` 的用户通常对每个元素在列表中插入的位置有精确控制，并可以通过其整数索引（位置）访问元素。如果你使用过 `Vector`，你就熟悉 `List` 的一般风格。另请参阅 [[集合-接口-List接口|List 接口]]一节。
- `Queue` —— 用于在处理之前保存多个元素的集合。除了基本的 `Collection` 操作外，`Queue` 还提供额外的插入、提取和检查操作。
	队列通常（但不一定）以 FIFO（先进先出）方式对元素排序。例外情况包括优先队列，它们根据提供的比较器或元素的自然顺序对元素排序。无论使用什么排序，队列的头部都是通过调用 `remove` 或 `poll` 移除的元素。在 FIFO 队列中，所有新元素都插入到队列的尾部。其他类型的队列可能使用不同的放置规则。每个 `Queue` 实现必须指定其排序属性。另请参阅 [[集合-接口-Queue接口|Queue 接口]]一节。
- `Deque` —— 用于在处理之前保存多个元素的集合。除了基本的 `Collection` 操作外，`Deque` 还提供额外的插入、提取和检查操作。
	双端队列可以同时用作 FIFO（先进先出）和 LIFO（后进先出）。在双端队列中，所有新元素都可以在两端插入、检索和移除。另请参阅 [[集合-接口-Deque接口|Deque 接口]]一节。
- `Map` —— 将键映射到值的对象。`Map` 不能包含重复键；每个键最多可以映射到一个值。如果你使用过 `Hashtable`，你就已经熟悉 `Map` 的基础知识。另请参阅 [[集合-接口-Map接口|Map 接口]]一节。

最后两个核心集合接口只是 `Set` 和 `Map` 的排序版本：

- `SortedSet` —— 按升序维护其元素的 `Set`。提供了若干额外的操作以利用排序。排序集合用于自然排序的集合，例如单词列表和成员名册。另请参阅 [[集合-接口-SortedSet接口|SortedSet 接口]]一节。
- `SortedMap` —— 按升序键顺序维护其映射的 `Map`。这是 `SortedSet` 的 `Map` 类比。排序映射用于自然排序的键/值对集合，例如字典和电话簿。另请参阅 [[集合-接口-SortedMap接口|SortedMap 接口]]一节。

要了解排序接口如何维护其元素的顺序，请参阅[[集合-接口-对象排序|对象排序]]一节。
