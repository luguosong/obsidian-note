## 自动装箱和拆箱

*自动装箱（Autoboxing）*是 Java 编译器在基本类型及其对应的对象包装类之间进行的自动转换。例如，将 int 转换为 Integer，将 double 转换为 Double，等等。如果转换方向相反，则称为*拆箱（unboxing）*。

下面是自动装箱的最简单示例：

```java
Character ch = 'a';
```

本节的其余示例使用了泛型。如果你还不熟悉泛型的语法，请参见[泛型（更新版）（Generics (Updated)）](https://docs.oracle.com/javase/tutorial/java/generics/index.html)课程。

考虑下面的代码：

```java
List<Integer> li = new ArrayList<>();
for (int i = 1; i < 50; i += 2)
    li.add(i);
```

虽然你将 int 值作为基本类型而非 Integer 对象添加到 li 中，但代码可以编译。因为 li 是 Integer 对象的列表，而非 int 值的列表，你可能想知道为什么 Java 编译器没有发出编译时错误。编译器没有生成错误，是因为它从 i 创建了一个 Integer 对象并将该对象添加到 li 中。因此，编译器在运行时将前面的代码转换为以下内容：

```java
List<Integer> li = new ArrayList<>();
for (int i = 1; i < 50; i += 2)
    li.add(Integer.valueOf(i));
```

将一个基本值（例如 int）转换为相应包装类（Integer）的对象，称为自动装箱。当基本值被下列情形处理时，Java 编译器会应用自动装箱：

- 作为参数传递给一个期望相应包装类对象的方法。
- 赋值给相应包装类的变量。

考虑下面的方法：

```java
public static int sumEven(List<Integer> li) {
    int sum = 0;
    for (Integer i: li)
        if (i % 2 == 0)
            sum += i;
        return sum;
}
```

因为取余（%）和复合赋值（+=）运算符不适用于 Integer 对象，你可能想知道为什么 Java 编译器能编译该方法而不发出任何错误。编译器没有生成错误，是因为它在运行时调用了 intValue 方法将 Integer 转换为 int：

```java
public static int sumEven(List<Integer> li) {
    int sum = 0;
    for (Integer i : li)
        if (i.intValue() % 2 == 0)
            sum += i.intValue();
        return sum;
}
```

将一个包装类型（Integer）的对象转换为其相应的基本类型（int）值，称为拆箱。当包装类的对象被下列情形处理时，Java 编译器会应用拆箱：

- 作为参数传递给一个期望相应基本类型值的方法。
- 赋值给相应基本类型的变量。

[`Unboxing`](https://docs.oracle.com/javase/tutorial/java/data/examples/Unboxing.java) 示例展示了这是如何工作的：

```java
import java.util.ArrayList;
import java.util.List;

public class Unboxing {

    public static void main(String[] args) {
        Integer i = new Integer(-8);

        // 1. 通过方法调用拆箱
        int absVal = absoluteValue(i);
        System.out.println("absolute value of " + i + " = " + absVal);

        List<Double> ld = new ArrayList<>();
        ld.add(3.1416);    // Π 通过方法调用自动装箱。

        // 2. 通过赋值拆箱
        double pi = ld.get(0);
        System.out.println("pi = " + pi);
    }

    public static int absoluteValue(int i) {
        return (i < 0) ? -i : i;
    }
}
```

该程序打印如下内容：

```text
absolute value of -8 = 8
pi = 3.1416
```

自动装箱和拆箱使开发者能够编写更整洁的代码，使其更易于阅读。下表列出了基本类型及其对应的包装类，Java 编译器用它们来进行自动装箱和拆箱：

| 基本类型 | 包装类 |
| --- | --- |
| boolean | Boolean |
| byte | Byte |
| char | Character |
| float | Float |
| int | Integer |
| long | Long |
| short | Short |
| double | Double |
