## 作为运行时类型标记的类字面量

JDK 5.0 的更改之一是 `java.lang.Class` 类是泛型的。这是一个将泛型用于容器类以外的东西的有趣示例。

现在 `Class` 有类型参数 `T`，你可能会问，`T` 代表什么？它代表 `Class` 对象所表示的类型。

例如，`String.class` 的类型是 `Class<String>`，`Serializable.class` 的类型是 `Class<Serializable>`。这可用于提高反射代码的类型安全。

特别是，由于 `Class` 中的 `newInstance()` 方法现在返回 `T`，你可以在反射创建对象时获得更精确的类型。

例如，假设你需要编写一个实用工具方法，它执行以 SQL 字符串给出的数据库查询，并返回数据库中与该查询匹配的对象集合。

一种方法是显式传入一个工厂对象，编写如下代码：

```java
interface Factory<T> { T make();} 

public <T> Collection<T> select(Factory<T> factory, String statement) { 
    Collection<T> result = new ArrayList<T>(); 

    /* 使用 jdbc 运行 sql 查询 */  
    for (/* 遍历 jdbc 结果。 */) { 
        T item = factory.make();
        /* 使用反射并从 sql 结果
         * 设置 item 的所有字段。
         */ 
        result.add(item); 
    } 
    return result; 
}
```

你可以这样调用它

```java
select(new Factory<EmpInfo>(){ 
    public EmpInfo make() {
        return new EmpInfo();
    }}, "selection string");
```

或者你可以声明一个类 `EmpInfoFactory` 来支持 `Factory` 接口

```java
class EmpInfoFactory implements Factory<EmpInfo> {
    ...
    public EmpInfo make() { 
        return new EmpInfo();
    }
}
```

并调用它

```java
select(getMyEmpInfoFactory(), "selection string");
```

此解决方案的缺点是它需要：

- 在调用点使用冗长的匿名工厂类，或
- 为使用的每种类型声明一个工厂类并在调用点传递工厂实例，这有些不自然。

将类字面量作为工厂对象使用是很自然的，然后可以通过反射使用它。今天（没有泛型）代码可以写成：

```java
Collection emps = sqlUtility.select(EmpInfo.class, "select * from emps");
...
public static Collection select(Class c, String sqlStatement) { 
    Collection result = new ArrayList();
    /* 使用 jdbc 运行 sql 查询。 */
    for (/* 遍历 jdbc 结果。 */ ) { 
        Object item = c.newInstance(); 
        /* 使用反射并从 sql 结果
         * 设置 item 的所有字段。
         */  
        result.add(item); 
    } 
    return result; 
}
```

然而，这不会给我们所需精确类型的集合。现在 `Class` 是泛型的，我们可以改为编写以下内容：

```java
Collection<EmpInfo> 
    emps = sqlUtility.select(EmpInfo.class, "select * from emps");
...
public static <T> Collection<T> select(Class<T> c, String sqlStatement) { 
    Collection<T> result = new ArrayList<T>();
    /* 使用 jdbc 运行 sql 查询。 */
    for (/* 遍历 jdbc 结果。 */ ) { 
        T item = c.newInstance(); 
        /* 使用反射并从 sql 结果
         * 设置 item 的所有字段。
         */  
        result.add(item);
    } 
    return result; 
}
```

上面的代码以类型安全的方式为我们提供了精确类型的集合。

这种将类字面量用作运行时类型标记的技术是一个非常实用的技巧。例如，它是一种在操作注解的新 API 中广泛使用的惯用法。
