## 课程：实现

实现是用于存储集合的数据对象，它们实现了[[集合-接口|接口]]一节中描述的接口。本课描述以下几种实现：

- **通用实现** 是最常用的实现，为日常使用而设计。它们在标题为"通用实现"的表中汇总。
- **专用实现** 为在特殊情况下使用而设计，表现出非标准的性能特征、使用限制或行为。
- **并发实现** 为支持高并发而设计，通常以单线程性能为代价。这些实现是 `java.util.concurrent` 包的一部分。
- **包装器实现** 与其他类型的实现（通常是通用实现）结合使用，以提供附加或受限的功能。
- **便利实现** 是小型实现，通常通过静态工厂方法提供，为特殊集合（例如单例集合）提供通用实现的便利、高效替代方案。
- **抽象实现** 是促进自定义实现构建的骨架实现——稍后在[[集合-自定义实现|自定义集合实现]]一节中描述。这是一个高级主题，并不特别困难，但相对较少的人需要这样做。

通用实现在下表中汇总。

| 接口 | 哈希表实现 | 可调整数组实现 | 树实现 | 链表实现 | 哈希表+链表实现 |
| --- | --- | --- | --- | --- | --- |
| `Set` | `HashSet` |  | `TreeSet` |  | `LinkedHashSet` |
| `List` |  | `ArrayList` |  | `LinkedList` |  |
| `Queue` |  |  |  |  |  |
| `Deque` |  | `ArrayDeque` |  | `LinkedList` |  |
| `Map` | `HashMap` |  | `TreeMap` |  | `LinkedHashMap` |

从表中可以看出，Java 集合框架提供了 [`Set`](https://docs.oracle.com/javase/8/docs/api/java/util/Set.html)、[`List`](https://docs.oracle.com/javase/8/docs/api/java/util/List.html) 和 [`Map`](https://docs.oracle.com/javase/8/docs/api/java/util/Map.html) 接口的几种通用实现。在每种情况下，一个实现——[`HashSet`](https://docs.oracle.com/javase/8/docs/api/java/util/HashSet.html)、[`ArrayList`](https://docs.oracle.com/javase/8/docs/api/java/util/ArrayList.html) 和 [`HashMap`](https://docs.oracle.com/javase/8/docs/api/java/util/HashMap.html)——显然是大多数应用程序（在所有其他条件相同的情况下）应该使用的。注意，[`SortedSet`](https://docs.oracle.com/javase/8/docs/api/java/util/SortedSet.html) 和 [`SortedMap`](https://docs.oracle.com/javase/8/docs/api/java/util/SortedMap.html) 接口在表中没有行。这些接口各有一个实现 [(`TreeSet`](https://docs.oracle.com/javase/8/docs/api/java/util/TreeSet.html) 和 [`TreeMap`](https://docs.oracle.com/javase/8/docs/api/java/util/TreeMap.html))，并列在 `Set` 和 `Map` 行中。有两种通用的 `Queue` 实现——[`LinkedList`](https://docs.oracle.com/javase/8/docs/api/java/util/LinkedList.html)（它也是 `List` 实现）和 [`PriorityQueue`](https://docs.oracle.com/javase/8/docs/api/java/util/PriorityQueue.html)（从表中省略）。这两种实现提供非常不同的语义：`LinkedList` 提供 FIFO 语义，而 `PriorityQueue` 根据元素的值对元素排序。

每个通用实现都提供其接口中包含的所有可选操作。所有都允许 `null` 元素、键和值。都不是同步的（线程安全的）。所有都有*快速失败迭代器(fail-fast iterators)*，它们在迭代期间检测非法的并发修改，并快速干净地失败，而不是在未来某个不确定的时间冒着任意、非确定性行为的风险。所有都是 `Serializable` 的，并且都支持公共 `clone` 方法。

这些实现不同步的事实代表了与过去的决裂：遗留集合 `Vector` 和 `Hashtable` 是同步的。采取当前方法是因为集合经常在同步无益时使用。此类使用包括单线程使用、只读使用以及作为执行自身同步的较大数据对象的一部分使用。一般来说，良好的 API 设计实践是不让用户为他们不使用的功能付费。此外，不必要的同步在某些情况下可能导致死锁。

如果你需要线程安全的集合，[[集合-实现-包装器实现|包装器实现]]一节中描述的同步包装器允许将*任何*集合转换为同步集合。因此，同步对于通用实现是可选的，而对于遗留实现是强制的。此外，`java.util.concurrent` 包提供了 `BlockingQueue` 接口（扩展 `Queue`）和 `ConcurrentMap` 接口（扩展 `Map`）的并发实现。这些实现比单纯的同步实现提供高得多的并发性。

通常，你应该考虑的是接口而*不是*实现。这就是为什么本节没有编程示例的原因。在大多数情况下，实现的选择只影响性能。如[[集合-接口|接口]]一节中提到的，首选风格是在创建 `Collection` 时选择一个实现，并立即将新集合分配给相应接口类型的变量（或将集合传递给期望接口类型参数的方法）。这样，程序不依赖于给定实现中的任何附加方法，使程序员能够随时根据性能问题或行为细节自由地更改实现。

以下各节简要讨论这些实现。实现的性能使用*常量时间*、*对数*、*线性*、*n log(n)* 和*二次*等词来描述执行操作的时间复杂度的渐近上界。这一切听起来很复杂，如果你不知道它的意思也没关系。如果你有兴趣了解更多，请参阅任何好的算法教科书。需要记住的一件事是，这种性能指标有其局限性。有时，名义上较慢的实现可能更快。如有疑问，请测量性能！
