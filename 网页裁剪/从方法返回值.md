## 从方法返回值

一个方法在以下情况发生时（以先到者为准）返回到调用它的代码：

- 完成方法中的所有语句；
- 到达 `return` 语句；或
- 抛出异常（稍后介绍）。

您在方法声明中声明方法的返回类型。在方法体内，您使用 `return` 语句来返回值。

任何声明为 `void` 的方法都不返回值。它不需要包含 `return` 语句，但也可以包含。在这种情况下，`return` 语句可用于从一个控制流块中分支跳出并退出方法，其用法很简单：

```text
return;
```

如果您试图从一个声明为 `void` 的方法中返回一个值，将会得到一个编译器错误。

任何未声明为 `void` 的方法都必须包含一个带有相应返回值的 `return` 语句，如下所示：

```text
return returnValue;
```

返回值的数据类型必须与该方法声明的返回类型匹配；您不能从一个声明为返回 boolean 的方法中返回一个整数值。

关于对象的各节中讨论过的 `Rectangle` [`Rectangle`](https://docs.oracle.com/javase/tutorial/java/javaOO/examples/Rectangle.java) 类中的 `getArea()` 方法返回一个整数：

```text
// a method for computing the area of the rectangle
public int getArea() {
    return width * height;
}
```

该方法返回表达式 `width*height` 求值所得的整数。

`getArea` 方法返回一个基本类型。方法也可以返回一个引用类型。例如，在一个操纵 `Bicycle` 对象的程序中，我们可能有这样的方法：

```text
public Bicycle seeWhosFastest(Bicycle myBike, Bicycle yourBike,
                              Environment env) {
    Bicycle fastest;
    // code to calculate which bike is 
    // faster, given each bike's gear 
    // and cadence and given the 
    // environment (terrain and wind)
    return fastest;
}
```

## 返回类或接口

如果本节让您感到困惑，可以跳过它，等到学完关于接口和继承的课程之后再回来看。

当一个方法使用类名作为其返回类型时（例如 `whosFastest` 那样），所返回对象的类型必须是返回类型的子类，或与返回类型完全相同的类。假设您有一个类层次结构，其中 `ImaginaryNumber` 是 `java.lang.Number` 的子类，而 `Number` 又是 `Object` 的子类，如下图所示。

![[returnvalue-classes-hierarchy.gif]]

ImaginaryNumber 的类层次结构

现在假设您有一个声明为返回 `Number` 的方法：

```text
public Number returnANumber() {
    ...
}
```

`returnANumber` 方法可以返回一个 `ImaginaryNumber`，但不能返回一个 `Object`。`ImaginaryNumber` 是一个 `Number`，因为它是 `Number` 的子类。然而，一个 `Object` 不一定是 `Number`——它可能是一个 `String` 或其它类型。

您可以覆盖(override)一个方法，并将其定义为返回原方法返回类型的子类，如下所示：

```text
public ImaginaryNumber returnANumber() {
    ...
}
```

这种技术被称为*协变返回类型(covariant return type)*，意思是允许返回类型与子类沿相同方向变化。

---

**注意：** 您也可以使用接口名作为返回类型。在这种情况下，返回的对象必须实现指定的接口。
