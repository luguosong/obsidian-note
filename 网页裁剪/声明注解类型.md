## 声明注解类型

许多注解用来替代代码中的注释。

假设一个软件团队习惯在每一个类的类体开头用注释提供重要信息：

```java
public class Generation3List extends Generation2List {

   // Author: John Doe
   // Date: 3/17/2002
   // Current revision: 6
   // Last modified: 4/12/2004
   // By: Jane Doe
   // Reviewers: Alice, Bill, Cindy

   // class code goes here

}
```

要用注解添加同样的元数据，你必须首先定义*注解类型(annotation type)*。其语法如下：

```java
@interface ClassPreamble {
   String author();
   String date();
   int currentRevision() default 1;
   String lastModified() default "N/A";
   String lastModifiedBy() default "N/A";
   // Note use of array
   String[] reviewers();
}
```

注解类型定义看起来很像接口定义，只是关键字 `interface` 前面有一个 at 符号（`@`）（@ = AT，即 annotation type 中的 AT）。注解类型是一种*接口(interface)*形式，这将在后续课程中讲解。目前你还不需要理解接口。

上面注解定义的类体中包含若干*注解类型元素(annotation type element)*声明，它们看起来很像方法。注意它们可以定义可选的默认值。

定义注解类型之后，你就可以使用该类型的注解，并填入相应的值，如下所示：

```java
@ClassPreamble (
   author = "John Doe",
   date = "3/17/2002",
   currentRevision = 6,
   lastModified = "4/12/2004",
   lastModifiedBy = "Jane Doe",
   // Note array notation
   reviewers = {"Alice", "Bob", "Cindy"}
)
public class Generation3List extends Generation2List {

// class code goes here

}
```

---

**注意：** 要让 `@ClassPreamble` 中的信息出现在 Javadoc 生成的文档里，你必须用 `@Documented` 注解去标注 `@ClassPreamble` 的定义：

```java
// import this to use @Documented
import java.lang.annotation.*;

@Documented
@interface ClassPreamble {

   // Annotation element definitions
   
}
```
