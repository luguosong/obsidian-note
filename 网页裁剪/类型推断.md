## 类型推断

*类型推断（type inference）*是 Java 编译器查看每个方法调用及其相应声明以确定使调用适用的类型实参（一个或多个）的能力。推断算法会确定实参的类型，如果可用，还会确定结果被赋值或返回的类型。最后，推断算法试图找到适用于所有实参的*最具体*类型。

为说明最后这一点，在下面的示例中，推断确定传递给 pick 方法的第二个实参的类型是 Serializable：

```java
static <T> T pick(T a1, T a2) { return a2; }
Serializable s = pick("d", new ArrayList<String>());
```

## 类型推断与泛型方法

[泛型方法（Generic Methods）](https://docs.oracle.com/javase/tutorial/java/generics/methods.html)向你介绍了类型推断，它使你能够像调用普通方法一样调用泛型方法，而无需在尖括号之间指定类型。考虑下面的示例 [`BoxDemo`](https://docs.oracle.com/javase/tutorial/java/generics/examples/BoxDemo.java)，它需要 [`Box`](https://docs.oracle.com/javase/tutorial/java/generics/examples/Box.java) 类：

```java
public class BoxDemo {

  public static <U> void addBox(U u, 
      java.util.List<Box<U>> boxes) {
    Box<U> box = new Box<>();
    box.set(u);
    boxes.add(box);
  }

  public static <U> void outputBoxes(java.util.List<Box<U>> boxes) {
    int counter = 0;
    for (Box<U> box: boxes) {
      U boxContents = box.get();
      System.out.println("Box #" + counter + " contains [" +
             boxContents.toString() + "]");
      counter++;
    }
  }

  public static void main(String[] args) {
    java.util.ArrayList<Box<Integer>> listOfIntegerBoxes =
      new java.util.ArrayList<>();
    BoxDemo.<Integer>addBox(Integer.valueOf(10), listOfIntegerBoxes);
    BoxDemo.addBox(Integer.valueOf(20), listOfIntegerBoxes);
    BoxDemo.addBox(Integer.valueOf(30), listOfIntegerBoxes);
    BoxDemo.outputBoxes(listOfIntegerBoxes);
  }
}
```

下面是这个示例的输出：

```text
Box #0 contains [10]
Box #1 contains [20]
Box #2 contains [30]
```

泛型方法 `addBox` 定义了一个名为 `U` 的类型参数。通常，Java 编译器可以推断出泛型方法调用的类型参数。因此，在大多数情况下，你无需指定它们。例如，要调用泛型方法 `addBox`，你可以使用*类型见证（type witness）*指定类型参数，如下所示：

```java
BoxDemo.<Integer>addBox(Integer.valueOf(10), listOfIntegerBoxes);
```

或者，如果你省略类型见证，Java 编译器会自动（从方法的实参）推断出类型参数是 `Integer`：

```java
BoxDemo.addBox(Integer.valueOf(20), listOfIntegerBoxes);
```

## 类型推断与泛型类的实例化

只要编译器能够从上下文推断出类型实参，你就可以用一组空的类型参数（`<>`）来替换调用泛型类构造方法所需的类型实参。这对尖括号被非正式地称为[菱形（the diamond）](https://docs.oracle.com/javase/tutorial/java/generics/types.html#diamond)。

例如，考虑下面的变量声明：

```java
Map<String, List<String>> myMap = new HashMap<String, List<String>>();
```

你可以用一组空的类型参数（<>）替换构造方法的参数化类型：

```java
Map<String, List<String>> myMap = new HashMap<>();
```

注意，要在泛型类实例化期间利用类型推断，你必须使用菱形。在下面的示例中，编译器会生成一个未经检查的转换警告，因为 `HashMap()` 构造方法指的是 `HashMap` 原始类型，而非 `Map<String, List<String>>` 类型：

```java
Map<String, List<String>> myMap = new HashMap(); // 未经检查的转换警告
```

## 类型推断与泛型类及非泛型类的泛型构造方法

注意，构造方法可以是泛型的（换言之，声明自己的形式类型参数），无论是在泛型类还是非泛型类中。考虑下面的示例：

```java
class MyClass<X> {
  <T> MyClass(T t) {
    // ...
  }
}
```

考虑下面的 `MyClass` 实例化：

```java
new MyClass<Integer>("")
```

此语句创建了参数化类型 `MyClass<Integer>` 的一个实例；该语句为泛型类 `MyClass<X>` 的形式类型参数 `X` 显式指定了类型 `Integer`。注意，此泛型类的构造方法包含一个形式类型参数 `T`。编译器为此泛型类构造方法的形式类型参数 `T` 推断出类型 `String`（因为此构造方法的实际参数是一个 `String` 对象）。

Java SE 7 之前版本的编译器能够推断泛型构造方法的实际类型参数，类似于泛型方法。然而，Java SE 7 及更高版本中的编译器在你使用菱形（`<>`）时，可以推断出正在实例化的泛型类的实际类型参数。考虑下面的示例：

```java
MyClass<Integer> myObject = new MyClass<>("");
```

在此示例中，编译器为泛型类 `MyClass<X>` 的形式类型参数 `X` 推断出类型 `Integer`。它为此泛型类构造方法的形式类型参数 `T` 推断出类型 `String`。

---

**注意：** 重要的是，推断算法只使用调用实参、目标类型，以及可能一个明显的期望返回类型来推断类型。推断算法不使用程序后续部分的结果。

## 目标类型

Java 编译器利用目标类型（target typing）来推断泛型方法调用的类型参数。一个表达式的*目标类型（target type）*是 Java 编译器根据表达式出现的位置所期望的数据类型。考虑方法 `Collections.emptyList`，它声明如下：

```java
static <T> List<T> emptyList();
```

考虑下面的赋值语句：

```java
List<String> listOne = Collections.emptyList();
```

此语句期望一个 `List<String>` 的实例；此数据类型就是目标类型。因为方法 `emptyList` 返回类型为 `List<T>` 的值，所以编译器推断类型实参 `T` 必须是值 `String`。这在 Java SE 7 和 8 中都有效。或者，你可以使用类型见证并按如下方式指定 `T` 的值：

```java
List<String> listOne = Collections.<String>emptyList();
```

然而，在此上下文中这是不必要的。不过，在其他上下文中是必要的。考虑下面的方法：

```java
void processStringList(List<String> stringList) {
    // 处理 stringList
}
```

假设你想用一个空列表调用方法 `processStringList`。在 Java SE 7 中，下面的语句无法编译：

```java
processStringList(Collections.emptyList());
```

Java SE 7 编译器生成类似于下面的错误消息：

```text
List<Object> cannot be converted to List<String>
```

编译器需要类型实参 `T` 的一个值，因此它从值 `Object` 开始。因此，`Collections.emptyList` 的调用返回类型为 `List<Object>` 的值，这与方法 `processStringList` 不兼容。因此，在 Java SE 7 中，你必须按如下方式指定类型实参的值：

```java
processStringList(Collections.<String>emptyList());
```

这在 Java SE 8 中不再必要。关于什么是目标类型的概念已被扩展，以包含方法实参，例如方法 `processStringList` 的实参。在这种情况下，`processStringList` 需要一个类型为 `List<String>` 的实参。方法 `Collections.emptyList` 返回 `List<T>` 的值，因此使用 `List<String>` 这个目标类型，编译器推断类型实参 `T` 的值为 `String`。因此，在 Java SE 8 中，下面的语句可以编译：

```java
processStringList(Collections.emptyList());
```

有关更多信息，请参见 [Lambda 表达式（Lambda Expressions）](https://docs.oracle.com/javase/tutorial/java/javaOO/lambdaexpressions.html)中的[目标类型（Target Typing）](https://docs.oracle.com/javase/tutorial/java/javaOO/lambdaexpressions.html#target-typing)。
