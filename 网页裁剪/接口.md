## 接口

在软件工程中，有许多情形下，互不相关的程序员群体之间必须就一份规定其软件如何交互的"契约"达成一致，这一点至关重要。每个群体都应能在对另一群体代码如何编写毫无了解的情况下，独立编写自己的代码。一般而言，*接口(interface)* 就是这样的契约。

例如，设想一个未来社会，由计算机控制的机器人汽车在没有人类驾驶员操作的情况下，载着乘客穿梭于城市街道。汽车制造商编写（当然是用 Java）控制汽车运行的软件——停车、启动、加速、左转等等。另一个工业群体，即电子导航仪器制造商，生产能够接收 GPS（全球定位系统，Global Positioning System）位置数据和交通状况无线传输信息的计算机系统，并利用这些信息来驾驶汽车。

汽车制造商必须发布一份行业标准接口，详细规定可以调用哪些方法来使汽车行驶（任何制造商生产的任何汽车）。随后，导航仪器制造商就可以编写软件，调用接口中所描述的方法来指挥汽车。两个工业群体都无需知道对方软件是*如何*实现的。事实上，每个群体都将其软件视为高度专有资产，并保留随时对其进行修改的权利，只要它继续遵循所发布的接口即可。

## Java 中的接口

在 Java 编程语言中，*接口* 是一种引用类型，类似于类，它*只能*包含常量、方法签名、默认方法、静态方法和嵌套类型。方法体仅存在于默认方法和静态方法中。接口不能被实例化——它们只能被类*实现(implement)*，或被其他接口*扩展(extend)*。扩展将在本课程后续讨论。

定义接口与创建新类类似：

```java
public interface OperateCar {

   // 常量声明（如果有）

   // 方法签名
   
   // 一个取值为 RIGHT、LEFT 的枚举
   int turn(Direction direction,
            double radius,
            double startSpeed,
            double endSpeed);
   int changeLanes(Direction direction,
                   double startSpeed,
                   double endSpeed);
   int signalTurn(Direction direction,
                  boolean signalOn);
   int getRadarFront(double distanceToCar,
                     double speedOfCar);
   int getRadarRear(double distanceToCar,
                    double speedOfCar);
         ......
   // 更多方法签名
}
```

注意，方法签名没有大括号，并以分号结尾。

要使用接口，你需要编写一个*实现*该接口的类。当一个可实例化的类实现了某个接口时，它必须为接口中声明的每个方法提供方法体。例如，

```java
public class OperateBMW760i implements OperateCar {

    // OperateCar 方法签名及其实现 ——
    // 例如：
    public int signalTurn(Direction direction, boolean signalOn) {
       // 点亮 BMW 左转向指示灯的代码
       // 熄灭 BMW 左转向指示灯的代码
       // 点亮 BMW 右转向指示灯的代码
       // 熄灭 BMW 右转向指示灯的代码
    }

    // 视需要的其他成员 —— 例如，对接口客户端不可见的
    // 辅助类
}
```

在上面的机器人汽车示例中，正是汽车制造商来实现该接口。当然，雪佛兰(Chevrolet)的实现将与丰田(Toyota)的实现大不相同，但两家制造商都会遵循同一个接口。作为接口客户端的导航仪器制造商，将构建使用汽车位置的 GPS 数据、数字街道地图和交通数据来驾驶汽车的系统。在此过程中，导航系统会调用接口方法：转弯、变道、刹车、加速等等。

## 作为 API 的接口

上面的机器人汽车示例展示了一个接口被用作行业标准*应用程序编程接口(Application Programming Interface, API)*的情形。API 在商业软件产品中也很常见。通常，一家公司销售一个包含复杂方法的软件包，而另一家公司希望在自己的软件产品中使用这些方法。一个例子是，一组数字图像处理方法被卖给那些开发面向最终用户的图形程序的公司。图像处理公司编写自己的类来实现某个接口，并将该接口向其客户公开。随后，图形公司使用接口中定义的签名和返回类型来调用这些图像处理方法。虽然图像处理公司的 API 是公开的（向其客户公开），但其 API 的实现却被作为一项严加守护的秘密——事实上，只要它继续实现其客户所依赖的原始接口，它就可以在日后修改其实现。
