## 学习路径：扩展机制

扩展机制（extension mechanism）提供了一种标准、可扩展的方式，使自定义 API 可供运行在 Java 平台上的所有应用程序使用。*Java 扩展（Java extensions）* 也称为*可选包（optional packages）*。本学习路径中两种说法可互换使用。

*扩展（Extensions）* 是通过扩展机制增强 Java 平台的包和类的集合。扩展机制使运行时环境能够查找并加载扩展类，而无需扩展类在类路径（class path）上被命名。就此而言，扩展类类似于 Java 平台的核心类。这也是「扩展」一词的由来——它们实际上扩展了平台的核心 API。

由于该机制扩展了平台的核心 API，因此应审慎使用。最常见的情况是用于标准化良好的接口，例如由 Java 社区过程（Java Community Process）定义的接口，不过它也可能适用于站点范围的接口。

![[java-tutorial-exta.gif]]

如图所示，扩展充当 Java 平台的「附加（add-on）」模块。它们的类和公共 API 对运行在该平台上的任何应用程序都是自动可用的。

扩展机制还提供了一种方式，使扩展类可以从远程位置下载供 applet 使用。

扩展以 Java 归档（JAR）文件的形式打包，本学习路径假定您已熟悉 JAR 文件格式。如果您对 JAR 文件还不熟练，在继续学习本学习路径的课程之前，您可能需要复习一些 JAR 文件文档：

- 本教程中的 [用 JAR 文件打包程序（Packaging Programs in JAR Files）](https://docs.oracle.com/javase/tutorial/deployment/jar/index.html) 课程。
- JDK™ 文档中的 [JAR 指南（JAR Guide）](https://docs.oracle.com/javase/8/docs/technotes/guides/jar/jarGuide.html)。
