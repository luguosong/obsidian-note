## 路线：扩展机制

扩展机制提供了一种标准、可扩展的方式，使自定义 API 可供运行在 Java 平台上的所有应用程序使用。*Java 扩展(Java extensions)* 也被称为*可选包(optional packages)*。本路线中这两个术语可能会互换使用。

*扩展*是通过扩展机制来增强 Java 平台的包和类的集合。扩展机制使运行时环境能够查找并加载扩展类，而无需在类路径上命名扩展类。在这方面，扩展类类似于 Java 平台的核心类。这也是扩展名称的由来——它们实际上扩展了平台的核心 API。

由于此机制扩展了平台的核心 API，其使用应谨慎。它最常用于标准化良好的接口，例如由 Java 社区进程(Java Community Process)定义的接口，不过它也可能适用于站点范围的接口。

![[ext-exta.gif]]

如图所示，扩展充当 Java 平台的"附加"模块。它们的类和公共 API 自动可供运行在该平台上的任何应用程序使用。

扩展机制还提供了一种方式，使扩展类可以从远程位置下载以供 applet 使用。

扩展以 Java 归档(JAR)文件的形式打包，本路线假设你已熟悉 JAR 文件格式。如果你对 JAR 文件尚不熟悉，在继续本路线的课程之前，你可能需要查阅一些 JAR 文件文档：
