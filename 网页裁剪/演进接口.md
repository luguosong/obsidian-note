## 演进接口

考虑一个你已经开发的、名为 `DoIt` 的接口：

```java
public interface DoIt {
   void doSomething(int i, double x);
   int doSomethingElse(String s);
}
```

假设在稍后的某个时间，你想为 `DoIt` 增加第三个方法，使该接口变为：

```java
public interface DoIt {

   void doSomething(int i, double x);
   int doSomethingElse(String s);
   boolean didItWork(int i, double x, String s);
   
}
```

如果你做了这样的修改，那么所有实现了旧 `DoIt` 接口的类都将因为不再实现旧接口而被破坏。依赖此接口的程序员将会强烈抗议。

尽量预见接口的所有用途，并从一开始就完整地规定它。如果你想为接口添加额外的方法，你有几种选择。你可以创建一个扩展 `DoIt` 的 `DoItPlus` 接口：

```java
public interface DoItPlus extends DoIt {

   boolean didItWork(int i, double x, String s);
   
}
```

这样一来，你的代码的用户可以选择继续使用旧接口，或者升级到新接口。

或者，你可以将新方法定义为[默认方法（default method）](https://docs.oracle.com/javase/tutorial/java/IandI/defaultmethods.html)。下面的示例定义了一个名为 `didItWork` 的默认方法：

```java
public interface DoIt {

   void doSomething(int i, double x);
   int doSomethingElse(String s);
   default boolean didItWork(int i, double x, String s) {
       // 方法体
   }
   
}
```

注意，你必须为默认方法提供实现。你还可以为现有接口定义新的[静态方法（static method）](https://docs.oracle.com/javase/tutorial/java/IandI/defaultmethods.html#static)。那些拥有实现了被新默认方法或静态方法增强的接口的类的用户，无需为了适配这些额外的方法而修改或重新编译它们的类。
