## 理解类成员

本节讨论如何使用 `static` 关键字来创建属于类（而非属于类的某个实例）的字段和方法。

## 类变量

当从同一个类蓝图创建多个对象时，它们各自拥有*实例变量(instance variable)*的独立副本。在 `Bicycle` 类的情况下，实例变量是 `cadence`、`gear` 和 `speed`。每个 `Bicycle` 对象都有自己对这些变量的取值，存储在不同的内存位置。

有时，您希望拥有所有对象共用的变量。这通过 `static` 修饰符来实现。声明中带有 `static` 修饰符的字段被称为*静态字段(static field)*或*类变量(class variable)*。它们与类相关联，而不是与任何对象相关联。类的每个实例都共享一个类变量，该变量位于内存中的某个固定位置。任何对象都可以更改类变量的值，但也可以在不创建类的实例的情况下操纵类变量。

例如，假设您想创建若干个 `Bicycle` 对象，并为每个对象分配一个序列号，从第一个对象的 1 开始。这个 ID 编号对每个对象都是唯一的，因此是一个实例变量。同时，您需要一个字段来跟踪已经创建了多少个 `Bicycle` 对象，以便知道该给下一个对象分配什么 ID。这样的字段与任何单独的对象无关，而是与整个类相关。为此，您需要一个类变量 `numberOfBicycles`，如下所示：

```java
public class Bicycle {
        
    private int cadence;
    private int gear;
    private int speed;
        
    // add an instance variable for the object ID
    private int id;
    
    // add a class variable for the
    // number of Bicycle objects instantiated
    private static int numberOfBicycles = 0;
        ...
}
```

类变量通过类名本身来引用，例如

```text
Bicycle.numberOfBicycles
```

这能清楚地表明它们是类变量。

---

**注意：** 您也可以通过对象引用来引用静态字段，例如

```text
myBike.numberOfBicycles
```

但不鼓励这样做，因为这无法清楚地表明它们是类变量。

---

您可以使用 `Bicycle` 构造方法来设置 `id` 实例变量，并递增 `numberOfBicycles` 类变量：

```java
public class Bicycle {
        
    private int cadence;
    private int gear;
    private int speed;
    private int id;
    private static int numberOfBicycles = 0;
        
    public Bicycle(int startCadence, int startSpeed, int startGear){
        gear = startGear;
        cadence = startCadence;
        speed = startSpeed;

        // increment number of Bicycles
        // and assign ID number
        id = ++numberOfBicycles;
    }

    // new method to return the ID instance variable
    public int getID() {
        return id;
    }
        ...
}
```

## 类方法

Java 编程语言既支持静态变量，也支持静态方法。静态方法在其声明中带有 `static` 修饰符，应当通过类名来调用，而无需创建类的实例，例如

```text
ClassName.methodName(args)
```

---

**注意：** 您也可以通过对象引用来引用静态方法，例如

```text
instanceName.methodName(args)
```

但不鼓励这样做，因为这无法清楚地表明它们是类方法。

---

静态方法的一个常见用途是访问静态字段。例如，我们可以在 `Bicycle` 类中添加一个静态方法来访问 `numberOfBicycles` 静态字段：

```text
public static int getNumberOfBicycles() {
    return numberOfBicycles;
}
```

并非实例变量与类变量、实例方法与类方法的所有组合都是允许的：

- 实例方法可以直接访问实例变量和实例方法。
- 实例方法可以直接访问类变量和类方法。
- 类方法可以直接访问类变量和类方法。
- 类方法**不能**直接访问实例变量或实例方法——它们必须使用对象引用。此外，类方法不能使用 `this` 关键字，因为没有实例可供 `this` 引用。

## 常量

`static` 修饰符与 `final` 修饰符结合使用，也可用于定义常量。`final` 修饰符表示该字段的值不能改变。

例如，下面的变量声明定义了一个名为 `PI` 的常量，其值是圆周率 pi（圆的周长与其直径之比）的近似值：

```java
static final double PI = 3.141592653589793;
```

以这种方式定义的常量不能被重新赋值，如果程序尝试这样做，将产生编译时错误。按照约定，常量值的名称使用大写字母拼写。如果名称由多个单词组成，则单词之间用下划线（`_`）分隔。

---

**注意：** 如果一个基本类型或字符串被定义为常量，且其值在编译时已知，编译器会将代码中所有出现该常量名的地方替换为其值。这被称为*编译时常量(compile-time constant)*。如果外部世界中该常量的值发生了变化（例如，立法规定 pi 实际上应为 3.975），您将需要重新编译所有使用该常量的类，才能获得当前值。

---

## Bicycle 类

经过本节所做的所有修改之后，`Bicycle` 类现在如下：

```java
public class Bicycle {
        
    private int cadence;
    private int gear;
    private int speed;
        
    private int id;
    
    private static int numberOfBicycles = 0;

        
    public Bicycle(int startCadence,
                   int startSpeed,
                   int startGear) {
        gear = startGear;
        cadence = startCadence;
        speed = startSpeed;

        id = ++numberOfBicycles;
    }

    public int getID() {
        return id;
    }

    public static int getNumberOfBicycles() {
        return numberOfBicycles;
    }

    public int getCadence() {
        return cadence;
    }
        
    public void setCadence(int newValue) {
        cadence = newValue;
    }
        
    public int getGear(){
        return gear;
    }
        
    public void setGear(int newValue) {
        gear = newValue;
    }
        
    public int getSpeed() {
        return speed;
    }
        
    public void applyBrake(int decrement) {
        speed -= decrement;
    }
        
    public void speedUp(int increment) {
        speed += increment;
    }
}
```
