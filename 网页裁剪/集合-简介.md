## 课程：集合简介

*集合(collection)*——有时称为容器——就是将多个元素组合成单个单元的对象。集合用于存储、检索、操作和传递聚合数据。通常，它们表示形成自然分组的数据项，例如一手扑克牌（卡片的集合）、邮件文件夹（信件的集合）或电话簿（从姓名到电话号码的映射）。如果你使用过 Java 编程语言——或几乎任何其他编程语言——你都已经熟悉集合了。

## 什么是集合框架？

*集合框架(collections framework)* 是表示和操作集合的统一架构。所有集合框架都包含以下内容：

- **接口：** 这些是表示集合的抽象数据类型。接口允许独立于其表示的细节来操作集合。在面向对象语言中，接口通常形成层次结构。
- **实现：** 这些是集合接口的具体实现。本质上，它们是可重用的数据结构。
- **算法：** 这些是在实现集合接口的对象上执行有用计算（如搜索和排序）的方法。这些算法被称为*多态的(polymorphic)*：即，同一方法可用于相应集合接口的许多不同实现。本质上，算法是可重用的功能。

除 Java 集合框架外，最著名的集合框架示例是 C++ 标准模板库(STL) 和 Smalltalk 的集合层次结构。从历史上看，集合框架相当复杂，这使它们以学习曲线陡峭而闻名。我们相信 Java 集合框架打破了这一传统，正如你将在本章中自己学到的那样。

## Java 集合框架的好处

Java 集合框架提供以下好处：

- **减少编程工作量：** 通过提供有用的数据结构和算法，集合框架使你能专注于程序的重要部分，而不是使其工作所需的低级"管道"。通过促进不相关 API 之间的互操作性，Java 集合框架使你免于编写适配器对象或转换代码来连接 API。
- **提高程序速度和质量：** 此集合框架提供了有用数据结构和算法的高性能、高质量实现。每个接口的各种实现是可互换的，因此可以通过切换集合实现轻松调整程序。由于你不必编写自己的数据结构，你将有更多时间致力于提高程序的质量和性能。
- **允许不相关 API 之间的互操作性：** 集合接口是 API 来回传递集合的通用语言。如果我的网络管理 API 提供节点名称的集合，而你的 GUI 工具包期望列标题的集合，我们的 API 将无缝互操作，即使它们是独立编写的。
- **减少学习和使用新 API 的工作量：** 许多 API 自然地接受集合作为输入并将其作为输出提供。过去，每个这样的 API 都有一个专门用于操作其集合的小型子 API。这些临时集合子 API 之间几乎没有一致性，因此你必须从头学习每一个，而且使用它们时很容易出错。随着标准集合接口的出现，这个问题消失了。
- **减少设计新 API 的工作量：** 这是前一个优势的反面。设计者和实现者不必每次创建依赖集合的 API 时都重新发明轮子；相反，他们可以使用标准集合接口。
- **促进软件复用：** 符合标准集合接口的新数据结构本质上是可重用的。对于操作实现这些接口的对象的新算法也是如此。
