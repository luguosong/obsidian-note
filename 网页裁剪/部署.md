## 学习路径：部署

Java 富互联网应用程序（Rich Internet Applications，RIA）是具有类似桌面应用程序特征、但通过互联网部署的应用程序。Java RIA 可以作为 Java applet 或 Java Web Start 应用程序进行开发和部署。

- applet —— Java applet 在浏览器的上下文中运行。Java 插件（Java Plug-in）软件控制 Java applet 的执行和生命周期。
- Java Web Start 应用程序 —— Java Web Start 应用程序首次通过浏览器启动。之后可以从桌面快捷方式启动。一旦 Java Web Start 应用程序被下载且其安全证书被用户接受，它的行为就*几乎*与独立应用程序一样。

## RIA 的基于组件的架构

过去，决定将 Java 富互联网应用程序作为 applet 部署在浏览器内、还是作为 Java Web Start 应用程序部署在浏览器外，可能会显著影响应用程序的设计。借助最新的 Java 插件，这一决策已被大大简化。

传统上，应用程序在 `main` 方法中构建其用户界面，包括顶层 `Frame`。这种编程风格阻碍了应用程序在浏览器中的轻松重新部署，因为它假定应用程序创建自己的 `Frame`。当作为 applet 在浏览器中运行时，applet 是承载应用程序用户界面的顶层容器。不需要顶层 `Frame`。

在设计 Java 富互联网应用程序时，请使用*基于组件的架构（component-based architecture）*。尝试将其功能组织为一个或多个可组合在一起的组件。在此语境下，「组件（component）」一词指的是 AWT `Component` 类、Swing `JComponent` 类或其另一个子类的 GUI 元素。例如，您可以有一个顶层 `JPanel`，其中包含其他 UI 组件（如更多嵌套的 JPanel 和文本字段、组合框等的组合）。采用这样的设计，将核心功能部署为 applet 或 Java Web Start 应用程序就变得相对容易。

要部署为 Java applet，您只需将核心功能封装在 `Applet` 或 `JApplet` 中，并在必要时添加浏览器特定的功能。要部署为 Java Web Start 应用程序，则将功能封装在 `JFrame` 中。

## 在 Java applet 和 Java Web Start 应用程序之间做选择

[富互联网应用程序决策指南（Rich Internet Applications Decision Guide）](https://docs.oracle.com/javase/tutorial/deployment/_riaDecisionGuide.html) 包含详细信息，帮助您决定将代码部署为 Java applet 还是 Java Web Start 应用程序。

## 独立式应用程序（Self-Contained Application）这一替代方案

独立式应用程序提供了一种不需要浏览器的部署选项。用户在本地安装您的应用程序并像原生应用程序一样运行。独立式应用程序包含运行应用程序所需的 JRE，因此用户始终拥有正确的 JRE。

本学习路径讨论 RIA 和独立式应用程序的开发与部署。如需了解客户端 Java 运行时环境（JRE）软件各版本引入的功能，请参见 [新特性（What's New）](https://docs.oracle.com/javase/8/docs/technotes/guides/deploy/whatsnew_deployment.html)。
