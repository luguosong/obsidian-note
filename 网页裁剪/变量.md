## 变量

正如您在上一课中所学到的，对象将其状态存储在*字段(field)*中。

```java
int cadence = 0;
int speed = 0;
int gear = 1;
```

[什么是对象（What Is an Object?）](https://docs.oracle.com/javase/tutorial/java/concepts/object.html)的讨论已经向您介绍了字段，但您可能仍有几个疑问，例如：命名字段有哪些规则和约定？除了 `int`，还有哪些数据类型？字段在声明时必须初始化吗？如果没有显式初始化，字段会被赋予默认值吗？我们将在本课中探讨这些问题的答案，但在那之前，您必须先了解几个技术上的区别。在 Java 编程语言中，「字段(field)」和「变量(variable)」这两个术语都会使用；这是新开发者中常见的混淆来源，因为两者似乎常常指的是同一回事。

Java 编程语言定义了以下几种变量：

- **实例变量（Instance Variables，非静态字段）** 从技术上讲，对象将各自的状态存储在「非静态字段(non-static field)」中，也就是声明时不带 `static` 关键字的字段。非静态字段也称为*实例变量(instance variable)*，因为它们的值对类的每个*实例*（换言之，对每个对象）都是唯一的；一辆自行车的 `currentSpeed` 与另一辆自行车的 `currentSpeed` 相互独立。
- **类变量（Class Variables，静态字段）** *类变量(class variable)* 是用 `static` 修饰符声明的任何字段；它告诉编译器，无论该类被实例化多少次，这个变量都只存在一份副本。定义某种特定类型自行车挡位数目的字段可以标记为 `static`，因为从概念上讲，相同数目的挡位适用于所有实例。代码 `static int numGears = 6;` 就会创建这样一个静态字段。此外，可以添加关键字 `final` 来表示挡位数永远不会改变。
- **局部变量（Local Variables）** 类似于对象将其状态存储在字段中，方法通常会将其临时状态存储在*局部变量(local variable)*中。声明局部变量的语法与声明字段类似（例如 `int count = 0;`）。没有任何特殊关键字将一个变量指定为局部变量；这一判定完全取决于变量声明的位置——即位于一对方法的开括号和闭括号之间。因此，局部变量仅对声明它们的方法可见；它们不能从类的其余部分访问。
- **参数（Parameters）** 您已经见过参数的示例，无论是在 `Bicycle` 类中还是在「Hello World!」应用程序的 `main` 方法中。回想一下，`main` 方法的签名是 `public static void main(String[] args)`。这里，`args` 变量就是该方法的参数。需要记住的重要一点是：参数始终被归类为「变量」而非「字段」。这也适用于其它接受参数的构造（如构造器和异常处理器），您将在本教程后续部分学习到它们。

话虽如此，本教程的其余部分在讨论字段和变量时遵循以下一般准则。如果我们谈论的是「一般意义上的字段」（不包括局部变量和参数），我们可能就直接说「字段」。如果讨论适用于「上述所有情况」，我们可能就直接说「变量」。如果上下文需要区分，我们会酌情使用具体术语（静态字段、局部变量等）。您偶尔也会看到「成员(member)」这个词被使用。一个类型的字段、方法和嵌套类型合称为它的*成员(member)*。

## 命名

每种编程语言都有自己的一套规则和约定，规定您可以使用哪些名称，Java 编程语言也不例外。命名变量的规则和约定可总结如下：

- 变量名区分大小写。变量名可以是任何合法的标识符(identifier)——一个不限长度的 Unicode 字母和数字序列，以字母、美元符号「`$`」或下划线字符「`_`」开头。不过，约定俗成的做法是始终以字母（而非「`$`」或「`_`」）作为变量名的开头。此外，按照约定，美元符号字符根本不会被使用。您可能会遇到一些自动生成的名称包含美元符号的情况，但您的变量名应始终避免使用它。下划线字符也有类似的约定；虽然从技术上讲以「`_`」作为变量名开头是合法的，但不鼓励这种做法。变量名中不允许出现空格。
- 后续字符可以是字母、数字、美元符号或下划线字符。约定（以及常识）同样适用于此规则。为变量命名时，请使用完整的单词，而不是晦涩的缩写。这样做会使您的代码更易于阅读和理解。在许多情况下，它还会使您的代码具备自说明性(self-documenting)；例如，命名为 `cadence`、`speed` 和 `gear` 的字段比 `s`、`c` 和 `g` 这样的缩写版本直观得多。还要记住，您选择的名称不能是[关键字或保留字（keyword or reserved word）](https://docs.oracle.com/javase/tutorial/java/nutsandbolts/_keywords.html)。
- 如果您选择的名称只包含一个单词，请将该单词全部以小写字母拼写。如果它包含多个单词，请将每个后续单词的首字母大写。名称 `gearRatio` 和 `currentGear` 就是这种约定的典型示例。如果您的变量存储的是常量值，例如 `static final int NUM_GEARS = 6`，约定会有所不同：每个字母都大写，并用下划线字符分隔后续单词。按照约定，下划线字符不会在其它任何地方使用。
