## 编译器问题 (Compiler Problems)

**Microsoft Windows 系统上的常见错误信息**

`**'javac' is not recognized as an internal or external command, operable program or batch file**`

如果你收到此错误，说明 Windows 找不到编译器（`javac`）。

下面是一种告诉 Windows 在哪里能找到 `javac` 的方法。假设你将 JDK 安装在 `C:\jdk1.8.0`。在提示符下，你会输入以下命令并按 Enter：

```
C:\jdk1.8.0\bin\javac HelloWorldApp.java
```

如果你选择此方案，每次编译或运行程序时都必须在 `javac` 和 `java` 命令前加上 `C:\jdk1.8.0\bin\`。为避免这些额外输入，请参阅 JDK 8 安装说明中的[更新 PATH 变量](https://docs.oracle.com/javase/8/docs/technotes/guides/install/windows_jdk_install.html#BABGDJFH)一节。

`**Class names, 'HelloWorldApp', are only accepted if annotation processing is explicitly requested**`

如果你收到此错误，说明你在编译程序时忘记包含 `.java` 后缀。请记住，命令是 `javac HelloWorldApp.java` 而非 `javac HelloWorldApp`。

**UNIX 系统上的常见错误信息**

`javac: Command not found`

如果你收到此错误，说明 UNIX 找不到编译器 `javac`。

下面是一种告诉 UNIX 在哪里能找到 `javac` 的方法。假设你将 JDK 安装在 `/usr/local/jdk1.8.0`。在提示符下，你会输入以下命令并按 Return：

```
/usr/local/jdk1.8.0/javac HelloWorldApp.java
```

**注意：** 如果你选择此方案，每次编译或运行程序时都必须在 `javac` 和 `java` 命令前加上 `/usr/local/jdk1.8.0/`。为避免这些额外输入，你可以将此信息添加到你的 PATH 变量中。具体步骤因你当前所运行的 shell 而异。

`**Class names, 'HelloWorldApp', are only accepted if annotation processing is explicitly requested**`

如果你收到此错误，说明你在编译程序时忘记包含 `.java` 后缀。请记住，命令是 `javac HelloWorldApp.java` 而非 `javac HelloWorldApp`。

**语法错误（所有平台） (Syntax Errors (All Platforms))**

如果你输错了程序的某一部分，编译器可能会发出一个*语法(syntax)*错误。该消息通常显示错误的类型、检测到错误的行号、该行上的代码，以及代码中错误的位置。下面是一个由在语句末尾省略分号（`;`）引起的错误：

```
Testing.java:8: error: ';' expected
            count++
                   ^
1 error
```

如果你看到任何编译器错误，说明你的程序没有成功编译，编译器也没有创建 `.class` 文件。请仔细检查程序，修正你发现的任何错误，然后再试一次。

**语义错误 (Semantic Errors)**

除了验证你的程序在语法上是否正确外，编译器还会检查其他基本的正确性。例如，每次你使用一个尚未初始化的变量时，编译器都会发出警告：

```java
Testing.java:8: error: variable count might not have been initialized
            count++;
            ^
Testing.java:9: error: variable count might not have been initialized
        System.out.println("Input has " + count + " chars.");
                                          ^
2 errors
```

同样，你的程序没有成功编译，编译器也没有创建 `.class` 文件。修正错误后再试一次。

## 运行时问题 (Runtime Problems)

Microsoft Windows 系统上的错误信息

`Exception in thread "main" java.lang.NoClassDefFoundError: HelloWorldApp`

如果你收到此错误，说明 `java` 找不到你的字节码文件 `HelloWorldApp.class`。

`java` 尝试查找你的 `.class` 文件的位置之一是你的当前目录。因此，如果你的 `.class` 文件在 `C:\java` 中，你应当将当前目录更改为该目录。要更改目录，在提示符下输入以下命令并按 Enter：

```
cd c:\java
```

提示符应当变为 `C:\java>`。如果你在提示符下输入 `dir`，应该能看到你的 `.java` 和 `.class` 文件。现在再次输入 `java HelloWorldApp`。

如果仍有问题，你可能需要更改 CLASSPATH 变量。要确认是否有此必要，请尝试用以下命令清除类路径：

```batch
set CLASSPATH=
```

现在再次输入 `java HelloWorldApp`。如果程序现在可以运行，你就需要更改 CLASSPATH 变量。要设置此变量，请参阅 JDK 8 安装说明中的[更新 PATH 变量](https://docs.oracle.com/javase/8/docs/technotes/guides/install/windows_jdk_install.html#BABGDJFH)一节。CLASSPATH 变量的设置方式与之相同。

`** Could not find or load main class HelloWorldApp.class**`

初学程序员常犯的一个错误是，尝试对编译器创建的 `.class` 文件运行 `java` 启动器。例如，如果你尝试用 `java HelloWorldApp.class` 而非 `java HelloWorldApp` 来运行程序，就会收到此错误。请记住，参数是你想要使用的*类名*，*而非*文件名。

**`Exception in thread "main" java.lang.NoSuchMethodError: main`**

Java 虚拟机要求你用它执行的类拥有一个 `main` 方法，作为应用程序开始执行的位置。[[深入剖析HelloWorld应用程序|深入剖析 "Hello World!" 应用程序]] 对 `main` 方法进行了详细讨论。

**UNIX 系统上的错误信息**

`Exception in thread "main" java.lang.NoClassDefFoundError: HelloWorldApp`

如果你收到此错误，说明 `java` 找不到你的字节码文件 `HelloWorldApp.class`。

`java` 尝试查找你的字节码文件的位置之一是你的当前目录。例如，如果你的字节码文件在 `/home/jdoe/java` 中，你应当将当前目录更改为该目录。要更改目录，在提示符下输入以下命令并按 Return：

```
cd /home/jdoe/java
```

如果你在提示符下输入 `pwd`，应该能看到 `/home/jdoe/java`。如果你在提示符下输入 `ls`，应该能看到你的 `.java` 和 `.class` 文件。现在再次输入 `java HelloWorldApp`。

如果仍有问题，你可能需要更改 CLASSPATH 环境变量。要确认是否有此必要，请尝试用以下命令清除类路径：

```
unset CLASSPATH
```

现在再次输入 `java HelloWorldApp`。如果程序现在可以运行，你就需要按照上面 PATH 变量的相同方式更改 CLASSPATH 变量。

`**Exception in thread "main" java.lang.NoClassDefFoundError: HelloWorldApp/class**`

初学程序员常犯的一个错误是，尝试对编译器创建的 `.class` 文件运行 `java` 启动器。例如，如果你尝试用 `java HelloWorldApp.class` 而非 `java HelloWorldApp` 来运行程序，就会收到此错误。请记住，参数是你想要使用的*类名*，*而非*文件名。

**`Exception in thread "main" java.lang.NoSuchMethodError: main`**

Java 虚拟机要求你用它执行的类拥有一个 `main` 方法，作为应用程序开始执行的位置。[[深入剖析HelloWorld应用程序|深入剖析 "Hello World!" 应用程序]] 对 `main` 方法进行了详细讨论。

**小程序或 Java Web Start 应用程序被阻止 (Applet or Java Web Start Application Is Blocked)**

如果你正在通过浏览器运行应用程序，并收到表示应用程序被阻止的安全警告，请检查以下各项：

- 验证 JAR 文件清单(manifest)中的属性已针对应用程序运行的环境进行了正确设置。Permissions 属性是必需的。在 NetBeans 项目中，你可以通过展开项目文件夹并双击 manifest.mf，从 NetBeans IDE 的 Files 标签页打开清单文件。
- 验证应用程序已由有效证书签名，并且该证书位于 Signer CA 密钥库(keystore)中。
- 如果你正在运行本地小程序(local applet)，请设置一个用于测试的 Web 服务器。你也可以将你的应用程序添加到例外站点列表(exception site list)中，该列表在 Java 控制面板(Control Panel)的 Security 标签页中管理。
