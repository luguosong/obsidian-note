## 学习路径：反射 API

## 反射的用途

反射通常被那些需要检查或修改运行在 Java 虚拟机中的应用程序的运行时行为的程序所使用。这是一项相对高级的特性，只应由对语言基础有扎实掌握的开发者使用。在此提醒之下，反射仍是一项强大的技术，能使应用程序执行原本不可能的操作。

可扩展性特性

应用程序可以通过使用外部、用户定义类的完全限定名创建扩展性对象的实例来使用这些类。

类浏览器和可视化开发环境

类浏览器需要能够枚举类的成员。可视化开发环境可以利用反射中可用的类型信息来帮助开发者编写正确的代码，从而从中受益。

调试器和测试工具

调试器需要能够检查类的私有成员。测试框架可以利用反射系统地调用类上定义的一组可发现的 API，以确保测试套件中的高代码覆盖率。

## 反射的缺点

反射虽然强大，但不应滥用。如果可以在不使用反射的情况下执行某项操作，则最好避免使用反射。通过反射访问代码时，应牢记以下问题。

性能开销

由于反射涉及动态解析的类型，因此无法执行某些 Java 虚拟机优化。因此，反射操作的性能比其非反射对应操作要慢，在性能敏感的应用程序中频繁调用的代码段应避免使用反射。

安全限制

反射需要运行时权限，而在安全管理器下运行时可能不具备该权限。对于必须在受限安全上下文（如 Applet）中运行的代码，这是一个重要的考量。

暴露内部实现

由于反射允许代码执行在非反射代码中非法的操作（例如访问 `private` 字段和方法），使用反射可能导致意外的副作用，可能使代码失去功能并破坏可移植性。反射代码破坏了抽象，因此可能随平台的升级而改变行为。

## 学习路径课程

本学习路径涵盖使用反射访问和操作类、字段、方法和构造器的常见用法。每节课都包含代码示例、技巧和故障排除信息。

[**类（Classes）**](https://docs.oracle.com/javase/tutorial/reflect/class/index.html)

本课程展示获取 [`Class`](https://docs.oracle.com/javase/8/docs/api/java/lang/Class.html) 对象的各种方式，并用它检查类的属性，包括其声明和内容。

[**成员（Members）**](https://docs.oracle.com/javase/tutorial/reflect/member/index.html)

本课程描述如何使用反射 API 查找类的字段、方法和构造器。提供了设置和获取字段值、调用方法以及使用特定构造器创建对象新实例的示例。

[**数组和枚举类型（Arrays and Enumerated Types）**](https://docs.oracle.com/javase/tutorial/reflect/special/index.html)

本课程介绍两种特殊类型的类：在运行时生成的数组，以及定义唯一命名对象实例的 `enum` 类型。示例代码展示了如何检索数组的组件类型，以及如何设置和获取具有数组或 `enum` 类型的字段。
