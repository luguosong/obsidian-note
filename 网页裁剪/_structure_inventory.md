## 结构问题清单（捋顺结构的核心发现）

### 问题1：课程父节点大面积未引用其叶子页（最严重）

这是"网状混乱"的根本原因。Oracle 原网页靠叶子页自身的 Documentation 侧边栏横向互链，**父节点没有在正文里用 wikilink 列出子页**。导致图谱里只有横向边、没有纵向边。

受影响课程（引用叶子数 = 0 或极少）：
- **Swing 全部课程**：Swing-组件 / Swing-事件监听 / Swing-布局 / Swing-其他特性 / Swing-外观 / Swing-并发 / Swing-拖放 / Swing-自定义绘制 / Swing-入门 —— 全部未列叶子（仅 Swing-事件监听误引1个）。
- **大量其他课程**同样未列。

**处理方案**：在课程父节点正文中补充 `[[叶子页]]` 列表，建立纵向树边。这是把网状改为树状的关键动作（删除叶子横向边后，必须由父节点提供纵向边，否则叶子变孤儿）。

### 问题2：文件名前缀与父节点名不一致（归属靠 URL 兜底）

以下系列的叶子前缀 ≠ 父节点名，靠 URL 路径才正确归属：

| 叶子前缀 | 实际归属课程/路线 | 说明 |
|---|---|---|
| `Swing-basicdemo` 等（无二级前缀） | Swing-拖放 | URL 第4段 `dnd` |
| `日期时间-ISO标准-*` | 日期时间-datetime | URL 第3段 `datetime` |
| `语言基础-*` | 学习Java语言-语言基础 | 课程名带路线前缀 |
| `面向对象概念-*` | 学习Java语言-面向对象概念 | 同上 |
| `泛型进阶-*` | 泛型进阶（lesson） | URL 在 `extra/generics` 下 |
| `包-*` | （无对应课程父节点） | 缺父节点，见问题3 |

### 问题3：缺父节点的系列

- **`包-*` 系列**（8个叶子：createpkgs/managingfiles/namingpkgs/usepkgs/summary 等）：URL 在 `java/package` 下，应属于"学习Java语言"路线，但**没有 `包` 课程父节点**。这些叶子目前无课程归属。

### 问题4：dnd 叶子的特殊命名

`Swing-拖放` 课程的 23 个叶子文件名是 `Swing-intro`/`Swing-basicdemo`/`Swing-export` 等（无 `dnd` 段），而其他 Swing 课程叶子是 `Swing-组件-*`/`Swing-事件监听-*` 格式。命名不统一，但归属明确（URL 的 `dnd` 段）。
