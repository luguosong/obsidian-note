## 预定义注解类型

Java SE API 中预定义了一组注解类型。其中一些由 Java 编译器使用，另一些则作用于其它注解。

## Java 语言使用的注解类型

在 `java.lang` 中定义的预定义注解类型有 `@Deprecated`、`@Override` 和 `@SuppressWarnings`。

**@Deprecated** [`@Deprecated`](https://docs.oracle.com/javase/8/docs/api/java/lang/Deprecated.html) 注解表示所标记的元素已*弃用(deprecated)*，不应再使用。每当程序使用了带有 `@Deprecated` 注解的方法、类或字段时，编译器都会生成一个警告。当某个元素被弃用时，还应使用 Javadoc 的 `@deprecated` 标签进行文档说明，如下例所示。在 Javadoc 注释和注解中都使用 at 符号（`@`）并非巧合：它们在概念上是相关联的。另请注意，Javadoc 标签以小写字母 *d* 开头，而注解以大写字母 *D* 开头。

```text
// Javadoc comment follows
    /**
     * @deprecated
     * explanation of why it was deprecated
     */
    @Deprecated
    static void deprecatedMethod() { }
}
```

**@Override** [`@Override`](https://docs.oracle.com/javase/8/docs/api/java/lang/Override.html) 注解告知编译器该元素旨在覆盖超类中声明的元素。覆盖方法将在 [[接口和继承]] 中讨论。

```text
// mark method as a superclass method
// that has been overridden
@Override 
int overriddenMethod() { }
```

在覆盖方法时并非必须使用此注解，但它有助于防止错误。如果一个被 `@Override` 标记的方法未能正确覆盖其某个超类中的方法，编译器会生成一个错误。

**@SuppressWarnings** [`@SuppressWarnings`](https://docs.oracle.com/javase/8/docs/api/java/lang/SuppressWarnings.html) 注解告诉编译器抑制它本会生成的特定警告。在下面的例子中，使用了一个已弃用的方法，编译器通常会生成警告。但此时该注解会抑制这个警告。

```text
// use a deprecated method and tell 
// compiler not to generate a warning
@SuppressWarnings("deprecation")
 void useDeprecatedMethod() {
     // deprecation warning
     // - suppressed
     objectOne.deprecatedMethod();
 }
```

每条编译器警告都属于一个类别。Java 语言规范列出了两个类别：`deprecation` 和 `unchecked`。当与泛型(generics)出现之前编写的遗留代码交互时，可能会出现 `unchecked` 警告。要抑制多个类别的警告，使用如下语法：

```text
@SuppressWarnings({"unchecked", "deprecation"})
```

**@SafeVarargs** [`@SafeVarargs`](https://docs.oracle.com/javase/8/docs/api/java/lang/SafeVarargs.html) 注解应用于方法或构造方法时，断言该代码不会对其 `varargs` 参数执行潜在不安全的操作。使用此注解类型时，与 `varargs` 用法相关的 unchecked 警告会被抑制。

**@FunctionalInterface** [`@FunctionalInterface`](https://docs.oracle.com/javase/8/docs/api/java/lang/FunctionalInterface.html) 注解（Java SE 8 引入）表示该类型声明旨在成为一个函数式接口(functional interface)，这由 Java 语言规范所定义。

## 作用于其它注解的注解

作用于其它注解的注解称为*元注解(meta-annotation)*。`java.lang.annotation` 中定义了若干元注解类型。

**@Retention** [`@Retention`](https://docs.oracle.com/javase/8/docs/api/java/lang/annotation/Retention.html) 注解指定被标记注解的存储方式：

- `RetentionPolicy.SOURCE` —— 被标记的注解仅保留在源码层级，被编译器忽略。
- `RetentionPolicy.CLASS` —— 被标记的注解在编译期由编译器保留，但被 Java 虚拟机(JVM)忽略。
- `RetentionPolicy.RUNTIME` —— 被标记的注解由 JVM 保留，因此运行时环境可以使用它。

**@Documented** [`@Documented`](https://docs.oracle.com/javase/8/docs/api/java/lang/annotation/Documented.html) 注解表示，每当使用了指定注解时，那些元素都应使用 Javadoc 工具进行文档化。（默认情况下，注解不包含在 Javadoc 中。）更多信息请参见 [Javadoc 工具页面](https://docs.oracle.com/javase/8/docs/technotes/guides/javadoc/index.html)。

**@Target** [`@Target`](https://docs.oracle.com/javase/8/docs/api/java/lang/annotation/Target.html) 注解用于标记另一个注解，以限制该注解可以应用于哪些 Java 元素。一个 target 注解的值指定以下元素类型之一：

- `ElementType.ANNOTATION_TYPE` 可以应用于注解类型。
- `ElementType.CONSTRUCTOR` 可以应用于构造方法。
- `ElementType.FIELD` 可以应用于字段或属性。
- `ElementType.LOCAL_VARIABLE` 可以应用于局部变量。
- `ElementType.METHOD` 可以应用于方法级别的注解。
- `ElementType.PACKAGE` 可以应用于包声明。
- `ElementType.PARAMETER` 可以应用于方法的参数。
- `ElementType.TYPE` 可以应用于类的任何元素。

**@Inherited** [`@Inherited`](https://docs.oracle.com/javase/8/docs/api/java/lang/annotation/Inherited.html) 注解表示该注解类型可以从超类继承。（默认情况下并非如此。）当用户查询该注解类型而当前类没有此类型的注解时，就会查询该类的超类是否具有该注解类型。此注解仅适用于类声明。

**@Repeatable** [`@Repeatable`](https://docs.oracle.com/javase/8/docs/api/java/lang/annotation/Repeatable.html) 注解（Java SE 8 引入）表示被标记的注解可以在同一声明或类型使用上多次应用。更多信息请参见 [[重复注解|重复注解（Repeating Annotations）]]。
