## 局部类

局部类是定义在一个*块(block)*中的类，块是位于配对花括号之间的零条或多条语句的集合。您通常会在方法体内找到局部类的定义。

本节涵盖以下主题：

## 声明局部类

您可以在任何块中定义局部类（更多信息请参见 [表达式、语句和块（Expressions, Statements, and Blocks）](https://docs.oracle.com/javase/tutorial/java/nutsandbolts/expressions.html)）。例如，您可以在方法体、`for` 循环或 `if` 子句中定义局部类。

下面的示例 [`LocalClassExample`](https://docs.oracle.com/javase/tutorial/java/javaOO/examples/LocalClassExample.java) 验证两个电话号码。它在 `validatePhoneNumber` 方法中定义了局部类 `PhoneNumber`：

```java
public class LocalClassExample {
  
    static String regularExpression = "[^0-9]";
  
    public static void validatePhoneNumber(
        String phoneNumber1, String phoneNumber2) {
      
        final int numberLength = 10;
        
        // Valid in JDK 8 and later:
       
        // int numberLength = 10;
       
        class PhoneNumber {
            
            String formattedPhoneNumber = null;

            PhoneNumber(String phoneNumber){
                // numberLength = 7;
                String currentNumber = phoneNumber.replaceAll(
                  regularExpression, "");
                if (currentNumber.length() == numberLength)
                    formattedPhoneNumber = currentNumber;
                else
                    formattedPhoneNumber = null;
            }

            public String getNumber() {
                return formattedPhoneNumber;
            }
            
            // Valid in JDK 8 and later:

//            public void printOriginalNumbers() {
//                System.out.println("Original numbers are " + phoneNumber1 +
//                    " and " + phoneNumber2);
//            }
        }

        PhoneNumber myNumber1 = new PhoneNumber(phoneNumber1);
        PhoneNumber myNumber2 = new PhoneNumber(phoneNumber2);
        
        // Valid in JDK 8 and later:

//        myNumber1.printOriginalNumbers();

        if (myNumber1.getNumber() == null) 
            System.out.println("First number is invalid");
        else
            System.out.println("First number is " + myNumber1.getNumber());
        if (myNumber2.getNumber() == null)
            System.out.println("Second number is invalid");
        else
            System.out.println("Second number is " + myNumber2.getNumber());

    }

    public static void main(String... args) {
        validatePhoneNumber("123-456-7890", "456-7890");
    }
}
```

该示例通过先从电话号码中删除除数字 0 到 9 之外的所有字符来验证电话号码。然后，它检查电话号码是否恰好包含十位数字（北美电话号码的长度）。此示例打印如下：

```text
First number is 1234567890
Second number is invalid
```

## 访问外部类的成员

局部类可以访问其外部类的成员。在前面的示例中，`PhoneNumber` 构造方法访问了成员 `LocalClassExample.regularExpression`。

此外，局部类还可以访问局部变量。但是，局部类只能访问声明为 final 的局部变量。当局部类访问其所在块的局部变量或参数时，它会*捕获(capture)*该变量或参数。例如，`PhoneNumber` 构造方法可以访问局部变量 `numberLength`，因为它被声明为 final；`numberLength` 是一个*被捕获的变量(captured variable)*。

然而，从 Java SE 8 开始，局部类可以访问其所在块中声明为 final 或*事实上为 final(effectively final)*的局部变量和参数。在初始化之后其值从不改变的变量或参数，就是事实上为 final 的。例如，假设变量 `numberLength` 没有声明为 final，并且您在 `PhoneNumber` 构造方法中添加了突出显示的赋值语句，将有效电话号码的长度改为 7 位：

```java
PhoneNumber(String phoneNumber) {
    numberLength = 7;
    String currentNumber = phoneNumber.replaceAll(
        regularExpression, "");
    if (currentNumber.length() == numberLength)
        formattedPhoneNumber = currentNumber;
    else
        formattedPhoneNumber = null;
}
```

由于这条赋值语句，变量 `numberLength` 不再是事实上为 final 的。因此，当内部类 `PhoneNumber` 试图访问 `numberLength` 变量时，Java 编译器会生成一条类似于「从内部类引用的局部变量必须是 final 或事实上为 final 的」的错误信息：

```text
if (currentNumber.length() == numberLength)
```

从 Java SE 8 开始，如果您在方法中声明局部类，它可以访问该方法的参数。例如，您可以在 `PhoneNumber` 局部类中定义以下方法：

```java
public void printOriginalNumbers() {
    System.out.println("Original numbers are " + phoneNumber1 +
        " and " + phoneNumber2);
}
```

方法 `printOriginalNumbers` 访问了 `validatePhoneNumber` 方法的参数 `phoneNumber1` 和 `phoneNumber2`。

### 遮蔽与局部类

局部类中某种类型（例如变量）的声明会遮蔽外部作用域中同名的声明。更多信息请参见[遮蔽（Shadowing）](https://docs.oracle.com/javase/tutorial/java/javaOO/nested.html#shadowing)。

## 局部类与内部类相似

局部类与内部类相似，因为它们不能定义或声明任何静态成员。静态方法中的局部类（例如定义在静态方法 `validatePhoneNumber` 中的类 `PhoneNumber`）只能引用外部类的静态成员。例如，如果您没有将成员变量 `regularExpression` 定义为 static，那么 Java 编译器会生成一条类似于「非静态变量 `regularExpression` 不能从静态上下文中引用」的错误。

局部类是非静态的，因为它们可以访问其所在块的实例成员。因此，它们不能包含大多数类型的静态声明。

您不能在块中声明接口；接口本质上是静态的。例如，下面的代码片段无法编译，因为接口 `HelloThere` 定义在方法 `greetInEnglish` 的体内：

```java
public void greetInEnglish() {
    interface HelloThere {
       public void greet();
    }
    class EnglishHelloThere implements HelloThere {
        public void greet() {
            System.out.println("Hello " + name);
        }
    }
    HelloThere myGreeting = new EnglishHelloThere();
    myGreeting.greet();
}
```

您不能在局部类中声明静态初始化器或成员接口。下面的代码片段无法编译，因为方法 `EnglishGoodbye.sayGoodbye` 被声明为 `static`。编译器在遇到此方法定义时会生成一条类似于「修饰符 'static' 仅允许在常量变量声明中使用」的错误：

```java
public void sayGoodbyeInEnglish() {
    class EnglishGoodbye {
        public static void sayGoodbye() {
            System.out.println("Bye bye");
        }
    }
    EnglishGoodbye.sayGoodbye();
}
```

只要局部类的静态成员是常量变量，局部类就可以拥有静态成员。（*常量变量(constant variable)*是声明为 final 并用编译时常量表达式初始化的基本类型或 `String` 类型变量。编译时常量表达式通常是可以在编译时求值的字符串或算术表达式。更多信息请参见[理解类成员（Understanding Class Members）](https://docs.oracle.com/javase/tutorial/java/javaOO/classvars.html)。）下面的代码片段可以编译，因为静态成员 `EnglishGoodbye.farewell` 是一个常量变量：

```java
public void sayGoodbyeInEnglish() {
    class EnglishGoodbye {
        public static final String farewell = "Bye bye";
        public void sayGoodbye() {
            System.out.println(farewell);
        }
    }
    EnglishGoodbye myEnglishGoodbye = new EnglishGoodbye();
    myEnglishGoodbye.sayGoodbye();
}
```
