## 路线：声音

Java Sound API 是一个低级 API，用于影响和控制声音媒体（包括音频和乐器数字接口(MIDI) 数据）的输入和输出。Java Sound API 在促进可扩展性和灵活性的框架中，对声音输入和输出通常所需的功能提供显式控制。

Java Sound API 满足各种应用程序开发者的需求。潜在应用领域包括：

- 通信框架，如会议和电话
- 最终用户内容交付系统，如媒体播放器和使用流式内容的音乐
- 交互式应用程序，如使用动态内容的游戏和网站
- 内容创建和编辑
- 工具、工具包和实用程序

Java Sound API 提供 Java 平台上最低级别的声音支持。它为应用程序提供对声音操作的大量控制，并且是可扩展的。例如，Java Sound API 提供安装、访问和操作系统资源（如音频混音器、MIDI 合成器、其他音频或 MIDI 设备、文件读取器和写入器以及声音格式转换器）的机制。Java Sound API 不包含复杂的声音编辑器或图形工具，但它提供了可以构建此类程序的功能。它强调超越最终用户通常期望的低级控制。

Java Sound API 包括对数字音频和 MIDI 数据的支持。这两个主要功能模块在不同的包中提供：

- [`javax.sound.sampled`](https://docs.oracle.com/javase/8/docs/api/javax/sound/sampled/package-summary.html) —— 此包指定用于捕获、混合和回放数字（采样）音频的接口。
- [`javax.sound.midi`](https://docs.oracle.com/javase/8/docs/api/javax/sound/midi/package-summary.html) —— 此包提供用于 MIDI 合成、排序和事件传输的接口。

另外两个包允许服务提供者（而非应用程序开发者）创建扩展 Java Sound API 实现功能的自定义软件组件：

- [`javax.sound.sampled.spi`](https://docs.oracle.com/javase/8/docs/api/javax/sound/sampled/spi/package-summary.html)
- [`javax.sound.midi.spi`](https://docs.oracle.com/javase/8/docs/api/javax/sound/midi/spi/package-summary.html)

本页介绍采样音频系统、MIDI 系统和 SPI 包。本教程稍后将更详细地讨论每个包。

---

**注意：**

还有其他 Java 平台 API 也具有与声音相关的元素。[Java 媒体框架 API(JMF)](http://www.oracle.com/technetwork/java/javase/tech/index-jsp-140239.html) 是一个更高级的 API，目前作为 Java 平台的标准扩展提供。JMF 指定了用于捕获和回放基于时间的媒体的统一架构、消息协议和编程接口。JMF 为基本媒体播放器应用程序提供更简单的解决方案，并使不同媒体类型（如音频和视频）之间能够同步。另一方面，专注于声音的程序可以从 Java Sound API 中受益，特别是如果它们需要更高级的功能，例如仔细控制缓冲音频回放或直接操作 MIDI 合成器的能力。其他具有声音方面的 Java API 包括 Java 3D 以及电话和语音 API。这些 API 中任何一个的实现都可能在内部使用 Java Sound API 的实现，但不要求这样做。

## 什么是采样音频？

[`javax.sound.sampled`](https://docs.oracle.com/javase/8/docs/api/javax/sound/sampled/package-summary.html) 包处理数字音频数据，Java Sound API 将其称为采样音频。*样本(Samples)* 是信号的连续快照。对于音频，信号是声波。麦克风将声信号转换为相应的模拟电信号，模数转换器将该模拟信号转换为采样的数字形式。下图显示了录音中的短暂时刻。

![[sound-chapter1.anc.gif]]

采样声波

此图在垂直轴上绘制声压（振幅），在水平轴上绘制时间。以一定速率定期测量模拟声波的振幅，产生构成数字音频信号的离散样本（图中的红色数据点）。中心水平线表示零振幅；线上方的点是正值样本，下方的点是负值。模拟信号的数字近似精度取决于其时间分辨率（*采样率*）和其*量化*或振幅分辨率（用于表示每个样本的位数）。作为参考，为存储在光盘上而录制的音频每秒采样 44,100 次，每个样本用 16 位表示。

这里使用"采样音频"一词略微宽松。声波可以在以模拟形式保留的同时在离散间隔处采样。然而，就 Java Sound API 而言，"采样音频"等同于"数字音频"。

通常，计算机上的采样音频来自录音，但声音也可以是合成生成的（例如，创建按键电话的声音）。"采样音频"一词指的是数据类型，而不是其来源。

Java Sound API 不假设特定的音频硬件配置；它旨在允许在系统上安装不同类型的音频组件并由 API 访问。Java Sound API 支持常见功能，如声卡的输入和输出（例如，用于录制和回放声音文件）以及多个音频流的混合。以下是典型音频架构的一个示例：

## 什么是 MIDI？

[javax.sound.midi](https://docs.oracle.com/javase/8/docs/api/javax/sound/midi/package-summary.html) 包含用于传输和排序 MIDI 事件以及从这些事件合成声音的 API。

采样音频是声音本身的直接表示，而 *MIDI 数据*可以被视为创建声音（尤其是乐音）的配方。与音频数据不同，MIDI 数据不直接描述声音。相反，它描述影响启用 MIDI 的设备或乐器（如合成器）执行的声音（或动作）的事件。MIDI 数据类似于图形用户界面的键盘和鼠标事件。对于 MIDI，事件可以被视为对音乐键盘的操作，以及对该乐器上各种踏板、滑块、开关和旋钮的操作。这些事件实际上不需要源自硬件乐器；它们可以在软件中模拟，也可以存储在 MIDI 文件中。可以创建、编辑和执行这些文件的程序称为音序器。许多计算机声卡包含 MIDI 可控音乐合成器芯片，音序器可以将其 MIDI 事件发送到这些芯片。合成器也可以完全用软件实现。合成器解释它们接收的 MIDI 事件并产生音频输出。通常从 MIDI 数据合成的声音是乐音（而非语音）。MIDI 合成器也能够生成各种音效。

一些声卡包含 MIDI 输入和输出端口，可以连接外部 MIDI 硬件设备（如键盘合成器或其他乐器）。从 MIDI 输入端口，应用程序可以接收由外部配备 MIDI 的乐器生成的事件。程序可以使用计算机的内部合成器播放音乐表演，将其作为 MIDI 文件保存到磁盘，或将其渲染成乐谱。程序可以使用 MIDI 输出端口播放外部乐器，或控制其他外部设备（如录音设备）。

可能的 MIDI 配置

在此示例中，应用程序通过加载存储为磁盘上标准 MIDI 文件（图左侧）的乐谱来准备音乐表演。标准 MIDI 文件包含轨道，每个轨道都是带时间戳的 MIDI 事件列表。大多数事件表示音符（音高和节奏）。此 MIDI 文件被读取然后由软件音序器"演奏"。音序器通过将 MIDI 消息发送到其他设备（如内部或外部合成器）来演奏其音乐。合成器本身可以读取包含模拟某些乐器声音指令的音色库文件。如果没有，合成器将使用已加载到其中的任何乐器声音播放存储在 MIDI 文件中的音符。

如图所示，MIDI 事件必须转换为原始（无时间戳）MIDI，然后才能通过 MIDI 输出端口发送到外部 MIDI 乐器。类似地，从外部 MIDI 源（图中的键盘乐器）进入计算机的原始 MIDI 数据被转换为带时间戳的 MIDI 消息，这些消息可以控制合成器，或音序器可以存储以供以后使用。

## 服务提供者接口

[`javax.sound.sampled.spi`](https://docs.oracle.com/javase/8/docs/api/javax/sound/sampled/spi/package-summary.html) 和 [`javax.sound.midi.spi`](https://docs.oracle.com/javase/8/docs/api/javax/sound/midi/spi/package-summary.html) 包包含允许软件开发者创建新的音频或 MIDI 资源的 API，这些资源可以单独提供给用户并"插入"到 Java Sound API 的现有实现中。以下是可以以这种方式添加的服务（资源）的一些示例：

- 音频混音器
- MIDI 合成器
- 可以读取或写入新型音频或 MIDI 文件的文件解析器
- 在不同声音数据格式之间转换的转换器

在某些情况下，服务是硬件设备功能（如声卡）的软件接口，服务提供者可能与硬件供应商相同。在其他情况下，服务纯粹存在于软件中。例如，合成器或混音器可以是声卡上芯片的接口，也可以在没有任何硬件支持的情况下实现。

Java Sound API 的实现包含一组基本服务，但服务提供者接口(SPI) 包允许第三方创建新服务。这些第三方服务以与内置服务相同的方式集成到系统中。[`AudioSystem`](https://docs.oracle.com/javase/8/docs/api/javax/sound/sampled/AudioSystem.html) 类和 [`MidiSystem`](https://docs.oracle.com/javase/8/docs/api/javax/sound/midi/MidiSystem.html) 类充当协调器，允许应用程序显式或隐式地访问服务。通常，服务对其使用的应用程序是完全透明的。服务提供者机制使基于 Java Sound API 的应用程序用户受益，因为可以将新的声音功能添加到程序中，而无需发布新的 JDK 或运行时环境，并且在许多情况下，甚至不需要发布应用程序本身的新版本。
