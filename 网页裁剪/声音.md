## 学习路径：声音

Java Sound API 是一个底层 API，用于实现和控制声音媒体的输入与输出，包括音频和乐器数字接口（Musical Instrument Digital Interface，MIDI）数据。Java Sound API 在一个促进可扩展性和灵活性的框架中，对声音输入和输出通常所需的各项功能提供了明确控制。

Java Sound API 满足了各类应用开发者的需求。潜在的应用领域包括：

- 通信框架，如会议和电话
- 最终用户内容分发系统，如使用流媒体内容的媒体播放器和音乐播放器
- 交互式应用程序，如使用动态内容的游戏和网站
- 内容创作和编辑
- 工具、工具包和实用程序

Java Sound API 提供了 Java 平台上最低层的声音支持。它为应用程序提供了对声音操作的大量控制，并且是可扩展的。例如，Java Sound API 提供了安装、访问和操作系统资源（如音频混音器、MIDI 合成器、其它音频或 MIDI 设备、文件读写器以及声音格式转换器）的机制。Java Sound API 不包含复杂的声音编辑器或图形工具，但它提供了可在此基础上构建此类程序的能力。它强调超越最终用户通常期望的底层控制。

Java Sound API 包括对数字音频和 MIDI 数据的支持。这两大功能模块在各自的包中提供：

- [`javax.sound.sampled`](https://docs.oracle.com/javase/8/docs/api/javax/sound/sampled/package-summary.html) —— 此包指定了用于数字（采样）音频的捕获、混音和回放的接口。
- [`javax.sound.midi`](https://docs.oracle.com/javase/8/docs/api/javax/sound/midi/package-summary.html) —— 此包提供了用于 MIDI 合成、排序和事件传输的接口。

另外两个包允许服务提供者（而非应用程序开发者）创建自定义软件组件，以扩展 Java Sound API 实现的能力：

- [`javax.sound.sampled.spi`](https://docs.oracle.com/javase/8/docs/api/javax/sound/sampled/spi/package-summary.html)
- [`javax.sound.midi.spi`](https://docs.oracle.com/javase/8/docs/api/javax/sound/midi/spi/package-summary.html)

本页介绍采样音频系统、MIDI 系统以及 SPI 包。随后，本教程将更详细地讨论每个包。

---

**注意：**

还有其它 Java 平台 API 也具有与声音相关的元素。[Java 媒体框架 API（Java Media Framework API，JMF）](http://www.oracle.com/technetwork/java/javase/tech/index-jsp-140239.html) 是一种更高层的 API，目前作为 Java 平台的标准扩展提供。JMF 为捕获和回放基于时间的媒体指定了统一的架构、消息协议和编程接口。JMF 为基本的媒体播放器应用程序提供了更简单的解决方案，并能实现不同媒体类型（如音频和视频）之间的同步。另一方面，专注于声音的程序可以从 Java Sound API 中受益，尤其是当它们需要更高级的功能（如精心控制缓冲音频回放或直接操作 MIDI 合成器）时。其它具有声音方面的 Java API 包括 Java 3D 以及用于电话和语音的 API。这些 API 的实现可能会在内部使用 Java Sound API 的实现，但并不要求这样做。

## 什么是采样音频？

[`javax.sound.sampled`](https://docs.oracle.com/javase/8/docs/api/javax/sound/sampled/package-summary.html) 包处理数字音频数据，Java Sound API 将其称为采样音频（sampled audio）。*采样（samples）* 是信号的连续快照。在音频的情况下，信号就是声波。麦克风将声信号转换为相应的模拟电信号，模数转换器将该模拟信号转换为采样的数字形式。下图展示了声音录制中的短暂时刻。

![[java-tutorial-chapter1.anc.gif]]

采样声波

此图纵轴表示声压（振幅），横轴表示时间。模拟声波的振幅以一定速率定期测量，产生构成数字音频信号的离散采样（图中的红色数据点）。中间的水平线表示零振幅；线上方的点是正值采样，线下方的点是负值采样。模拟信号的数字近似精度取决于其在时间上的分辨率（*采样率（sampling rate）*）及其*量化（quantization）*，即振幅分辨率（用于表示每个采样的位数）。作为参考，为存储在光盘上而录制的音频每秒采样 44,100 次，每个采样用 16 位表示。

此处使用的「采样音频」一词略有宽泛。声波可以在离散间隔上采样而保持模拟形式。然而，就 Java Sound API 而言，「采样音频」等同于「数字音频」。

通常，计算机上的采样音频来自声音录制，但声音也可以合成生成（例如，用于创建按键电话的声音）。「采样音频」一词指的是数据类型，而非其来源。

Java Sound API 不假定特定的音频硬件配置；它被设计为允许在系统上安装不同类型的音频组件并由 API 访问。Java Sound API 支持常见的功能，例如从声卡输入和输出（例如，用于录制和回放声音文件），以及多个音频流的混音。以下是一种典型音频架构的示例：

## 什么是 MIDI？

[javax.sound.midi](https://docs.oracle.com/javase/8/docs/api/javax/sound/midi/package-summary.html) 包包含用于传输和排序 MIDI 事件以及从这些事件合成声音的 API。

采样音频是对声音本身的直接表示，而 *MIDI 数据（MIDI data）* 则可以看作是创造声音（尤其是乐音）的「配方」。MIDI 数据与音频数据不同，它不直接描述声音。相反，它描述影响支持 MIDI 的设备或乐器（如合成器）所执行的声音（或动作）的事件。MIDI 数据类似于图形用户界面的键盘和鼠标事件。在 MIDI 的情况下，可以将事件视为对音乐键盘的操作，以及对该乐器上各种踏板、滑块、开关和旋钮的操作。这些事件实际上不一定源自硬件乐器；它们可以在软件中模拟，也可以存储在 MIDI 文件中。能够创建、编辑和演奏这些文件的程序称为音序器（sequencer）。许多计算机声卡包含可由音序器发送 MIDI 事件的音乐合成器芯片。合成器也可以完全用软件实现。合成器解释其接收的 MIDI 事件并产生音频输出。通常，由 MIDI 数据合成的声音是乐音（例如，与语音相对）。MIDI 合成器还能生成各种音效。

某些声卡包含 MIDI 输入和输出端口，可连接外部 MIDI 硬件设备（如键盘合成器或其它乐器）。从 MIDI 输入端口，应用程序可以接收外部 MIDI 乐器生成的事件。程序可以使用计算机内部合成器演奏音乐表演、将其作为 MIDI 文件保存到磁盘，或将其渲染成乐谱。程序可以使用 MIDI 输出端口演奏外部乐器，或控制其它外部设备（如录音设备）。

![[java-tutorial-chapter1.anc2.gif]]

一种可能的 MIDI 配置

在此示例中，应用程序通过加载存储在磁盘上作为标准 MIDI 文件（图左侧）的乐谱来准备音乐表演。标准 MIDI 文件包含音轨，每条音轨是带时间标记的 MIDI 事件列表。大多数事件表示音符（音高和节奏）。此 MIDI 文件被读取，然后由软件音序器「演奏」。音序器通过将 MIDI 消息发送到其它设备（如内部或外部合成器）来演奏音乐。合成器本身可以读取包含用于模拟某些乐器声音的指令的音色库文件。如果不能，合成器将使用已加载到其中的任何乐器声音来演奏 MIDI 文件中存储的音符。

如图所示，MIDI 事件必须转换为原始（无时间标记的）MIDI，然后才能通过 MIDI 输出端口发送到外部 MIDI 乐器。同样，从外部 MIDI 源（图中为键盘乐器）进入计算机的原始 MIDI 数据被转换为可控制合成器或由音序器存储以备后用的带时间标记的 MIDI 消息。

## 服务提供者接口

[`javax.sound.sampled.spi`](https://docs.oracle.com/javase/8/docs/api/javax/sound/sampled/spi/package-summary.html) 和 [`javax.sound.midi.spi`](https://docs.oracle.com/javase/8/docs/api/javax/sound/midi/spi/package-summary.html) 包包含允许软件开发者创建新的音频或 MIDI 资源的 API，这些资源可以单独提供给用户并「插入」到 Java Sound API 的现有实现中。以下是可以以这种方式添加的服务（资源）示例：

- 音频混音器
- MIDI 合成器
- 可读写新型音频或 MIDI 文件的文件解析器
- 在不同声音数据格式之间进行转换的转换器

在某些情况下，服务是硬件设备能力的软件接口（如声卡），服务提供者可能与硬件供应商相同。在其它情况下，服务纯粹以软件形式存在。例如，合成器或混音器可以是声卡上芯片的接口，也可以完全在没有任何硬件支持的情况下实现。

Java Sound API 的实现包含一组基本服务，但服务提供者接口（SPI）包允许第三方创建新服务。这些第三方服务以与内置服务相同的方式集成到系统中。[`AudioSystem`](https://docs.oracle.com/javase/8/docs/api/javax/sound/sampled/AudioSystem.html) 类和 [`MidiSystem`](https://docs.oracle.com/javase/8/docs/api/javax/sound/midi/MidiSystem.html) 类充当协调器，允许应用程序显式或隐式访问这些服务。通常，服务的存在对使用它的应用程序是完全透明的。服务提供者机制使基于 Java Sound API 的应用程序的用户受益，因为可以在不需要 JDK 或运行时环境的新发行版的情况下向程序添加新的声音功能，且在许多情况下甚至不需要应用程序本身的新发行版。
