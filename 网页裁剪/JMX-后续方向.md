## Java 管理扩展 (JMX)：路线终点

你已到达"Java 管理扩展(JMX)"路线的终点。

如果你对本路线有意见或建议，请使用我们的[反馈页面](https://docs.oracle.com/javase/feedback.html)告诉我们。

本路线的目的是向你简要介绍 JMX 技术的基本元素。更高级的材料可在其他地方找到。

如果你希望进一步学习 JMX 技术，以下资源将很有用：

- [Java SE 平台的 JMX 技术文档](https://docs.oracle.com/javase/8/docs/technotes/guides/jmx/index.html)。JMX 技术文档包含在 [Java SE 技术文档](https://docs.oracle.com/javase/)中。
- [JMX 技术概述](https://docs.oracle.com/javase/8/docs/technotes/guides/jmx/overview/JMXoverviewTOC.html)。Java SE 平台的 JMX 技术文档包含 JMX 技术概述，它提供了比本路线更详细的 JMX 技术描述。
- [JMX 技术教程](https://docs.oracle.com/javase/8/docs/technotes/guides/jmx/tutorial/tutorialTOC.html)。本路线的课程基于 JMX 技术教程中使用的示例。除了本路线中提供的基本元素外，JMX 技术教程还介绍了 JMX 技术的更高级方面。
- 除上述教程中提供的示例外，一旦安装了 Java 开发工具包(JDK) 6，可以在以下目录中找到一个演示 JMX API 实际实现的示例应用程序：
	```bash
	jdk_home/sample/jmx/jmx-scandir
	```
	其中 `jdk_home` 是安装 JDK 软件的目录。`jmx-scandir` 示例是一个高级示例，它在真实场景中展示了 JMX API 的高级概念。
- [Java 平台的监控和管理](https://docs.oracle.com/javase/8/docs/technotes/guides/management/)。为 Java SE 平台的监控和管理提供的技术和 API 严重依赖 JMX 技术。*Java SE 监控和管理指南*包含一章关于[使用 JMX 技术进行监控和管理](https://docs.oracle.com/javase/8/docs/technotes/guides/management/agent.html)。
- [Java 管理扩展(JMX)技术](http://www.oracle.com/technetwork/java/javase/tech/javamanagement-140525.html)。JMX 技术主页是获取有关 JMX 技术所有最新消息的地方。
