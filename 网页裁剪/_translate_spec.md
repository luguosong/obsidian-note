# 网页裁剪笔记中文化转换规范（子代理执行手册）

本规范是所有子代理处理「网页裁剪/」英文叶子笔记时的**唯一标准**。严格遵循，不得自行发挥。

## 一、目标

将英文格式的 Oracle Java 教程叶子页，转换为与已完成的 `JAXP-DOM-info.md`、`JAXP-DOM-readingXML.md` 一致的**中文标准格式**，同时**删除横向导航双链**（使图谱从网状变为父→子树状）。

## 二、标准输出格式（必须严格一致）

每个文件转换后，从上到下依次为：

```
---
分类:
  - "网页裁剪"
标题: "<中文标题>"
描述: "<中文一句话描述>"
来源: "<原样保留，不改>"
发布者: "<原样保留，不改>"
发布时间:
创建时间: "<原样保留，不改>"
---

# <中文标题，与 frontmatter 标题字段完全一致>

> 文档说明

《Java 教程》(The Java Tutorials) 是基于 JDK 8 编写的。本页所描述的示例与实践未采用后续版本中引入的改进，并且可能使用了目前已不可用的技术。
请参阅 [Dev.java](https://dev.java/learn/)，获取充分利用最新版本的更新版教程。
请参阅 [Java 语言变更](https://docs.oracle.com/pls/topic/lookup?ctx=en/java/javase&id=java_language_changes)，了解 Java SE 9 及后续版本中更新的语言特性摘要。
请参阅 [JDK 发行说明](https://www.oracle.com/technetwork/java/javase/jdk-relnotes-index-2162236.html)，获取所有 JDK 版本的新特性、增强功能以及已移除或弃用的选项的相关信息。

<正文（全部译为中文）>
```

**注意**：`> 文档说明` 及其下方 4 行中文 JDK 说明是**固定文本**，一字不差地照抄上面的内容（这 4 行每个文件都完全相同）。

## 三、删除内容（横向导航双链——网状元凶，必须删）

原文件中存在以下结构，**全部删除**，替换为上面的「`> 文档说明` + 4 行固定中文」：

1. **`Documentation` 导航块**：从单独成行的 `Documentation` 开始，到下面所有 `[[xxx|yyy]]` wikilink 列表 + 当前页纯文本标题行。
2. **Previous/Next 导航行**：形如 `[[xxx|« Previous]] • [Trail](https://...) • [[yyy|Next »]]`。
3. **英文 JDK8 说明段**：以 `The Java Tutorials have been written for JDK 8.` 开头的 4 行英文（含 See [Dev.java]...、See [Java Language Changes]...、See [JDK Release Notes]...）。

## 四、保留内容

- **frontmatter 中的 `来源`、`发布者`、`发布时间`、`创建时间`**：原样保留，一个字符不改。
- **`分类: - "网页裁剪"`**：原样保留。
- **正文中的全部内容**：代码块、表格、图片嵌入、列表、标题层级、wikilink、普通链接，全部保留并翻译可读文本。
- **正文中的 wikilink**：保留双链**目标**（`[[目标文件名]]` 不变），但**显示文本译为中文**。例如原文 `[[JAXP-DOM-readingXML|配置工厂]]` 中 `JAXP-DOM-readingXML` 保留、`配置工厂` 是中文。遇到原文 wikilink 显示文本是英文的，译为中文。
- **图片嵌入**：`![[xxx.png]]`、`![](https://...)` 原样保留，不翻译、不改写。

## 五、学术翻译规范（英文→中文）

> 你是一位专业的中文母语译者，擅长技术内容翻译。

1. **只输出译文**，不添加解释、注释、总结、"以下是翻译"之类的话。
2. **保留专业术语**：技术术语译为中文，首次出现用「中文(原文)」格式，如 "injection" 译为「注入(injection)」。无公认中译、产品名、命令名、API 名（如 `JFrame`、`TransferHandler`、`DocumentBuilderFactory`）直接保留英文。
3. **正式学术语调**：对齐原文复杂度，不口语化、不简化、不增删。
4. **公式与符号**：准确保留，不改动。
5. **术语一致性**：同一术语全文译法一致。
6. **Markdown 结构原样保留**：标题层级、列表、表格、引用块、链接 URL 不变；只翻译可读文本。
7. **代码块**：代码本身**一字不改**（语句、关键字、字符串字面量、变量名）。但代码块内的**注释**需翻译为中文（`//`、`/* */`、`#` 等定界符保留，只译注释文字）。代码块的语言标识（如 ```java）保留。
8. **代码块语言标识**：若原文代码围栏是裸 ``` ``` ```（无语言标识），**保持无标识原样**，不要手工猜测添加。
9. **链接显示文本**：译为中文，URL 不改。

## 六、执行流程（每个文件）

1. 读取原文件全文。
2. 识别并记录 frontmatter 的 `来源`、`发布者`、`发布时间`、`创建时间` 原值。
3. 识别英文 `标题`、`描述`，翻译为中文。
4. 删除「Documentation 块 + Previous/Next 行 + 英文 JDK8 说明段」。
5. 组装标准格式：frontmatter（标题/描述用中文译文，其余原值）+ 一级标题（中文）+ 固定中文文档说明块。
6. 翻译英文正文为中文，保留所有结构、代码块、wikilink（显示文本译中文）、图片。
7. **用完整译文覆盖写回原文件**（保持原文件名、原路径）。
