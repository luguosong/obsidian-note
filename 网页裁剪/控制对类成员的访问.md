## 控制对类成员的访问

访问级别修饰符(access level modifier)决定其它类能否使用某个特定字段或调用某个特定方法。访问控制有两个级别：

- 顶层级别—— `public`，或*包级私有(package-private)*（无显式修饰符）。
- 成员级别—— `public`、`private`、`protected`，或*包级私有*（无显式修饰符）。

一个类可以用 `public` 修饰符声明，在这种情况下，该类对所有地方的所有类都可见。如果一个类没有修饰符（默认情况，也称为*包级私有*），它仅在其自身所在的包内可见（包是相关类的命名分组——您将在后面的课程中学到）。

在成员级别，您可以像顶层类那样使用 `public` 修饰符或无修饰符（*包级私有*），含义相同。对于成员，还有两个额外的访问修饰符：`private` 和 `protected`。`private` 修饰符规定该成员只能在其自身所在的类中被访问。`protected` 修饰符规定该成员只能在其自身所在的包内被访问（与*包级私有*相同），此外，还可以被位于其它包中、属于该类的子类访问。

下表显示了每个修饰符所允许的对成员的访问权限。

| 修饰符 | 类 | 包 | 子类 | 全部 |
| --- | --- | --- | --- | --- |
| `public` | 是 | 是 | 是 | 是 |
| `protected` | 是 | 是 | 是 | 否 |
| 无修饰符 | 是 | 是 | 否 | 否 |
| `private` | 是 | 否 | 否 | 否 |

第一个数据列表示类本身是否能够访问由该访问级别所定义的成员。如您所见，一个类始终能够访问其自身的成员。第二列表示与该类处于同一包中的类（不论其父类关系如何）是否能够访问该成员。第三列表示在此包之外声明的该类的子类是否能够访问该成员。第四列表示所有类是否都能访问该成员。

访问级别在两个方面影响您。首先，当您使用来自其它来源的类（例如 Java 平台中的类）时，访问级别决定了您的类可以使用那些类的哪些成员。其次，当您编写一个类时，需要决定该类中每个成员变量和每个方法应当具有什么访问级别。

**关于选择访问级别的提示：**

如果其它程序员使用您的类，您会希望确保因误用而导致的错误不会发生。访问级别可以帮助您做到这一点。

- 对某个特定成员使用有意义的、最严格的访问级别。除非有充分的理由，否则请使用 `private`。
- 避免使用 `public` 字段，常量(constant)除外。（本教程中的许多示例使用了 public 字段。这样做可能有助于简洁地说明某些要点，但不建议用于生产代码。）public 字段往往会使您与某个特定实现绑定在一起，从而限制您更改代码的灵活性。
