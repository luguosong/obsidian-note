## 接口

在软件工程中，有许多情况需要不同的程序员群体同意一个"契约"，详细说明他们的软件如何交互。每个群体都应该能够在不了解另一群体代码如何编写的情况下编写自己的代码。一般来说，*接口(interfaces)* 就是这样的契约。

例如，想象一个未来社会，计算机控制的机器汽车在没有人类操作员的情况下通过城市街道运送乘客。汽车制造商编写软件（当然是 Java）来操作汽车——停止、启动、加速、左转等。另一个工业群体——电子导航仪器制造商——制造接收 GPS（全球定位系统）位置数据和交通条件的无线传输并使用该信息驾驶汽车的计算机系统。

汽车制造商必须发布一个行业标准接口，详细说明可以调用哪些方法使汽车移动（任何制造商的任何汽车）。导航制造商然后可以编写调用接口中描述的方法来指挥汽车的软件。两个工业群体都不需要知道另一群体的软件是*如何*实现的。事实上，每个群体都认为其软件是高度专有的，并保留随时修改它的权利，只要它继续遵守已发布的接口。

## Java 中的接口

在 Java 编程语言中，*接口(interface)* 是一种引用类型，类似于类，只能包含常量、方法签名、默认方法、静态方法和嵌套类型。方法体仅存在于默认方法和静态方法中。接口不能被实例化——它们只能被类*实现*或被其他接口*扩展*。扩展在本课后面讨论。

定义接口类似于创建新类：

```java
public interface OperateCar {

   // 常量声明（如果有）

   // 方法签名

   // 一个具有 RIGHT、LEFT 值的枚举
   int turn(Direction direction,
            double radius,
            double startSpeed,
            double endSpeed);
   int changeLanes(Direction direction,
                   double startSpeed,
                   double endSpeed);
   int signalTurn(Direction direction,
                  boolean signalOn);
   int getRadarFront(double distanceToCar,
                     double speedOfCar);
   int getRadarRear(double distanceToCar,
                    double speedOfCar);
         ......
   // 更多方法签名
}
```

注意方法签名没有大括号，以分号终止。

要使用接口，你编写一个*实现*该接口的类。当可实例化的类实现接口时，它为接口中声明的每个方法提供方法体。例如，

```java
public class OperateBMW760i implements OperateCar {

    // OperateCar 方法签名及实现 --
    // 例如：
    public int signalTurn(Direction direction, boolean signalOn) {
       // 打开 BMW 左转向指示灯的代码
       // 关闭 BMW 左转向指示灯的代码
       // 打开 BMW 右转向指示灯的代码
       // 关闭 BMW 右转向指示灯的代码
    }

    // 其他成员，根据需要 -- 例如，接口客户端
    // 不可见的辅助类
}
```

在上面的机器汽车示例中，是汽车制造商实现接口。Chevrolet 的实现当然与 Toyota 的实现大不相同，但两家制造商都遵守相同的接口。作为接口客户端的导航制造商将构建系统，使用汽车位置的 GPS 数据、数字街道地图和交通数据来驾驶汽车。这样，导航系统将调用接口方法：转弯、变道、刹车、加速等。

## 接口作为 API

机器汽车示例展示了接口被用作行业标准*应用程序编程接口(API)*。API 在商业软件产品中也很常见。通常，一家公司销售一个软件包，其中包含另一家公司希望在其自己的软件产品中使用的复杂方法。一个例子是销售给制作终端用户图形程序的公司的数字图像处理方法包。图像处理公司编写其类来实现接口，并将其公开给客户。图形公司然后使用接口中定义的签名和返回类型调用图像处理方法。虽然图像处理公司的 API 被公开（对其客户），但其 API 的实现被作为严守的秘密——事实上，它可以在以后修改实现，只要它继续实现其客户所依赖的原始接口。
