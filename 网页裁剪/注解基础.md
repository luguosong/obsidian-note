## 注解的格式

在最简单的形式下，注解看起来如下所示：

```text
@Entity
```

at 符号字符（`@`）向编译器表明其后跟的是一个注解。在下面的例子中，注解的名称是 `Override`：

```text
@Override
void mySuperMethod() { ... }
```

注解可以包含*元素（element）*，这些元素可以命名也可以不命名，并为这些元素提供值：

```text
@Author(
   name = "Benjamin Franklin",
   date = "3/27/2003"
)
class MyClass { ... }
```

或者

```text
@SuppressWarnings(value = "unchecked")
void myMethod() { ... }
```

如果只有一个名为 `value` 的元素，则可以省略名称，如下所示：

```text
@SuppressWarnings("unchecked")
void myMethod() { ... }
```

如果注解没有元素，则可以省略括号，如前面的 `@Override` 例子所示。

也可以在同一声明上使用多个注解：

```text
@Author(name = "Jane Doe")
@EBook
class MyClass { ... }
```

如果这些注解属于同一类型，则称为重复注解(repeating annotation)：

```text
@Author(name = "Jane Doe")
@Author(name = "John Smith")
class MyClass { ... }
```

从 Java SE 8 版本开始支持重复注解。更多信息请参见 [[重复注解|重复注解（Repeating Annotations）]]。

注解类型可以是定义在 Java SE API 的 `java.lang` 或 `java.lang.annotation` 包中的某一类型。在前面的例子中，`Override` 和 `SuppressWarnings` 是 [[预定义注解类型|预定义 Java 注解]]。你也可以定义自己的注解类型。上面例子中的 `Author` 和 `Ebook` 注解就是自定义注解类型。

## 注解可以使用的位置

注解可以应用于各种声明：类、字段、方法以及其他程序元素的声明。当用于声明时，按照惯例，每个注解通常单独占一行。

从 Java SE 8 版本开始，注解还可以应用于类型的*使用*。下面是一些例子：

- 类实例创建表达式：
	```text
	new @Interned MyObject();
	```
- 类型转换：
	```text
	myString = (@NonNull String) str;
	```
- `implements` 子句：
	```text
	class UnmodifiableList<T> implements
	    @Readonly List<@Readonly T> { ... }
	```
- 抛出异常声明：
	```text
	void monitorTemperature() throws
	    @Critical TemperatureException { ... }
	```

这种形式的注解称为*类型注解(type annotation)*。更多信息请参见 [[类型注解与可插拔类型系统|类型注解与可插拔类型系统（Type Annotations and Pluggable Type Systems）]]。
