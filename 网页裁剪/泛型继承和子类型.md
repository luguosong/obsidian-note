## 泛型、继承和子类型

正如你所知，只要类型兼容，就可以将一种类型的对象赋值给另一种类型的对象。例如，你可以将 Integer 赋值给 Object，因为 Object 是 Integer 的超类型之一：

```java
Object someObject = new Object();
Integer someInteger = new Integer(10);
someObject = someInteger;   // OK
```

在面向对象术语中，这被称为"是一个（is a）"关系。由于 Integer *是一种* Object，因此赋值是允许的。但 Integer 也是一种 Number，所以下面的代码也是合法的：

```java
public void someMethod(Number n) { /* ... */ }

someMethod(new Integer(10));   // OK
someMethod(new Double(10.1));   // OK
```

泛型也是如此。你可以执行一次泛型类型调用，将 Number 作为其类型实参传入，只要实参与 Number 兼容，后续对 add 的调用都将被允许：

```java
Box<Number> box = new Box<Number>();
box.add(new Integer(10));   // OK
box.add(new Double(10.1));  // OK
```

现在考虑下面的方法：

```java
public void boxTest(Box<Number> n) { /* ... */ }
```

它接受什么类型的参数？通过查看它的签名，可以看到它接受一个类型为 Box<Number> 的参数。但这意味着什么呢？你是否可以像可能期望的那样传入 Box<Integer> 或 Box<Double>？答案是"不可以"，因为 Box<Integer> 和 Box<Double> 不是 Box<Number> 的子类型。

这是使用泛型编程时的常见误解，但这是一个需要学习的重要概念。

![[generics-generics-subtypeRelationship.gif]]

即使 Integer 是 Number 的子类型，Box<Integer> 也不是 Box<Number> 的子类型。

---

**注意：** 给定两个具体类型 A 和 B（例如 Number 和 Integer），无论 A 和 B 是否相关，MyClass<A> 与 MyClass<B> 都没有任何关系。MyClass<A> 和 MyClass<B> 的共同父类是 Object。

有关在类型参数相关时如何在两个泛型类之间创建类子类型关系的信息，请参见[通配符和子类型（Wildcards and Subtyping）](https://docs.oracle.com/javase/tutorial/java/generics/subtyping.html)。

## 泛型类和子类型

你可以通过扩展或实现来子类型化一个泛型类或接口。一个类或接口的类型参数与另一个的类型参数之间的关系由 extends 和 implements 子句决定。

以集合类为例，ArrayList<E> 实现了 List<E>，而 List<E> 扩展了 Collection<E>。因此 ArrayList<String> 是 List<String> 的子类型，而 List<String> 是 Collection<String> 的子类型。只要你不改变类型实参，子类型关系在这些类型之间就得以保留。

![[generics-generics-sampleHierarchy.gif]]

一个示例的集合层次结构

现在想象我们想要定义自己的列表接口 PayloadList，它将一个可选的泛型类型 P 的值与每个元素相关联。它的声明可能如下所示：

```java
interface PayloadList<E,P> extends List<E> {
  void setPayload(int index, P val);
  ...
}
```

PayloadList 的以下参数化是 List<String> 的子类型：

- PayloadList<String,String>
- PayloadList<String,Integer>
- PayloadList<String,Exception>
