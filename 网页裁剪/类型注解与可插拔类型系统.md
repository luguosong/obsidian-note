## 类型注解与可插拔类型系统

在 Java SE 8 版本之前，注解只能应用于声明。从 Java SE 8 版本开始，注解还可以应用于任何*类型使用(type use)*。这意味着，凡是你使用类型的地方都可以使用注解。类型被使用的若干例子包括：类实例创建表达式（`new`）、类型转换、`implements` 子句和 `throws` 子句。这种形式的注解称为*类型注解(type annotation)*，[[注解基础|注解基础（Annotations Basics）]] 中提供了若干示例。

引入类型注解是为了支持以更强的类型检查方式对 Java 程序进行分析。Java SE 8 版本本身并不提供类型检查框架，但它允许你编写（或下载）一个类型检查框架，该框架实现为一个或多个可插拔模块，与 Java 编译器配合使用。

例如，你希望确保程序中某个特定变量永远不会被赋值为 null，从而避免触发 `NullPointerException`。你可以编写一个自定义插件来检查这一点。然后修改你的代码，为该特定变量添加注解，表明它永远不会被赋值为 null。该变量声明可能如下所示：

```java
@NonNull String str;
```

当你在命令行编译该代码并包含 `NonNull` 模块时，如果编译器检测到潜在问题就会打印一条警告，让你修改代码以避免该错误。当你改正代码、消除所有警告之后，程序运行时就不会再发生这种特定错误。

你可以使用多个类型检查模块，每个模块检查不同类型的错误。这样，你就可以在 Java 类型系统之上进行扩展，在希望的时间和位置添加特定的检查。

通过审慎地使用类型注解，配合可插拔类型检查器，你可以编写出更健壮、更不易出错的代码。

在许多情况下，你不必自行编写类型检查模块。已有第三方为你完成了这些工作。例如，你可能希望使用由华盛顿大学创建的 Checker Framework。该框架包含一个 `NonNull` 模块，以及一个正则表达式模块和一个互斥锁模块。更多信息请参见 [Checker Framework](http://types.cs.washington.edu/checker-framework/)。
