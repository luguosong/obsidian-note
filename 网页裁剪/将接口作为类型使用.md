## 将接口作为类型使用

当你定义一个新接口时，你就在定义一个新的引用数据类型。你可以在任何能够使用其他数据类型名称的地方使用接口名。如果你定义了一个类型为接口的引用变量，那么赋值给它的任何对象都*必须*是实现该接口的某个类的实例。

例如，下面是一个在一对对象中找出最大对象的方法，适用于从实现了 `Relatable` 的类实例化的*任何*对象：

```java
public Object findLargest(Object object1, Object object2) {
   Relatable obj1 = (Relatable)object1;
   Relatable obj2 = (Relatable)object2;
   if ((obj1).isLargerThan(obj2) > 0)
      return object1;
   else 
      return object2;
}
```

通过将 `object1` 转换为 `Relatable` 类型，它就可以调用 `isLargerThan` 方法。

如果你在各种各样的类中都实现了 `Relatable`，那么从这些类中*任何一个*实例化的对象都可以用 `findLargest()` 方法进行比较——前提是两个对象属于同一个类。类似地，它们都可以用下面的方法进行比较：

```java
public Object findSmallest(Object object1, Object object2) {
   Relatable obj1 = (Relatable)object1;
   Relatable obj2 = (Relatable)object2;
   if ((obj1).isLargerThan(obj2) < 0)
      return object1;
   else 
      return object2;
}

public boolean isEqual(Object object1, Object object2) {
   Relatable obj1 = (Relatable)object1;
   Relatable obj2 = (Relatable)object2;
   if ( (obj1).isLargerThan(obj2) == 0)
      return true;
   else 
      return false;
}
```

这些方法适用于任何"可比较的（relatable）"对象，无论它们的类继承关系如何。当它们实现了 `Relatable` 时，它们既可以是其自身类（或超类）的类型，也可以是 `Relatable` 类型。这赋予了它们一些多重继承的优势——它们可以同时拥有来自超类和接口的行为。
