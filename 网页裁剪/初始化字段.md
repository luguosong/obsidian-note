## 初始化字段

如前所述，您通常可以在字段的声明中为其提供一个初始值：

```java
public class BedAndBreakfast {

    // initialize to 10
    public static int capacity = 10;

    // initialize to false
    private boolean full = false;
}
```

当初始值可用、且初始化可以放在一行内完成时，这种方式很有效。然而，由于其简单性，这种初始化形式有其局限性。如果初始化需要某些逻辑（例如错误处理，或用 `for` 循环填充一个复杂数组），简单的赋值就不够了。实例变量可以在构造方法中初始化，在构造方法中可以使用错误处理或其它逻辑。为了给类变量提供同样的能力，Java 编程语言包含了*静态初始化块(static initialization block)*。

---

**注意：** 不一定要在类定义的开头声明字段，尽管这是最常见的做法。只要在使用之前声明并初始化它们即可。

---

## 静态初始化块

*静态初始化块*是一段普通的代码块，由花括号 `{ }` 包围，并以 `static` 关键字作为前缀。下面是一个示例：

```text
static {
    // whatever code is needed for initialization goes here
}
```

一个类可以有任意数量的静态初始化块，它们可以出现在类体中的任何位置。运行时系统保证静态初始化块按照它们在源代码中出现的顺序被调用。

除了静态块，还有另一种替代方式——您可以编写一个私有的静态方法：

```text
class Whatever {
    public static varType myVar = initializeClassVariable();
        
    private static varType initializeClassVariable() {

        // initialization code goes here
    }
}
```

私有静态方法的优点在于，如果您以后需要重新初始化类变量，可以复用它。

## 初始化实例成员

通常，您会将初始化实例变量的代码放在构造方法中。除了使用构造方法初始化实例变量之外，还有两种替代方式：初始化块和 final 方法。

实例变量的初始化块(initializer block)看起来与静态初始化块完全一样，只是没有 `static` 关键字：

```text
{
    // whatever code is needed for initialization goes here
}
```

Java 编译器会将初始化块拷贝到每个构造方法中。因此，这种方式可用于在多个构造方法之间共享一段代码。

*final 方法(final method)*不能在子类中被覆盖。这将在关于接口和继承的课程中讨论。下面是一个使用 final 方法来初始化实例变量的示例：

```text
class Whatever {
    private varType myVar = initializeInstanceVariable();
        
    protected final varType initializeInstanceVariable() {

        // initialization code goes here
    }
}
```

如果子类可能想复用该初始化方法，这种方式尤其有用。该方法被声明为 final，因为在实例初始化期间调用非 final 方法可能会导致问题。
