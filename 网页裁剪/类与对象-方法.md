## 定义方法

以下是典型方法声明的示例：

```java
public double calculateAnswer(double wingSpan, int numberOfEngines,
                              double length, double grossTons) {
    // 在此进行计算
}
```

方法声明唯一必需的元素是方法的返回类型、名称、一对圆括号 `()` 和大括号之间的方法体 `{}`。

更一般地，方法声明按顺序有六个组件：

1. 修饰符——如 `public`、`private` 以及你稍后将学到的其他修饰符。
2. 返回类型——方法返回值的数据类型，如果方法不返回值则为 `void`。
3. 方法名——字段名的规则同样适用于方法名，但约定略有不同。
4. 圆括号中的参数列表——以逗号分隔的输入参数列表，前面带有它们的数据类型，用圆括号 `()` 括起。如果没有参数，你必须使用空圆括号。
5. 异常列表——稍后讨论。
6. 大括号之间的方法体——方法的代码，包括局部变量的声明，放在这里。

修饰符、返回类型和参数将在本课后讨论。异常在后面的课程中讨论。

---

**定义：** 方法声明的两个组件构成*方法签名(method signature)*——方法名和参数类型。

---

上面声明的方法的签名是：

```
calculateAnswer(double, int, double, double)
```

## 方法命名

虽然方法名可以是任何合法的标识符，但代码约定限制了方法名。按照约定，方法名应该是小写的动词，或以小写动词开头的多词名称，后跟形容词、名词等。在多词名称中，第二个及后续单词的每个首字母应大写。以下是一些示例：

```
run
runFast
getBackground
getFinalData
compareTo
setX
isEmpty
```

通常，方法在其类中具有唯一的名称。但是，由于*方法重载(method overloading)*，方法可能与其他方法同名。

## 重载方法

Java 编程语言支持*重载(overloading)*方法，Java 可以区分具有不同*方法签名*的方法。这意味着如果一个类中的方法具有不同的参数列表，它们可以有相同的名称（有一些限定条件将在题为"接口与继承"的课程中讨论）。

假设你有一个类，可以使用书法绘制各种类型的数据（字符串、整数等），并包含一个绘制每种数据类型的方法。为每个方法使用新名称很麻烦——例如，`drawString`、`drawInteger`、`drawFloat` 等。在 Java 编程语言中，你可以对所有绘制方法使用相同的名称，但向每个方法传递不同的参数列表。因此，数据绘制类可以声明四个名为 `draw` 的方法，每个方法都有不同的参数列表。

```java
public class DataArtist {
    ...
    public void draw(String s) {
        ...
    }
    public void draw(int i) {
        ...
    }
    public void draw(double f) {
        ...
    }
    public void draw(int i, double f) {
        ...
    }
}
```

重载方法通过传递给方法的参数的数量和类型来区分。在代码示例中，`draw(String s)` 和 `draw(int i)` 是不同且唯一的方法，因为它们需要不同的参数类型。

你不能声明多个具有相同名称以及相同数量和类型参数的方法，因为编译器无法区分它们。

编译器在区分方法时不考虑返回类型，因此即使它们有不同的返回类型，你也不能声明两个具有相同签名的方法。

---

**注意：** 应谨慎使用重载方法，因为它们会使代码的可读性大大降低。
