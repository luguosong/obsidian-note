## 为什么使用泛型？

简而言之，泛型使得*类型（type）*（类和接口）在定义类、接口和方法时可以成为参数。这与方法声明中更熟悉的*形式参数（formal parameter）*非常相似，类型参数提供了一种让你可以用不同的输入复用相同代码的方式。不同之处在于，形式参数的输入是值，而类型参数的输入是类型。

使用泛型的代码相比非泛型代码有许多好处：

- 更强的编译时类型检查。
	Java 编译器对泛型代码应用强类型检查，如果代码违反了类型安全性，就会发出错误。修复编译时错误比修复运行时错误更容易，因为后者可能难以发现。
- 消除强制类型转换。
	下面这段没有使用泛型的代码片段需要进行强制类型转换：
	```java
	List list = new ArrayList();
	list.add("hello");
	String s = (String) list.get(0);
	```
	当改写为使用泛型时，代码就不再需要强制类型转换：
	```java
	List<String> list = new ArrayList<String>();
	list.add("hello");
	String s = list.get(0);   // 无需强制转换
	```
- 使程序员能够实现通用算法。
	通过使用泛型，程序员可以实现适用于不同类型集合的通用算法，这些算法可被定制，且类型安全、更易阅读。
