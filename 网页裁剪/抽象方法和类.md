## 抽象方法和类

*抽象类（abstract class）*是被声明为 `abstract` 的类——它可能包含、也可能不包含抽象方法。抽象类不能被实例化，但可以被继承。

*抽象方法（abstract method）*是被声明但未提供实现（没有大括号，后面跟一个分号）的方法，如下所示：

```java
abstract void moveTo(double deltaX, double deltaY);
```

如果一个类包含抽象方法，那么该类本身*必须*被声明为 `abstract`，如下所示：

```java
public abstract class GraphicObject {
   // 声明字段
   // 声明非抽象方法
   abstract void draw();
}
```

当一个抽象类被继承时，子类通常会为其父类的所有抽象方法提供实现。然而，如果它没有做到这一点，那么子类也必须被声明为 `abstract`。

---

**注意：** *接口*（参见[接口（Interfaces）](https://docs.oracle.com/javase/tutorial/java/IandI/createinterface.html)一节）中未被声明为默认方法或静态方法的方法都是*隐式*抽象的，因此 `abstract` 修饰符不会用于接口方法。（可以使用，但没必要。）

## 抽象类与接口的比较

抽象类与接口类似。它们都不能被实例化，并且都可以包含混合了有实现和无实现的方法。然而，对于抽象类，你可以声明非 static、非 final 的字段，并定义 public、protected 和 private 的具体方法。对于接口，所有字段都自动是 public、static 和 final 的，而你声明或定义（作为默认方法）的所有方法都是 public 的。此外，你只能扩展一个类（无论它是否为抽象类），而你可以实现任意数量的接口。

你应当使用抽象类还是接口？

- 如果以下任何陈述适用于你的情况，考虑使用抽象类：
	- 你希望在几个密切相关的类之间共享代码。
		- 你预期扩展你的抽象类的类有许多公共的方法或字段，或者需要除 public 之外的访问修饰符（如 protected 和 private）。
		- 你希望声明非 static 或非 final 的字段。这使你能够定义可以访问和修改它们所属对象状态的方法。
- 如果以下任何陈述适用于你的情况，考虑使用接口：
	- 你预期不相关的类会实现你的接口。例如，接口 [`Comparable`](https://docs.oracle.com/javase/8/docs/api/java/lang/Comparable.html) 和 [`Cloneable`](https://docs.oracle.com/javase/8/docs/api/java/lang/Cloneable.html) 被许多不相关的类实现。
		- 你希望指定某种特定数据类型的行为，但不关心由谁来实现其行为。
		- 你希望利用类型的多重继承。

JDK 中抽象类的一个例子是 [`AbstractMap`](https://docs.oracle.com/javase/8/docs/api/java/util/AbstractMap.html)，它是集合框架的一部分。它的子类（包括 `HashMap`、`TreeMap` 和 `ConcurrentHashMap`）共享许多由 `AbstractMap` 定义的方法（包括 `get`、`put`、`isEmpty`、`containsKey` 和 `containsValue`）。

JDK 中一个实现了多个接口的类的例子是 [`HashMap`](https://docs.oracle.com/javase/8/docs/api/java/util/HashMap.html)，它实现了 `Serializable`、`Cloneable` 和 `Map<K, V>` 接口。通过阅读这一接口列表，你可以推断出一个 `HashMap` 实例（无论实现该类的开发者或公司是谁）可以被克隆、是可序列化的（意味着它可以被转换为字节流；参见[可序列化对象（Serializable Objects）](https://docs.oracle.com/javase/tutorial/jndi/objects/serial.html)一节），并且具有映射的功能。此外，`Map<K, V>` 接口通过许多默认方法（如 `merge` 和 `forEach`）得到了增强，而实现了此接口的旧类无需定义这些方法。

注意，许多软件库同时使用抽象类和接口；`HashMap` 类既实现了多个接口，也扩展了抽象类 `AbstractMap`。

## 抽象类示例

在一个面向对象的绘图应用程序中，你可以绘制圆形、矩形、直线、贝塞尔曲线以及许多其他图形对象。这些对象都有某些共同的状态（例如：位置、方向、线条颜色、填充颜色）和行为（例如：moveTo、rotate、resize、draw）。其中一些状态和行为对所有图形对象都是相同的（例如：位置、填充颜色和 moveTo）。另一些则需要不同的实现（例如：resize 或 draw）。所有的 `GraphicObject` 都必须能够绘制或调整自身大小；它们的区别仅仅在于如何做。这正是抽象超类的完美用武之地。你可以利用这些相似性，将所有图形对象声明为继承自同一个抽象父对象（例如 `GraphicObject`），如下图所示。

![[abstract-classes-graphicObject.gif]]

类 Rectangle、Line、Bezier 和 Circle 继承自 GraphicObject

首先，你声明一个抽象类 `GraphicObject`，以提供所有子类完全共享的成员变量和方法，例如当前位置和 `moveTo` 方法。`GraphicObject` 还为那些需要由所有子类实现、但必须以不同方式实现的方法（如 `draw` 或 `resize`）声明抽象方法。`GraphicObject` 类看起来可能像这样：

```java
abstract class GraphicObject {
    int x, y;
    ...
    void moveTo(int newX, int newY) {
        ...
    }
    abstract void draw();
    abstract void resize();
}
```

`GraphicObject` 的每个非抽象子类（例如 `Circle` 和 `Rectangle`）都必须为 `draw` 和 `resize` 方法提供实现：

```java
class Circle extends GraphicObject {
    void draw() {
        ...
    }
    void resize() {
        ...
    }
}
class Rectangle extends GraphicObject {
    void draw() {
        ...
    }
    void resize() {
        ...
    }
}
```

## 当抽象类实现接口时

在[接口（Interfaces）](https://docs.oracle.com/javase/tutorial/java/IandI/createinterface.html)一节中曾指出，一个实现了接口的类必须实现该接口的*所有*方法。然而，也可以定义一个并不实现接口所有方法的类，前提是该类被声明为 `abstract`。例如，

```java
abstract class X implements Y {
  // 实现了 Y 中除一个方法外的所有方法
}

class XX extends X {
  // 实现 Y 中剩余的那个方法
}
```

在这种情况下，类 `X` 必须是 `abstract` 的，因为它没有完全实现 `Y`，而类 `XX` 实际上确实实现了 `Y`。

## 类成员

抽象类可以有 `static` 字段和 `static` 方法。你可以像对待任何其他类一样，通过类引用（例如 `AbstractClass.staticMethod()`）来使用这些静态成员。
