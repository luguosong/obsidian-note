## 向方法或构造方法传递信息

方法或构造方法的声明定义了该方法或构造方法的参数的数量和类型。例如，下面是一个根据贷款金额、利率、贷款期限（期数）和贷款未来价值来计算房屋贷款月供的方法：

```java
public double computePayment(
                  double loanAmt,
                  double rate,
                  double futureValue,
                  int numPeriods) {
    double interest = rate / 100.0;
    double partial1 = Math.pow((1 + interest), 
                    - numPeriods);
    double denominator = (1 - partial1) / interest;
    double answer = (-loanAmt / denominator)
                    - ((futureValue * partial1) / denominator);
    return answer;
}
```

这个方法有四个参数：贷款金额、利率、未来价值和期数。前三个是双精度浮点数，第四个是整数。这些参数在方法体中使用，在运行时将取得传入参数的值。

---

**注意：** *参数(parameter)*是指方法声明中的变量列表。*实参(argument)*则是调用方法时实际传入的值。当您调用一个方法时，所使用的实参必须在类型和顺序上与声明中的参数匹配。

## 参数类型

您可以为方法或构造方法的参数使用任何数据类型。这包括基本数据类型，如 `computePayment` 方法中所见的 double、float 和整数，以及引用数据类型，如对象和数组。

```text
public Polygon polygonFrom(Point[] corners) {
    // method body goes here
}
```

## 任意数量的参数

您可以使用一种称为 *varargs* 的构造，向方法传递任意数量的值。当您不知道会有多少个特定类型的参数被传递给方法时，就可以使用 varargs。它是一种手动创建数组的快捷方式（前面那个方法本可以使用 varargs 而不是数组）。

要使用 varargs，请在最后一个参数的类型之后跟一个省略号（三个点 `...`），然后一个空格，再写参数名。这样，该方法就可以使用任意数量的该参数来调用，包括不传。

```java
public Polygon polygonFrom(Point... corners) {
    int numberOfSides = corners.length;
    double squareOfSide1, lengthOfSide1;
    squareOfSide1 = (corners[1].x - corners[0].x)
                     * (corners[1].x - corners[0].x) 
                     + (corners[1].y - corners[0].y)
                     * (corners[1].y - corners[0].y);
    lengthOfSide1 = Math.sqrt(squareOfSide1);

    // more method body code follows that creates and returns a 
    // polygon connecting the Points
}
```

可以看到，在方法内部，`corners` 被当作数组来处理。该方法既可以用一个数组来调用，也可以用一串参数来调用。无论哪种情况，方法体中的代码都会把这个参数当作数组处理。

您最常见到 varargs 的地方是各种打印方法；例如，这个 `printf` 方法：

```text
public PrintStream printf(String format, Object... args)
```

允许您打印任意数量的对象。可以这样调用它：

```java
System.out.printf("%s: %d, %s%n", name, idnum, address);
```

也可以这样调用：

```java
System.out.printf("%s: %d, %s, %s, %s%n", name, idnum, address, phone, email);
```

或者使用其它不同的参数数量。

## 参数名

当您为方法或构造方法声明一个参数时，需要为该参数提供一个名称。这个名称在方法体中用于引用传入的实参。

参数的名称在其作用域内必须唯一。它不能与同一方法或构造方法的另一个参数同名，也不能与方法或构造方法内某个局部变量同名。

参数可以与类的某个字段同名。在这种情况下，称该参数*遮蔽(shadow)*了该字段。遮蔽字段会使您的代码难以阅读，通常只在构造方法和用于设置特定字段的方法中使用。例如，考虑下面的 `Circle` 类及其 `setOrigin` 方法：

```java
public class Circle {
    private int x, y, radius;
    public void setOrigin(int x, int y) {
        ...
    }
}
```

`Circle` 类有三个字段：`x`、`y` 和 `radius`。`setOrigin` 方法有两个参数，每个参数都与其中一个字段同名。每个方法参数都遮蔽了与其同名的字段。因此，在方法体中使用简单名称 `x` 或 `y` 引用的是参数，*而不是*字段。要访问字段，必须使用限定名。这将在本课稍后题为「使用 `this` 关键字」一节中讨论。

## 传递基本数据类型实参

基本类型实参（如 `int` 或 `double`）是*按值(by value)*传入方法的。这意味着对参数值的任何更改都只存在于方法的作用域之内。当方法返回时，参数就不复存在了，对它们所做的任何更改都会丢失。下面是一个示例：

```java
public class PassPrimitiveByValue {

    public static void main(String[] args) {
           
        int x = 3;
           
        // invoke passMethod() with 
        // x as argument
        passMethod(x);
           
        // print x to see if its 
        // value has changed
        System.out.println("After invoking passMethod, x = " + x);
           
    }
        
    // change parameter in passMethod()
    public static void passMethod(int p) {
        p = 10;
    }
}
```

当您运行这个程序时，输出为：

```text
After invoking passMethod, x = 3
```

## 传递引用数据类型实参

引用数据类型参数（如对象）同样是*按值*传入方法的。这意味着当方法返回时，传入的引用仍然引用着与之前相同的对象。*然而*，如果对象的字段具有适当的访问级别，这些字段的值*可以*在方法中被修改。

例如，考虑某个任意类中的一个移动 `Circle` 对象的方法：

```java
public void moveCircle(Circle circle, int deltaX, int deltaY) {
    // code to move origin of circle to x+deltaX, y+deltaY
    circle.setX(circle.getX() + deltaX);
    circle.setY(circle.getY() + deltaY);
        
    // code to assign a new reference to circle
    circle = new Circle(0, 0);
}
```

假设用这些参数调用该方法：

```text
moveCircle(myCircle, 23, 56)
```

在方法内部，`circle` 最初引用 `myCircle`。该方法将 `circle` 所引用对象（即 `myCircle`）的 x 和 y 坐标分别改变 23 和 56。这些更改在方法返回后会保留下来。然后，`circle` 被赋值为一个 `x = y = 0` 的新 `Circle` 对象的引用。然而，这种重新赋值不会持久生效，因为引用是按值传入的，无法被改变。在方法内部，`circle` 所指向的对象发生了改变，但当方法返回时，`myCircle` 仍然引用着与方法被调用之前相同的 `Circle` 对象。
