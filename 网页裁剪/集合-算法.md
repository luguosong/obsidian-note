## 排序

`sort` 算法对 `List` 重新排序，使其元素根据排序关系按升序排列。提供了两种形式的操作。简单形式接受一个 `List` 并根据其元素的*自然顺序*对其进行排序。如果你不熟悉自然顺序的概念，请阅读[[集合-接口-对象排序|对象排序]]一节。

`sort` 操作使用略微优化的*归并排序(merge sort)*算法，它快速且稳定：

- **快速**：保证在 `n log(n)` 时间内运行，并且在几乎已排序的列表上运行得明显更快。经验测试表明它与高度优化的快速排序一样快。快速排序通常被认为比归并排序快，但它不稳定且不保证 `n log(n)` 性能。
- **稳定**：它不会重新排序相等的元素。如果你在同一列表上按不同属性重复排序，这很重要。如果邮件程序的用户先按邮寄日期对收件箱排序，然后按发件人排序，用户自然期望现在连续的来自给定发件人的邮件列表（仍然）按邮寄日期排序。只有在第二次排序稳定时才能保证这一点。

以下[`简单程序`](https://docs.oracle.com/javase/tutorial/collections/algorithms/examples/Sort.java)按字典序（字母）顺序打印其参数。

```java
import java.util.*;

public class Sort {
    public static void main(String[] args) {
        List<String> list = Arrays.asList(args);
        Collections.sort(list);
        System.out.println(list);
    }
}
```

让我们运行该程序。

```
% java Sort i walk the line
```

产生以下输出。

```
[i, line, the, walk]
```

包含此程序只是为了向你展示算法确实如其看起来那样容易使用。

`sort` 的第二种形式除了 `List` 外还接受一个 [`Comparator`](https://docs.oracle.com/javase/8/docs/api/java/util/Comparator.html)，并使用 `Comparator` 对元素排序。假设你想按大小的逆序打印前面示例中的变位词组——最大的变位词组在前。以下示例展示如何借助 `sort` 方法的第二种形式实现这一点。

回想一下，变位词组以 `List` 实例的形式作为值存储在 `Map` 中。修订后的打印代码遍历 `Map` 的值视图，将每个通过最小大小测试的 `List` 放入一个 `List` 的 `List` 中。然后代码使用期望 `List` 实例的 `Comparator` 对此 `List` 排序，并实现逆序大小排序。最后，代码遍历已排序的 `List`，打印其元素（变位词组）。以下代码替换 `Anagrams` 示例中 `main` 方法末尾的打印代码。

```java
// 将所有超过大小阈值的变位词组放入 List
List<List<String>> winners = new ArrayList<List<String>>();
for (List<String> l : m.values())
    if (l.size() >= minGroupSize)
        winners.add(l);

// 根据大小对变位词组排序
Collections.sort(winners, new Comparator<List<String>>() {
    public int compare(List<String> o1, List<String> o2) {
        return o2.size() - o1.size();
    }});

// 打印变位词组。
for (List<String> l : winners)
    System.out.println(l.size() + ": " + l);
```

## 混洗

`shuffle` 算法执行与 `sort` 相反的操作，破坏 `List` 中可能存在的任何顺序痕迹。即，此算法根据来自随机源的输入对 `List` 重新排序，使得在公平随机源的假设下，所有可能的排列以相等的可能性出现。此算法在实现机会游戏中很有用。例如，它可以用于混洗表示一副牌的 `Card` 对象的 `List`。此外，它对生成测试用例很有用。

此操作有两种形式：一种接受 `List` 并使用默认随机源，另一种要求调用者提供 [Random](https://docs.oracle.com/javase/8/docs/api/java/util/Random.html) 对象作为随机源。此算法的代码在 [[集合-接口-List接口|List 一节]]中作为示例使用。

## 常规数据操作

`Collections` 类提供五种算法用于对 `List` 对象执行常规数据操作，它们都很简单：

- `reverse` —— 反转 `List` 中元素的顺序。
- `fill` —— 用指定值覆盖 `List` 中的每个元素。此操作对于重新初始化 `List` 很有用。
- `copy` —— 接受两个参数，一个目标 `List` 和一个源 `List`，并将源的元素复制到目标中，覆盖其内容。目标 `List` 必须至少与源一样长。如果更长，目标 `List` 中剩余的元素不受影响。
- `swap` —— 交换 `List` 中指定位置的元素。
- `addAll` —— 将所有指定元素添加到 `Collection`。要添加的元素可以单独指定或作为数组指定。

## 搜索

`binarySearch` 算法在已排序的 `List` 中搜索指定元素。此算法有两种形式。第一种接受 `List` 和要搜索的元素（"搜索键"）。此形式假设 `List` 根据元素的自然顺序按升序排序。第二种形式除了 `List` 和搜索键外还接受 `Comparator`，并假设 `List` 根据指定的 `Comparator` 按升序排序。可以在调用 `binarySearch` 之前使用 `sort` 算法对 `List` 排序。

两种形式的返回值相同。如果 `List` 包含搜索键，则返回其索引。如果不包含，则返回值为 `(-(插入点) - 1)`，其中插入点是将值插入 `List` 的位置，或大于该值的第一个元素的索引，如果 `List` 中的所有元素都小于指定值则为 `list.size()`。这个公认的丑陋公式保证当且仅当找到搜索键时返回值 `>= 0`。它基本上是一种将布尔值 `(found)` 和整数 `(index)` 组合成单个 `int` 返回值的技巧。

以下惯用法可用于 `binarySearch` 操作的两种形式，它查找指定的搜索键，如果尚不存在则在适当位置插入它。

```java
int pos = Collections.binarySearch(list, key);
if (pos < 0)
   l.add(-pos-1, key);
```

## 组合

frequency 和 disjoint 算法测试一个或多个 `Collection` 组合的某些方面：

- `frequency` —— 计算指定元素在指定集合中出现的次数
- `disjoint` —— 确定两个 `Collection` 是否不相交；即，它们是否不包含任何共同元素

## 查找极值

`min` 和 `max` 算法分别返回指定 `Collection` 中包含的最小和最大元素。这两个操作都有两种形式。简单形式只接受 `Collection` 并根据元素的自然顺序返回最小（或最大）元素。第二种形式除了 `Collection` 外还接受 `Comparator` 并根据指定的 `Comparator` 返回最小（或最大）元素。
