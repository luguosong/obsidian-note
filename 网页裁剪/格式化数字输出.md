## 格式化数字输出

前面你已经看到过使用 `print` 和 `println` 方法将字符串打印到标准输出（`System.out`）。由于所有数字都可以转换为字符串（你将在本课程后面看到），你可以使用这些方法打印出任意混合的字符串和数字。然而，Java 编程语言还有其他一些方法，当输出中包含数字时，它们允许你对打印输出进行更精细的控制。

## printf 和 format 方法

`java.io` 包中包含一个 `PrintStream` 类，它有两个格式化方法，可以用来替代 `print` 和 `println`。这两个方法，`format` 和 `printf`，彼此是等价的。你所熟悉的、一直在使用的 `System.out` 恰好就是一个 `PrintStream` 对象，因此你可以在 `System.out` 上调用 `PrintStream` 的方法。于是，你可以在代码中任何以前使用 `print` 或 `println` 的地方使用 `format` 或 `printf`。例如，

```java
System.out.format(.....);
```

这两个 [`java.io.PrintStream`](https://docs.oracle.com/javase/8/docs/api/java/io/PrintStream.html) 方法的语法是相同的：

```java
public PrintStream format(String format, Object... args)
```

其中 `format` 是一个指定要使用的格式设置的字符串，`args` 是要使用该格式打印的变量列表。一个简单的示例是

```java
System.out.format("The value of " + "the float variable is " +
     "%f, while the value of the " + "integer variable is %d, " +
     "and the string is %s", floatVar, intVar, stringVar);
```

第一个参数 `format` 是一个格式字符串，指定如何格式化第二个参数 `args` 中的对象。格式字符串包含纯文本以及*格式说明符（format specifier）*，后者是用于格式化 `Object... args` 参数的特殊字符。（`Object... args` 这种表示法被称为*可变参数（varargs）*，意味着参数的数量可以变化。）

格式说明符以百分号（%）开头，以一个*转换符（converter）*结尾。转换符是一个字符，指示要格式化的参数类型。在百分号（%）和转换符之间，你可以有可选的标志和说明符。转换符、标志和说明符有很多，它们都记录在 [`java.util.Formatter`](https://docs.oracle.com/javase/8/docs/api/java/util/Formatter.html) 中。

下面是一个基本示例：

```java
int i = 461012;
System.out.format("The value of i is: %d%n", i);
```

`%d` 指定这个单一变量是一个十进制整数。`%n` 是一个与平台无关的换行符。输出为：

```text
The value of i is: 461012
```

`printf` 和 `format` 方法被重载了。它们各有一个语法如下的版本：

```java
public PrintStream format(Locale l, String format, Object... args)
```

例如，要以法国系统打印数字（在浮点数的英文表示中使用小数点的地方，法国系统使用逗号），你可以使用：

```java
System.out.format(Locale.FRANCE,
    "The value of the float " + "variable is %f, while the " +
    "value of the integer variable " + "is %d, and the string is %s%n", 
    floatVar, intVar, stringVar);
```

## 示例

下表列出了在下面的示例程序 `TestFormat.java` 中用到的一些转换符和标志。

| 转换符 | 标志 | 说明 |
| --- | --- | --- |
| d |  | 一个十进制整数。 |
| f |  | 一个浮点数。 |
| n |  | 适合于运行该应用程序的平台的换行字符。你应当始终使用 `%n`，而不是 `\n`。 |
| tB |  | 一个日期与时间转换——特定于语言环境的月份全名。 |

| tl |  | 一个日期与时间转换——12 小时制下的小时。 |
| tM |  | 一个日期与时间转换——2 位数字的分钟，按需在前补零。 |
| tp |  | 一个日期与时间转换——特定于语言环境的 am/pm（小写）。 |
| tm |  | 一个日期与时间转换——2 位数字的月份，按需在前补零。 |
| tD |  | 一个日期与时间转换——日期为 %tm%td%ty |
|  | 08 | 宽度为八个字符，按需在前补零。 |
|  | + | 包含符号，无论正负。 |
|  | , | 包含特定于语言环境的分组字符。 |
|  | \- | 左对齐。。 |

|  | 10.3 | 宽度为十个字符，右对齐，小数点后三位。 |

下面的程序展示了你可以用 `format` 做的一些格式化。输出显示在嵌入注释中的双引号内：

```java
import java.util.Calendar;
import java.util.Locale;

public class TestFormat {
    
    public static void main(String[] args) {
      long n = 461012;
      System.out.format("%d%n", n);      //  -->  "461012"
      System.out.format("%08d%n", n);    //  -->  "00461012"
      System.out.format("%+8d%n", n);    //  -->  " +461012"
      System.out.format("%,8d%n", n);    // -->  " 461,012"
      System.out.format("%+,8d%n%n", n); //  -->  "+461,012"
      
      double pi = Math.PI;

      System.out.format("%f%n", pi);       // -->  "3.141593"
      System.out.format("%.3f%n", pi);     // -->  "3.142"
      System.out.format("%10.3f%n", pi);   // -->  "     3.142"
      System.out.format("%-10.3f%n", pi);  // -->  "3.142"
      System.out.format(Locale.FRANCE,
                        "%-10.4f%n%n", pi); // -->  "3,1416"

      Calendar c = Calendar.getInstance();
      System.out.format("%tB %te, %tY%n", c, c, c); // -->  "May 29, 2006"

      System.out.format("%tl:%tM %tp%n", c, c, c);  // -->  "2:34 am"

      System.out.format("%tD%n", c);    // -->  "05/29/06"
    }
}
```

## DecimalFormat 类

你可以使用 [`java.text.DecimalFormat`](https://docs.oracle.com/javase/8/docs/api/java/text/DecimalFormat.html) 类来控制前导和尾随零、前缀和后缀、分组（千位）分隔符以及小数分隔符的显示。`DecimalFormat` 在数字的格式化方面提供了极大的灵活性，但它可能会使你的代码变得更加复杂。

下面的示例通过向 `DecimalFormat` 构造方法传递一个模式字符串来创建一个 `DecimalFormat` 对象 `myFormatter`。然后由 `myFormatter` 调用 `DecimalFormat` 从 `NumberFormat` 继承来的 `format()` 方法——它接受一个 `double` 值作为参数，并返回格式化后的数字字符串：

下面是一个演示 `DecimalFormat` 用法的示例程序：

```java
import java.text.*;

public class DecimalFormatDemo {

   static public void customFormat(String pattern, double value ) {
      DecimalFormat myFormatter = new DecimalFormat(pattern);
      String output = myFormatter.format(value);
      System.out.println(value + "  " + pattern + "  " + output);
   }

   static public void main(String[] args) {

      customFormat("###,###.###", 123456.789);
      customFormat("###.##", 123456.789);
      customFormat("000000.000", 123.78);
      customFormat("$###,###.###", 12345.67);  
   }
}
```

输出为：

```text
123456.789  ###,###.###  123,456.789
123456.789  ###.##  123456.79
123.78  000000.000  000123.780
12345.67  $###,###.###  $12,345.67
```

下表解释了每一行输出。

| 值 | 模式 | 输出 | 说明 |
| --- | --- | --- | --- |
| 123456.789 | ###,###.### | 123,456.789 | 井号（#）表示一个数字，逗号是分组分隔符的占位符，句点是小数分隔符的占位符。 |
| 123456.789 | ###.## | 123456.79 | `值` 的小数点右侧有三位数字，但 `模式` 只有两个。`format` 方法通过向上舍入来处理这一点。 |
| 123.78 | 000000.000 | 000123.780 | `模式` 指定了前导零和尾随零，因为这里使用的是字符 0 而非井号（#）。 |
| 12345.67 | $###,###.### | $12,345.67 | `模式` 中的第一个字符是美元符号（$）。注意它紧接在格式化后的 `输出` 中最左边的数字之前。 |
