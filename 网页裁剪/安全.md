## 学习路径：Java SE 的安全特性

在本学习路径中，您将了解内置的 Java™ 安全特性如何保护您免受恶意程序的侵害。您将看到如何使用工具来控制对资源的访问、生成和检查数字签名，以及创建和管理签名生成与检查所需的密钥。您还将看到如何将加密服务（如数字签名的生成和检查）集成到您的程序中。

Java 开发工具包（JDK™）提供的安全特性面向多种受众：

- **运行程序的用户**：
	内置的安全功能可保护您免受恶意程序（包括病毒）的侵害，维护您的文件和有关您的信息的隐私，并验证每个代码提供者的身份。需要时，您可以对应用程序和 applet 施加安全控制。
- **开发者**：
	您可以使用 API 方法将安全功能（包括加密服务和安全检查）集成到您的程序中。API 框架使您能够定义和集成自己的权限（控制对特定资源的访问）、加密服务实现、安全管理器实现以及策略实现。此外，还提供了用于管理您的公钥/私钥对以及来自您信任之人的公钥证书的类。
- **系统管理员、开发者和用户**：
	JDK 工具管理您的密钥库（keystore，即密钥和证书的数据库）；为 JAR 文件生成数字签名，并验证此类签名的真实性和已签名内容的完整性；创建和修改定义您的安装安全策略的策略文件。

## 学习路径课程

[创建策略文件（Creating a Policy File）](https://docs.oracle.com/javase/tutorial/security/tour1/index.html) 展示如何通过策略文件控制对资源的访问。如需了解策略配置文件的最新信息，请参见 [策略指南（Policy Guide）](https://docs.oracle.com/javase/8/docs/technotes/guides/security/PolicyGuide.html) 页面。

[应用程序控制快速导览（Quick Tour of Controlling Applications）](https://docs.oracle.com/javase/tutorial/security/tour2/index.html) 在上一课的基础上构建，展示在安全管理器下运行的应用程序除非被策略文件中的权限明确允许，否则不允许执行诸如读写文件之类的资源访问。

[用于安全代码和文件交换的 API 与工具（API and Tools Use for Secure Code and File Exchanges）](https://docs.oracle.com/javase/tutorial/security/sigcert/index.html) 定义数字签名、证书和密钥库，并讨论为什么需要它们。它还回顾了适用于接下来三节课的信息，这些信息涉及使用工具或 API 生成签名、导出/导入证书等通常所需的步骤。

[为代码签名并授予权限（Signing Code and Granting It Permissions）](https://docs.oracle.com/javase/tutorial/security/toolsign/index.html) 说明所有与安全相关工具的使用。它展示了开发者为代码签名并将其分发给他人运行的步骤。本课程还展示了将要运行代码的人（或系统管理员）如何在策略文件中添加条目，以授予代码执行所需资源访问的权限。

[交换文件（Exchanging Files）](https://docs.oracle.com/javase/tutorial/security/toolfilex/index.html) 展示一个人如何使用工具为重要文档（如合同）签名，并导出与用于签名合同的私钥对应的公钥的公钥证书。然后，本课程展示了接收合同、签名和公钥证书的另一个人如何导入证书并验证签名。

[生成和验证签名（Generating and Verifying Signatures）](https://docs.oracle.com/javase/tutorial/security/apisign/index.html) 逐步带您了解一个示例：编写使用 JDK 安全 API 生成密钥、用私钥为数据生成数字签名、并将公钥和签名导出到文件的 Java 程序。然后，示例展示编写第二个程序（可能运行在另一个人的计算机上），它导入公钥并验证签名的真实性。最后，示例讨论基本程序所用方法的潜在弱点，并演示了可能的替代方法和密钥提供/导入方式（包括在证书中）。

[实现您自己的权限（Implementing Your Own Permission）](https://docs.oracle.com/javase/tutorial/security/userperm/index.html) 演示如何编写定义自己特殊权限的类。
