## 路线：Java SE 的安全特性

在本路线中，你将学习 Java™ 内置的安全特性如何保护你免受恶意程序侵害。你将看到如何使用工具来控制对资源的访问、生成和校验数字签名，以及创建和管理签名生成与校验所需的密钥。你还将看到如何将加密服务（例如数字签名的生成和校验）集成到你的程序中。

Java 开发工具包(JDK™)提供的安全特性面向多种受众：

- **运行程序的用户**：
	内置的安全功能保护你免受恶意程序（包括病毒）侵害，维护你的文件和关于你的信息的隐私，并验证每个代码提供者的身份。在需要时，你可以让应用程序和 applet 受到安全控制。
- **开发者**：
	你可以使用 API 方法将安全功能（包括加密服务和安全检查）集成到你的程序中。该 API 框架使你能够定义并集成自己的权限（控制对特定资源的访问）、加密服务实现、安全管理器实现和策略实现。此外，还提供了用于管理来自你信任的人的公钥/私钥对以及公钥证书的类。
- **系统管理员、开发者和用户**：
	JDK 工具管理你的密钥库(keystore)（密钥和证书的数据库）；为 JAR 文件生成数字签名，并验证此类签名的真实性和已签名内容的完整性；以及创建和修改定义你安装环境安全策略的策略文件。

## 路线课程

[[安全-策略文件|创建策略文件]] 展示如何通过策略文件控制资源访问。有关策略配置文件的最新信息，请参阅[策略指南](https://docs.oracle.com/javase/8/docs/technotes/guides/security/PolicyGuide.html)页面。

[[安全-控制应用|控制应用程序快速导览]] 在前一课基础上构建，展示在安全管理器下运行的应用程序如何不被允许进行诸如读取或写入文件之类的资源访问，除非策略文件中的权限明确允许。

[[安全-签名与证书|用于安全代码和文件交换的 API 与工具]] 定义了数字签名、证书和密钥库，并讨论了为什么需要它们。它还回顾了适用于接下来三课的信息，涉及使用工具或 API 生成签名、导出/导入证书等通常需要的步骤。

[[安全-代码签名|为代码签名并授予权限]] 说明所有与安全相关的工具的使用。它展示了开发者签名并分发代码供他人运行的步骤。本课还展示运行代码的人（或系统管理员）如何在策略文件中添加条目，以授予代码进行其所需资源访问的权限。

[[安全-交换文件|交换文件]] 展示一个人如何使用工具签署一份重要文档（例如合同），并为用于签署合同的私钥所对应的公钥导出公钥证书。然后本课展示接收合同、签名和公钥证书的另一个人如何导入证书并验证签名。

[[安全-生成验证签名|生成和验证签名]] 逐步带你编写一个使用 JDK 安全 API 的 Java 程序示例，生成密钥、使用私钥为数据生成数字签名，并将公钥和签名导出到文件。然后示例展示编写第二个程序（预期运行在另一个人的计算机上），它导入公钥并验证签名的真实性。最后，示例讨论基础程序所用方法的潜在弱点，并演示了供应和导入密钥（包括在证书中）的可行替代方案和方法。

[[安全-自定义权限|实现你自己的权限]] 演示如何编写一个定义自己特殊权限的类。
