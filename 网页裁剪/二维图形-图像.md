## 课程：使用图像

正如你已从[[二维图形-images|图像]]一课中学到的，`Image` 由宽度和高度（以像素为单位）描述，并具有独立于绘图表面的坐标系。

处理图像时有许多常见任务。

- 将外部 GIF、PNG JPEG 图像格式文件加载到 Java 2D 使用的内部图像表示中。
- 直接创建 Java 2D 图像并渲染到它。
- 将 Java 2D 图像的内容绘制到绘图表面上。
- 将 Java 2D 图像的内容保存到外部 GIF、PNG 或 JPEG 图像文件。

本课教你加载、显示和保存图像的基础知识。

你必须了解两个主要类来处理图像：

- [`java.awt.Image`](https://docs.oracle.com/javase/8/docs/api/java/awt/Image.html) 类是超类，将图形图像表示为像素的矩形数组。
- [`java.awt.image.BufferedImage`](https://docs.oracle.com/javase/8/docs/api/java/awt/image/BufferedImage.html) 类扩展 `Image` 类，允许应用程序直接操作图像数据（例如，检索或设置像素颜色）。应用程序可以直接构造此类的实例。

`BufferedImage` 类是 Java 2D 即时模式成像 API 的基石。它在内存中管理图像，并提供存储、解释和获取像素数据的方法。由于 `BufferedImage` 是 `Image` 的子类，它可以由接受 `Image` 参数的 `Graphics` 和 `Graphics2D` 方法渲染。

`BufferedImage` 本质上是具有可访问数据缓冲区的 `Image`。因此，直接使用 `BufferedImage` 更高效。`BufferedImage` 具有图像数据的*颜色模型(ColorModel)*和*光栅(Raster)*。ColorModel 提供图像像素数据的颜色解释。

Raster 执行以下功能：

- 表示图像的矩形坐标
- 在内存中维护图像数据
- 提供从单个图像数据缓冲区创建多个子图像的机制
- 提供访问图像中特定像素的方法
