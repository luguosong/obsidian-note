## 基本数据类型

Java 编程语言是静态类型(statically-typed)的，这意味着所有变量都必须先声明后才能使用。这包括说明变量的类型和名称，正如您已经看到的：

```java
int gear = 1;
```

这样做就告诉您的程序，存在一个名为「gear」的字段，它保存数值数据，初始值为「1」。变量的数据类型决定了它可能包含的值，以及可以对其执行的操作。除了 `int` 之外，Java 编程语言还支持另外七种*基本数据类型(primitive data type)*。基本类型由语言预先定义，并由保留关键字命名。基本值之间不共享状态。Java 编程语言支持的八种基本数据类型是：

- **byte**：`byte` 数据类型是 8 位有符号二进制补码整数。它的最小值为 -128，最大值为 127（含）。在大型[[数组|数组]]中，`byte` 数据类型对于节省内存很有用，在这种情况下内存的节省确实重要。也可以在变量的范围限制有助于澄清代码意图的地方用它来代替 `int`；变量范围受限这一事实本身就能起到一种文档说明的作用。
- **short**：`short` 数据类型是 16 位有符号二进制补码整数。它的最小值为 -32,768，最大值为 32,767（含）。与 `byte` 一样，适用相同的准则：在内存节省确实重要的情况下，可以使用 `short` 来节省大型数组中的内存。
- **int**：默认情况下，`int` 数据类型是 32 位有符号二进制补码整数，其最小值为 -2<sup>31</sup>，最大值为 2<sup>31</sup>-1。在 Java SE 8 及更高版本中，您可以使用 `int` 数据类型来表示无符号 32 位整数，其最小值为 0，最大值为 2<sup>32</sup>-1。使用 Integer 类即可将 `int` 数据类型作为无符号整数使用。更多信息请参见 Number 类一节。诸如 `compareUnsigned`、`divideUnsigned` 等静态方法已被添加到 [`Integer`](https://docs.oracle.com/javase/8/docs/api/java/lang/Integer.html) 类中，以支持无符号整数的算术运算。
- **long**：`long` 数据类型是 64 位二进制补码整数。有符号 long 的最小值为 -2<sup>63</sup>，最大值为 2<sup>63</sup>-1。在 Java SE 8 及更高版本中，您可以使用 `long` 数据类型来表示无符号 64 位 long，其最小值为 0，最大值为 2<sup>64</sup>-1。当您需要比 `int` 所提供的更宽的取值范围时，请使用此数据类型。[`Long`](https://docs.oracle.com/javase/8/docs/api/java/lang/Long.html) 类同样包含 `compareUnsigned`、`divideUnsigned` 等方法，以支持无符号 long 的算术运算。
- **float**：`float` 数据类型是单精度 32 位 IEEE 754 浮点数。它的取值范围超出了本讨论的范围，但在 Java 语言规范的[浮点类型、格式与取值（Floating-Point Types, Formats, and Values）](https://docs.oracle.com/javase/specs/jls/se7/html/jls-4.html#jls-4.2.3)一节中有规定。与对 `byte` 和 `short` 的建议一样，如果需要在大型浮点数数组中节省内存，请使用 `float`（而非 `double`）。此数据类型永远不应用于精确值，例如货币。为此，您需要改用 [java.math.BigDecimal](https://docs.oracle.com/javase/8/docs/api/java/math/BigDecimal.html) 类。[数字和字符串（Numbers and Strings）](https://docs.oracle.com/javase/tutorial/java/data/index.html)涵盖了 `BigDecimal` 及 Java 平台提供的其它有用类。
- **double**：`double` 数据类型是双精度 64 位 IEEE 754 浮点数。它的取值范围超出了本讨论的范围，但在 Java 语言规范的[浮点类型、格式与取值（Floating-Point Types, Formats, and Values）](https://docs.oracle.com/javase/specs/jls/se7/html/jls-4.html#jls-4.2.3)一节中有规定。对于十进制值，此数据类型通常是默认选择。如上所述，此数据类型永远不应用于精确值，例如货币。
- **boolean**：`boolean` 数据类型只有两个可能的值：`true` 和 `false`。将此数据类型用于跟踪真/假条件的简单标志。此数据类型表示一位信息，但它的「大小」并没有一个精确定义的值。
- **char**：`char` 数据类型是单个 16 位 Unicode 字符。它的最小值为 `'\u0000'`（即 0），最大值为 `'\uffff'`（即 65,535，含）。

除了上面列出的八种基本数据类型之外，Java 编程语言还通过 [java.lang.String](https://docs.oracle.com/javase/8/docs/api/java/lang/String.html) 类对字符串提供了特殊支持。用双引号将字符串括起来会自动创建一个新的 `String` 对象；例如 `String s = "this is a string";`。`String` 对象是*不可变(immutable)*的，这意味着一旦创建，它们的值就不能再改变。`String` 类从技术上讲并不是基本数据类型，但考虑到语言赋予它的特殊支持，您可能也会倾向于把它当作基本数据类型来看待。您将在[简单数据对象（Simple Data Objects）](https://docs.oracle.com/javase/tutorial/java/data/index.html)中学到更多关于 `String` 类的知识。

## 默认值

在声明字段时并不总是需要赋值。声明但未初始化的字段会被编译器设置为一个合理的默认值。一般而言，这个默认值将是零或 `null`，具体取决于数据类型。然而，依赖此类默认值通常被认为是不好的编程风格。

下面的图表总结了上述数据类型的默认值。

| **数据类型** | **默认值（针对字段）** |
| --- | --- |
| byte | 0 |
| short | 0 |
| int | 0 |
| long | 0L |
| float | 0.0f |
| double | 0.0d |
| char | '\\u0000' |
| String（或任何对象） | null |
| boolean | false |

局部变量略有不同；编译器永远不会为未初始化的局部变量赋默认值。如果无法在声明局部变量的地方对其进行初始化，请务必在尝试使用它之前为其赋值。访问未初始化的局部变量将导致编译时错误。

### 字面量

您可能已经注意到，在初始化基本类型的变量时并没有使用 `new` 关键字。基本类型是内置于语言中的特殊数据类型；它们不是从类创建的对象。*字面量(literal)*是固定值的源代码表示；字面量直接在您的代码中表示，无需进行计算。如下所示，可以将字面量赋给基本类型的变量：

```java
boolean result = true;
char capitalC = 'C';
byte b = 100;
short s = 10000;
int i = 100000;
```

#### 整数字面量

如果整数字面量(integer literal)以字母 `L` 或 `l` 结尾，则为 `long` 类型；否则为 `int` 类型。建议您使用大写字母 `L`，因为小写字母 `l` 很难与数字 `1` 区分。

整型类型 `byte`、`short`、`int` 和 `long` 的值可以由 `int` 字面量创建。超出 `int` 范围的 `long` 类型值可以由 `long` 字面量创建。整数字面量可以用以下这些数制来表示：

- 十进制：基数为 10，其数字由 0 到 9 组成；这是您每天使用的数制
- 十六进制：基数为 16，其数字由 0 到 9 以及字母 A 到 F 组成
- 二进制：基数为 2，其数字由 0 和 1 组成（您可以在 Java SE 7 及更高版本中创建二进制字面量）

对于通用编程而言，十进制很可能是您唯一会使用的数制。然而，如果您需要使用其它数制，下面的示例展示了正确的语法。前缀 `0x` 表示十六进制，`0b` 表示二进制：

```java
// The number 26, in decimal
int decVal = 26;
//  The number 26, in hexadecimal
int hexVal = 0x1a;
// The number 26, in binary
int binVal = 0b11010;
```

#### 浮点字面量

如果浮点字面量(floating-point literal)以字母 `F` 或 `f` 结尾，则为 `float` 类型；否则其类型为 `double`，并且可以选择以字母 `D` 或 `d` 结尾。

浮点类型（`float` 和 `double`）也可以使用 E 或 e（科学记数法）、F 或 f（32 位 float 字面量）以及 D 或 d（64 位 double 字面量；这是默认值，按约定省略）来表示。

```java
double d1 = 123.4;
// same value as d1, but in scientific notation
double d2 = 1.234e2;
float f1  = 123.4f;
```

#### 字符和字符串字面量

`char` 和 `String` 类型的字面量可以包含任何 Unicode（UTF-16）字符。如果您的编辑器和文件系统允许，您可以直接在代码中使用这些字符。如果不允许，您可以使用「Unicode 转义」，例如 `'\u0108'`（带扬抑符的大写 C），或 `"S\u00ED Se\u00F1or"`（西班牙语 Sí Señor）。对 `char` 字面量始终使用「单引号」，对 `String` 字面量始终使用「双引号」。Unicode 转义序列也可以用于程序中的其它地方（例如字段名），而不仅限于 `char` 或 `String` 字面量。

Java 编程语言还为 `char` 和 `String` 字面量支持一些特殊的转义序列：`\b`（退格）、`\t`（制表符）、`\n`（换行）、`\f`（换页）、`\r`（回车）、`\"`（双引号）、`\'`（单引号）和 `\\`（反斜杠）。

还有一种特殊的 `null` 字面量，可用作任何引用类型的值。可以将 `null` 赋给任何变量，基本类型的变量除外。除了测试它是否存在之外，您几乎无法用 `null` 值做其它事情。因此，`null` 常在程序中用作标记，表示某个对象不可用。

最后，还有一种特殊的字面量，称为*类字面量(class literal)*，通过取一个类型名并附加「`.class`」来形成；例如 `String.class`。它指的是表示该类型本身的对象（类型为 `Class`）。

## 在数字字面量中使用下划线字符

在 Java SE 7 及更高版本中，任意数量的下划线字符（`_`）可以出现在数字字面量中数字之间的任何位置。例如，此特性使您能够分隔数字字面量中的数字组，从而可以提高代码的可读性。

例如，如果您的代码包含许多位数字，您可以使用下划线字符将数字每三位分为一组，类似于您使用逗号或空格等标点符号作为分隔符的方式。

以下示例展示了您在数字字面量中使用下划线的其它方式：

```java
long creditCardNumber = 1234_5678_9012_3456L;
long socialSecurityNumber = 999_99_9999L;
float pi =  3.14_15F;
long hexBytes = 0xFF_EC_DE_5E;
long hexWords = 0xCAFE_BABE;
long maxLong = 0x7fff_ffff_ffff_ffffL;
byte nybbles = 0b0010_0101;
long bytes = 0b11010010_01101001_10010100_10010010;
```

下划线只能放在数字之间；不能在以下位置放置下划线：

- 在数字的开头或结尾
- 紧邻浮点字面量中的小数点
- 在 `F` 或 `L` 后缀之前
- 在需要一串数字的位置

以下示例演示了数字字面量中有效和无效的下划线放置（已高亮）：

```java
// Invalid: cannot put underscores
// adjacent to a decimal point
float pi1 = 3_.1415F;
// Invalid: cannot put underscores 
// adjacent to a decimal point
float pi2 = 3._1415F;
// Invalid: cannot put underscores 
// prior to an L suffix
long socialSecurityNumber1 = 999_99_9999_L;

// OK (decimal literal)
int x1 = 5_2;
// Invalid: cannot put underscores
// At the end of a literal
int x2 = 52_;
// OK (decimal literal)
int x3 = 5_______2;

// Invalid: cannot put underscores
// in the 0x radix prefix
int x4 = 0_x52;
// Invalid: cannot put underscores
// at the beginning of a number
int x5 = 0x_52;
// OK (hexadecimal literal)
int x6 = 0x5_2; 
// Invalid: cannot put underscores
// at the end of a number
int x7 = 0x52_;
```
