## 默认方法

[接口](https://docs.oracle.com/javase/tutorial/java/IandI/createinterface.html)一节描述了一个涉及计算机控制汽车制造商的示例，这些制造商发布了行业标准接口，规定了可以调用哪些方法来操作他们的汽车。如果这些计算机制造商为他们的汽车增加了诸如飞行之类的新功能呢？这些制造商将需要指定新的方法，以便让其他公司（例如电子导航仪器制造商）能够使其软件适配飞行汽车。这些汽车制造商会在哪里声明这些与飞行相关的新方法呢？如果他们将这些方法添加到原始接口中，那么已经实现了这些接口的程序员将不得不重写他们的实现。如果他们将这些方法添加为静态方法，那么程序员会把它们当作实用工具方法，而不是必不可少的、核心的方法。

默认方法（default method）使你能够在库的接口中添加新功能，并确保与为这些接口的旧版本编写的代码保持二进制兼容。

考虑下面的接口 [`TimeClient`](https://docs.oracle.com/javase/tutorial/java/IandI/examples/TimeClient.java)，如[接口的问题与练习答案（Answers to Questions and Exercises: Interfaces）](https://docs.oracle.com/javase/tutorial/java/IandI/QandE/interfaces-answers.html)中所述：

```java
import java.time.*; 
 
public interface TimeClient {
    void setTime(int hour, int minute, int second);
    void setDate(int day, int month, int year);
    void setDateAndTime(int day, int month, int year,
                               int hour, int minute, int second);
    LocalDateTime getLocalDateTime();
}
```

下面的类 [`SimpleTimeClient`](https://docs.oracle.com/javase/tutorial/java/IandI/examples/defaultmethods/SimpleTimeClient.java) 实现了 `TimeClient`：

```java
package defaultmethods;

import java.time.*;
import java.lang.*;
import java.util.*;

public class SimpleTimeClient implements TimeClient {
    
    private LocalDateTime dateAndTime;
    
    public SimpleTimeClient() {
        dateAndTime = LocalDateTime.now();
    }
    
    public void setTime(int hour, int minute, int second) {
        LocalDate currentDate = LocalDate.from(dateAndTime);
        LocalTime timeToSet = LocalTime.of(hour, minute, second);
        dateAndTime = LocalDateTime.of(currentDate, timeToSet);
    }
    
    public void setDate(int day, int month, int year) {
        LocalDate dateToSet = LocalDate.of(day, month, year);
        LocalTime currentTime = LocalTime.from(dateAndTime);
        dateAndTime = LocalDateTime.of(dateToSet, currentTime);
    }
    
    public void setDateAndTime(int day, int month, int year,
                               int hour, int minute, int second) {
        LocalDate dateToSet = LocalDate.of(day, month, year);
        LocalTime timeToSet = LocalTime.of(hour, minute, second); 
        dateAndTime = LocalDateTime.of(dateToSet, timeToSet);
    }
    
    public LocalDateTime getLocalDateTime() {
        return dateAndTime;
    }
    
    public String toString() {
        return dateAndTime.toString();
    }
    
    public static void main(String... args) {
        TimeClient myTimeClient = new SimpleTimeClient();
        System.out.println(myTimeClient.toString());
    }
}
```

假设你想为 `TimeClient` 接口添加新功能，例如通过 [`ZonedDateTime`](https://docs.oracle.com/javase/8/docs/api/java/time/ZonedDateTime.html) 对象指定时区的能力（它类似于 [`LocalDateTime`](https://docs.oracle.com/javase/8/docs/api/java/time/LocalDateTime.html) 对象，不同之处在于它存储了时区信息）：

```java
public interface TimeClient {
    void setTime(int hour, int minute, int second);
    void setDate(int day, int month, int year);
    void setDateAndTime(int day, int month, int year,
        int hour, int minute, int second);
    LocalDateTime getLocalDateTime();                           
    ZonedDateTime getZonedDateTime(String zoneString);
}
```

按照对 `TimeClient` 接口的这一修改，你还必须修改 `SimpleTimeClient` 类并实现 `getZonedDateTime` 方法。然而，你可以不把 `getZonedDateTime` 留作 `abstract`（如上一个示例所示），而是定义一个*默认实现*。（请记住，[抽象方法（abstract method）](https://docs.oracle.com/javase/tutorial/java/IandI/abstract.html)是没有实现就声明的方法。）

```java
package defaultmethods;
 
import java.time.*;

public interface TimeClient {
    void setTime(int hour, int minute, int second);
    void setDate(int day, int month, int year);
    void setDateAndTime(int day, int month, int year,
                               int hour, int minute, int second);
    LocalDateTime getLocalDateTime();
    
    static ZoneId getZoneId (String zoneString) {
        try {
            return ZoneId.of(zoneString);
        } catch (DateTimeException e) {
            System.err.println("Invalid time zone: " + zoneString +
                "; using default time zone instead.");
            return ZoneId.systemDefault();
        }
    }
        
    default ZonedDateTime getZonedDateTime(String zoneString) {
        return ZonedDateTime.of(getLocalDateTime(), getZoneId(zoneString));
    }
}
```

你在方法签名的开头使用 `default` 关键字来指定接口中的某个方法定义是默认方法。接口中的所有方法声明（包括默认方法）都隐式为 `public`，因此你可以省略 `public` 修饰符。

有了这个接口，你就不必再修改 `SimpleTimeClient` 类，而且该类（以及任何实现了 `TimeClient` 接口的类）都将已经定义好了 `getZonedDateTime` 方法。下面的示例 [`TestSimpleTimeClient`](https://docs.oracle.com/javase/tutorial/java/IandI/examples/defaultmethods/TestSimpleTimeClient.java) 从 `SimpleTimeClient` 的实例调用 `getZonedDateTime` 方法：

```java
package defaultmethods;
 
import java.time.*;
import java.lang.*;
import java.util.*;

public class TestSimpleTimeClient {
    public static void main(String... args) {
        TimeClient myTimeClient = new SimpleTimeClient();
        System.out.println("Current time: " + myTimeClient.toString());
        System.out.println("Time in California: " +
            myTimeClient.getZonedDateTime("Blah blah").toString());
    }
}
```

## 扩展包含默认方法的接口

当你扩展一个包含默认方法的接口时，你可以执行以下操作：

- 完全不提及该默认方法，从而让你的扩展接口继承该默认方法。
- 重新声明该默认方法，使其变为 `abstract`。
- 重新定义该默认方法，从而覆盖它。

假设你按如下方式扩展 `TimeClient` 接口：

```java
public interface AnotherTimeClient extends TimeClient { }
```

任何实现 `AnotherTimeClient` 接口的类都将拥有由默认方法 `TimeClient.getZonedDateTime` 指定的实现。

假设你按如下方式扩展 `TimeClient` 接口：

```java
public interface AbstractZoneTimeClient extends TimeClient {
    public ZonedDateTime getZonedDateTime(String zoneString);
}
```

任何实现 `AbstractZoneTimeClient` 接口的类都必须实现 `getZonedDateTime` 方法；该方法是一个 `abstract` 方法，就像接口中所有其他非默认（且非静态）的方法一样。

假设你按如下方式扩展 `TimeClient` 接口：

```java
public interface HandleInvalidTimeZoneClient extends TimeClient {
    default public ZonedDateTime getZonedDateTime(String zoneString) {
        try {
            return ZonedDateTime.of(getLocalDateTime(),ZoneId.of(zoneString)); 
        } catch (DateTimeException e) {
            System.err.println("Invalid zone ID: " + zoneString +
                "; using the default time zone instead.");
            return ZonedDateTime.of(getLocalDateTime(),ZoneId.systemDefault());
        }
    }
}
```

任何实现 `HandleInvalidTimeZoneClient` 接口的类都将使用该接口指定的 `getZonedDateTime` 实现，而非 `TimeClient` 接口指定的实现。

## 静态方法

除默认方法外，你还可以在接口中定义[静态方法（static method）](https://docs.oracle.com/javase/tutorial/java/javaOO/classvars.html)。（静态方法是与其定义所在的类相关联、而非与任何对象相关联的方法。该类的每个实例都共享其静态方法。）这使你能够更轻松地在库中组织辅助方法；你可以将特定于某个接口的静态方法保留在同一接口中，而不是放在单独的类中。下面的示例定义了一个静态方法，它检索与某个时区标识符相对应的 [`ZoneId`](https://docs.oracle.com/javase/8/docs/api/java/time/ZoneId.html) 对象；如果没有与给定标识符相对应的 `ZoneId` 对象，则使用系统默认时区。（因此，你可以简化 `getZonedDateTime` 方法）：

```java
public interface TimeClient {
    // ...
    static public ZoneId getZoneId (String zoneString) {
        try {
            return ZoneId.of(zoneString);
        } catch (DateTimeException e) {
            System.err.println("Invalid time zone: " + zoneString +
                "; using default time zone instead.");
            return ZoneId.systemDefault();
        }
    }

    default public ZonedDateTime getZonedDateTime(String zoneString) {
        return ZonedDateTime.of(getLocalDateTime(), getZoneId(zoneString));
    }    
}
```

与类中的静态方法一样，你在方法签名的开头使用 `static` 关键字来指定接口中的某个方法定义是静态方法。接口中的所有方法声明（包括静态方法）都隐式为 `public`，因此你可以省略 `public` 修饰符。

## 将默认方法集成到现有库中

默认方法使你能够为现有接口添加新功能，并确保与为这些接口的旧版本编写的代码保持二进制兼容。特别是，默认方法使你能够将接受 lambda 表达式作为参数的方法添加到现有接口中。本节演示 [`Comparator`](https://docs.oracle.com/javase/8/docs/api/java/util/Comparator.html) 接口是如何通过默认方法和静态方法被增强的。

考虑[类的问题与练习（Questions and Exercises: Classes）](https://docs.oracle.com/javase/tutorial/java/javaOO/QandE/creating-questions.html)中描述的 `Card` 和 `Deck` 类。本示例将 [`Card`](https://docs.oracle.com/javase/tutorial/java/IandI/examples/defaultmethods/Card.java) 和 [`Deck`](https://docs.oracle.com/javase/tutorial/java/IandI/examples/defaultmethods/Deck.java) 类重写为接口。`Card` 接口包含两个 `enum` 类型（`Suit` 和 `Rank`）以及两个抽象方法（`getSuit` 和 `getRank`）：

```java
package defaultmethods;

public interface Card extends Comparable<Card> {
    
    public enum Suit { 
        DIAMONDS (1, "Diamonds"), 
        CLUBS    (2, "Clubs"   ), 
        HEARTS   (3, "Hearts"  ), 
        SPADES   (4, "Spades"  );
        
        private final int value;
        private final String text;
        Suit(int value, String text) {
            this.value = value;
            this.text = text;
        }
        public int value() {return value;}
        public String text() {return text;}
    }
    
    public enum Rank { 
        DEUCE  (2 , "Two"  ),
        THREE  (3 , "Three"), 
        FOUR   (4 , "Four" ), 
        FIVE   (5 , "Five" ), 
        SIX    (6 , "Six"  ), 
        SEVEN  (7 , "Seven"),
        EIGHT  (8 , "Eight"), 
        NINE   (9 , "Nine" ), 
        TEN    (10, "Ten"  ), 
        JACK   (11, "Jack" ),
        QUEEN  (12, "Queen"), 
        KING   (13, "King" ),
        ACE    (14, "Ace"  );
        private final int value;
        private final String text;
        Rank(int value, String text) {
            this.value = value;
            this.text = text;
        }
        public int value() {return value;}
        public String text() {return text;}
    }
    
    public Card.Suit getSuit();
    public Card.Rank getRank();
}
```

`Deck` 接口包含操作一副牌中各张牌的各种方法：

```java
package defaultmethods; 
 
import java.util.*;
import java.util.stream.*;
import java.lang.*;
 
public interface Deck {
    
    List<Card> getCards();
    Deck deckFactory();
    int size();
    void addCard(Card card);
    void addCards(List<Card> cards);
    void addDeck(Deck deck);
    void shuffle();
    void sort();
    void sort(Comparator<Card> c);
    String deckToString();

    Map<Integer, Deck> deal(int players, int numberOfCards)
        throws IllegalArgumentException;

}
```

类 [`PlayingCard`](https://docs.oracle.com/javase/tutorial/java/IandI/examples/defaultmethods/PlayingCard.java) 实现了 `Card` 接口，类 [`StandardDeck`](https://docs.oracle.com/javase/tutorial/java/IandI/examples/defaultmethods/StandardDeck.java) 实现了 `Deck` 接口。

`StandardDeck` 类按如下方式实现抽象方法 `Deck.sort`：

```java
public class StandardDeck implements Deck {
    
    private List<Card> entireDeck;
    
    // ...
    
    public void sort() {
        Collections.sort(entireDeck);
    }
    
    // ...
}
```

方法 `Collections.sort` 对其元素类型实现了 [`Comparable`](https://docs.oracle.com/javase/8/docs/api/java/lang/Comparable.html) 接口的 `List` 实例进行排序。成员 `entireDeck` 是一个 `List` 实例，其元素类型为 `Card`，而 `Card` 扩展了 `Comparable`。`PlayingCard` 类按如下方式实现 [`Comparable.compareTo`](https://docs.oracle.com/javase/8/docs/api/java/lang/Comparable.html#compareTo-T-) 方法：

```java
public int hashCode() {
    return ((suit.value()-1)*13)+rank.value();
}

public int compareTo(Card o) {
    return this.hashCode() - o.hashCode();
}
```

`compareTo` 方法使得 `StandardDeck.sort()` 先按花色、再按点数对这副牌进行排序。

如果你想先按点数、再按花色对这副牌进行排序呢？你就需要实现 [`Comparator`](https://docs.oracle.com/javase/8/docs/api/java/util/Comparator.html) 接口来指定新的排序标准，并使用 [`sort(List<T> list, Comparator<? super T> c)`](https://docs.oracle.com/javase/8/docs/api/java/util/Collections.html#sort-java.util.List-java.util.Comparator-) 方法（即包含 `Comparator` 参数的 `sort` 方法版本）。你可以在 `StandardDeck` 类中定义如下方法：

```java
public void sort(Comparator<Card> c) {
    Collections.sort(entireDeck, c);
}
```

有了这个方法，你就可以指定 `Collections.sort` 如何对 `Card` 类的实例进行排序。一种方式是实现 `Comparator` 接口来指定你希望牌如何排序。示例 [`SortByRankThenSuit`](https://docs.oracle.com/javase/tutorial/java/IandI/examples/defaultmethods/SortByRankThenSuit.java) 正是这样做的：

```java
package defaultmethods;

import java.util.*;
import java.util.stream.*;
import java.lang.*;

public class SortByRankThenSuit implements Comparator<Card> {
    public int compare(Card firstCard, Card secondCard) {
        int compVal =
            firstCard.getRank().value() - secondCard.getRank().value();
        if (compVal != 0)
            return compVal;
        else
            return firstCard.getSuit().value() - secondCard.getSuit().value(); 
    }
}
```

下面的调用先按点数、再按花色对这副牌进行排序：

```java
StandardDeck myDeck = new StandardDeck();
myDeck.shuffle();
myDeck.sort(new SortByRankThenSuit());
```

然而，这种方式过于冗长；如果能够仅指定排序标准、避免创建多个排序实现，那就更好了。假设你是编写 `Comparator` 接口的开发者。你可以为 `Comparator` 接口添加哪些默认方法或静态方法，使其他开发者能够更轻松地指定排序标准？

首先，假设你想按点数对这副牌进行排序，不考虑花色。你可以按如下方式调用 `StandardDeck.sort` 方法：

```java
StandardDeck myDeck = new StandardDeck();
myDeck.shuffle();
myDeck.sort(
    (firstCard, secondCard) ->
        firstCard.getRank().value() - secondCard.getRank().value()
);
```

由于 `Comparator` 接口是一个[函数式接口（functional interface）](https://docs.oracle.com/javase/tutorial/java/javaOO/lambdaexpressions.html#approach6)，你可以将 lambda 表达式作为 `sort` 方法的参数。在这个示例中，lambda 表达式比较两个整数值。

如果开发者只需通过调用方法 `Card.getRank` 就能创建 `Comparator` 实例，对他们来说会更简单。特别是，如果开发者能够创建一个可以比较任何能够从诸如 `getValue` 或 `hashCode` 等方法返回数值的对象的 `Comparator` 实例，将会很有帮助。`Comparator` 接口通过静态方法 [`comparing`](https://docs.oracle.com/javase/8/docs/api/java/util/Comparator.html#comparing-java.util.function.Function-java.util.Comparator-) 获得了这种能力：

```java
myDeck.sort(Comparator.comparing((card) -> card.getRank()));
```

在这个示例中，你可以改用[方法引用（method reference）](https://docs.oracle.com/javase/tutorial/java/javaOO/methodreferences.html)：

```java
myDeck.sort(Comparator.comparing(Card::getRank));
```

这个调用更好地展示了如何指定不同的排序标准并避免创建多个排序实现。

`Comparator` 接口还通过 `comparing` 静态方法的其他版本得到了增强，例如 [`comparingDouble`](https://docs.oracle.com/javase/8/docs/api/java/util/Comparator.html#comparingDouble-java.util.function.ToDoubleFunction-java.util.Comparator-) 和 [`comparingLong`](https://docs.oracle.com/javase/8/docs/api/java/util/Comparator.html#comparingLong-java.util.function.ToLongFunction-)，它们使你能够创建比较其他数据类型的 `Comparator` 实例。

假设你的开发者希望创建一个能够按多个标准比较对象的 `Comparator` 实例。例如，你如何先按点数、再按花色对这副牌进行排序？与之前一样，你可以使用 lambda 表达式来指定这些排序标准：

```java
StandardDeck myDeck = new StandardDeck();
myDeck.shuffle();
myDeck.sort(
    (firstCard, secondCard) -> {
        int compare =
            firstCard.getRank().value() - secondCard.getRank().value();
        if (compare != 0)
            return compare;
        else
            return firstCard.getSuit().value() - secondCard.getSuit().value();
    }      
);
```

如果开发者能够从一系列 `Comparator` 实例构建出一个 `Comparator` 实例，对他们来说会更简单。`Comparator` 接口通过默认方法 [`thenComparing`](https://docs.oracle.com/javase/8/docs/api/java/util/Comparator.html#thenComparing-java.util.Comparator-) 获得了这种能力：

```java
myDeck.sort(
    Comparator
        .comparing(Card::getRank)
        .thenComparing(Comparator.comparing(Card::getSuit)));
```

`Comparator` 接口还通过 `thenComparing` 默认方法的其他版本得到了增强（例如 [`thenComparingDouble`](https://docs.oracle.com/javase/8/docs/api/java/util/Comparator.html#thenComparingDouble-java.util.function.ToDoubleFunction-) 和 [`thenComparingLong`](https://docs.oracle.com/javase/8/docs/api/java/util/Comparator.html#thenComparingLong-java.util.function.ToLongFunction-)），它们使你能够构建比较其他数据类型的 `Comparator` 实例。

假设你的开发者希望创建一个能够按相反顺序对对象集合进行排序的 `Comparator` 实例。例如，你如何先按点数降序、从 A 到 2（而不是从 2 到 A）对这副牌进行排序？与之前一样，你可以指定另一个 lambda 表达式。然而，如果开发者能够通过调用某个方法来反转一个现有的 `Comparator`，对他们来说会更简单。`Comparator` 接口通过默认方法 [`reversed`](https://docs.oracle.com/javase/8/docs/api/java/util/Comparator.html#reversed--) 获得了这种能力：

```java
myDeck.sort(
    Comparator.comparing(Card::getRank)
        .reversed()
        .thenComparing(Comparator.comparing(Card::getSuit)));
```

这个示例演示了 `Comparator` 接口是如何通过默认方法、静态方法、lambda 表达式和方法引用得到增强的，从而创建出更具表达力的库方法，程序员只需查看这些方法是如何被调用的，就能迅速推断出它们的功能。使用这些构造来增强你库中的接口。
