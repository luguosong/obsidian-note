## 泛型类型

*泛型类型（generic type）*是按类型参数化的泛型类或接口。下面的 Box 类将被修改以演示这一概念。

## 一个简单的 Box 类

首先来看一个操作任意类型对象的非泛型 Box 类。它只需提供两个方法：set，用于将一个对象加入盒子；get，用于取出它：

```java
public class Box {
    private Object object;

    public void set(Object object) { this.object = object; }
    public Object get() { return object; }
}
```

由于它的方法接受或返回的是 Object，你可以自由地传入任何你想要的东西，前提是它不是基本类型之一。在编译时无法验证该类是如何被使用的。代码的某一部分可能将一个 Integer 放入盒子，并期望从中取出 Integer，而代码的另一部分可能错误地传入一个 String，从而导致运行时错误。

## Box 类的泛型版本

*泛型类（generic class）*用以下格式定义：

```java
class name<T1, T2, ..., Tn> { /* ... */ }
```

类型参数部分由尖括号（<>）界定，跟在类名之后。它指定了*类型参数（type parameter）*（也称*类型变量（type variable）*）T1、T2、……、Tn。

要将 Box 类更新为使用泛型，你需要创建一个*泛型类型声明*，将代码"public class Box"改为"public class Box<T>"。这引入了类型变量 T，它可以在类内部的任何地方使用。

经过此修改，Box 类变为：

```java
/**
 * Box 类的泛型版本。
 * @param <T> 被装箱值的类型
 */
public class Box<T> {
    // T 代表"Type"
    private T t;

    public void set(T t) { this.t = t; }
    public T get() { return t; }
}
```

如你所见，所有出现 Object 的地方都被替换为 T。类型变量可以是你指定的任何**非基本**类型：任何类类型、任何接口类型、任何数组类型，甚至另一个类型变量。

同样的技术也可用于创建泛型接口。

## 类型参数命名约定

按照惯例，类型参数名是单个的大写字母。这与你已经了解的变量[命名](https://docs.oracle.com/javase/tutorial/java/nutsandbolts/variables.html#naming)约定形成了鲜明对比，而这有充分的理由：如果没有这一约定，就很难区分类型变量与普通的类或接口名。

最常用的类型参数名有：

- E - Element（元素，被 Java 集合框架广泛使用）
- K - Key（键）
- N - Number（数字）
- T - Type（类型）
- V - Value（值）
- S、U、V 等 - 第 2、3、4 个类型

你将在整个 Java SE API 和本课程的其余部分看到这些名称的使用。

## 调用和实例化泛型类型

要在代码中引用泛型 Box 类，你必须执行一次*泛型类型调用（generic type invocation）*，用某个具体值（如 Integer）替换 T：

```java
Box<Integer> integerBox;
```

你可以把泛型类型调用想象为类似于普通的方法调用，但你不是向方法传递参数，而是向 Box 类本身传递一个*类型实参（type argument）*——在本例中是 Integer。

---

**类型参数与类型实参术语：** 许多开发者将"类型参数（type parameter）"和"类型实参（type argument）"这两个术语混用，但它们并不相同。在编码时，需要提供类型实参以创建参数化类型。因此，Foo<T> 中的 T 是类型参数，而 Foo<String> f 中的 String 是类型实参。本课程在使用这些术语时遵循此定义。

---

与任何其他变量声明一样，这段代码实际上并不会创建一个新的 Box 对象。它只是声明 integerBox 将持有一个指向"Box of Integer"的引用，这就是 Box<Integer> 的读法。

泛型类型的调用通常被称为*参数化类型（parameterized type）*。

要实例化此类，像往常一样使用 new 关键字，但将 <Integer> 放在类名和圆括号之间：

```java
Box<Integer> integerBox = new Box<Integer>();
```

## 菱形

在 Java SE 7 及更高版本中，只要编译器能够从上下文中确定（推断）类型实参，你就可以用一组空的类型实参（<>）来替换调用泛型类构造方法所需的类型实参。这对尖括号 <> 被非正式地称为*菱形（the diamond）*。例如，你可以用以下语句创建 Box<Integer> 的一个实例：

```java
Box<Integer> integerBox = new Box<>();
```

有关菱形记法和类型推断的更多信息，请参见[类型推断（Type Inference）](https://docs.oracle.com/javase/tutorial/java/generics/genTypeInference.html)。

## 多个类型参数

如前所述，泛型类可以有多个类型参数。例如，实现了泛型 Pair 接口的泛型 OrderedPair 类：

```java
public interface Pair<K, V> {
    public K getKey();
    public V getValue();
}

public class OrderedPair<K, V> implements Pair<K, V> {

    private K key;
    private V value;

    public OrderedPair(K key, V value) {
    this.key = key;
    this.value = value;
    }

    public K getKey()    { return key; }
    public V getValue() { return value; }
}
```

以下语句创建了 OrderedPair 类的两个实例化：

```java
Pair<String, Integer> p1 = new OrderedPair<String, Integer>("Even", 8);
Pair<String, String>  p2 = new OrderedPair<String, String>("hello", "world");
```

代码 new OrderedPair<String, Integer> 将 K 实例化为 String、V 实例化为 Integer。因此，OrderedPair 构造方法的参数类型分别是 String 和 Integer。由于[自动装箱（autoboxing）](https://docs.oracle.com/javase/tutorial/java/data/autoboxing.html)，向该类传入一个 String 和一个 int 是合法的。

如[菱形](#diamond)一节所述，由于 Java 编译器可以从声明 OrderedPair<String, Integer> 中推断出 K 和 V 的类型，因此这些语句可以使用菱形记法简写：

```java
OrderedPair<String, Integer> p1 = new OrderedPair<>("Even", 8);
OrderedPair<String, String>  p2 = new OrderedPair<>("hello", "world");
```

要创建泛型接口，遵循与创建泛型类相同的约定。

## 参数化类型

你也可以用参数化类型（即 List<String>）来替换类型参数（即 K 或 V）。例如，使用 OrderedPair<K, V> 示例：

```java
OrderedPair<String, Box<Integer>> p = new OrderedPair<>("primes", new Box<Integer>(...));
```
