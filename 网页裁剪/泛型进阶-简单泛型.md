## 定义简单泛型

以下是 `java.util` 包中 `List` 和 `Iterator` 接口定义的一小段摘录：

```java
public interface List <E> {
    void add(E x);
    Iterator<E> iterator();
}

public interface Iterator<E> {
    E next();
    boolean hasNext();
}
```

这段代码你应该都熟悉，除了尖括号中的内容。那些是 `List` 和 `Iterator` 接口的*形式类型参数(formal type parameters)*声明。

类型参数可以在整个泛型声明中使用，几乎在你使用普通类型的任何地方（尽管有一些重要的限制；参见[[泛型进阶-细节问题|细节问题]]一节）。

在简介中，我们看到了泛型类型声明 `List` 的*调用(invocations)*，例如 `List<Integer>`。在调用（通常称为*参数化类型(parameterized type)*）中，形式类型参数（本例中为 `E`）的所有出现都被*实际类型参数(actual type argument)*（本例中为 `Integer`）替换。

你可以想象 `List<Integer>` 代表 `List` 的一个版本，其中 `E` 被统一替换为 `Integer`：

```java
public interface IntegerList {
    void add(Integer x);
    Iterator<Integer> iterator();
}
```

这种直觉可能有用，但也具有误导性。

它有用，因为参数化类型 `List<Integer>` 确实具有看起来就像这种扩展的方法。

它具有误导性，因为泛型的声明从不会以这种方式实际扩展。代码没有多个副本——源码中没有、二进制中没有、磁盘上没有、内存中也没有。如果你是 C++ 程序员，你会理解这与 C++ 模板非常不同。

泛型类型声明一次性编译，并转换为单个类文件，就像普通的类或接口声明一样。

类型参数类似于方法或构造器中使用的普通参数。就像方法有描述其操作值类型的*形式值参数(formal value parameters)*一样，泛型声明有形式类型参数。当调用方法时，*实际参数(actual arguments)*替换形式参数，并评估方法体。当调用泛型声明时，实际类型参数替换形式类型参数。

关于命名约定的说明。我们建议你为形式类型参数使用简洁（如果可能单个字符）且引人联想的名称。最好避免在这些名称中使用小写字符，使形式类型参数与普通类和接口易于区分。许多容器类型使用 `E` 表示元素，如上面的示例。我们将在后面的示例中看到一些附加约定。
