## 什么是接口？

正如您已经学到的，对象通过其暴露的方法来定义与外部世界的交互。方法构成了对象与外部世界的*接口(interface)*；例如电视机正面的按钮就是您与塑料外壳背后电线之间的接口，您按下「电源」按钮来开关电视。

在最常见的形式中，接口是一组带有空方法体的相关方法。如果把自行车的行为指定为一个接口，它可能如下所示：

```typescript
interface Bicycle {

    //  车轮每分钟转数
    void changeCadence(int newValue);

    void changeGear(int newValue);

    void speedUp(int increment);

    void applyBrakes(int decrement);
}
```

要实现这个接口，您的类名会发生改变（例如改为某个特定品牌的自行车，如 `ACMEBicycle`），并在类声明中使用 `implements` 关键字：

```java
class ACMEBicycle implements Bicycle {

    int cadence = 0;
    int speed = 0;
    int gear = 1;

   // 编译器现在会要求方法
   // changeCadence、changeGear、speedUp 和 applyBrakes
   // 全部都被实现。如果该类缺少这些方法，
   // 编译将会失败。

    void changeCadence(int newValue) {
         cadence = newValue;
    }

    void changeGear(int newValue) {
         gear = newValue;
    }

    void speedUp(int increment) {
         speed = speed + increment;   
    }

    void applyBrakes(int decrement) {
         speed = speed - decrement;
    }

    void printStates() {
         System.out.println("cadence:" +
             cadence + " speed:" + 
             speed + " gear:" + gear);
    }
}
```

实现接口使类对其承诺提供的行为表达得更加正式。接口在类与外部世界之间形成一份契约，这份契约由编译器在构建时强制执行。如果您的类声称实现了某个接口，那么该接口定义的所有方法都必须出现在它的源代码中，类才能成功编译。

---

**注意：** 要真正编译 `ACMEBicycle` 类，您需要在所实现接口方法的开头添加 `public` 关键字。其中的原因将在后续关于[[类和对象|类和对象（Classes and Objects）]]和[[接口和继承|接口和继承（Interfaces and Inheritance）]]的课程中讲解。
