## 继承

在前面的课程中，你已经多次见到*继承(inheritance)*这一概念。在 Java 语言中，类可以从其他类*派生(derive)*而来，从而从那些类*继承(inherit)*字段和方法。

---

**定义：** 从另一个类派生而来的类称为*子类(subclass)*（也称为*派生类(derived class)*、*扩展类(extended class)*或*子类(child class)*）。子类所派生的那个类称为*超类(superclass)*（也称为*基类(base class)*或*父类(parent class)*）。

除 `Object` 外（它没有超类），每个类都有且只有一个直接超类（单继承）。在没有其他显式超类的情况下，每个类隐式地都是 `Object` 的子类。

类可以派生自一个类，那个类又派生自另一个类，那个类又派生自再一个类，依此类推，最终派生自最顶层的类 `Object`。这样的类被称为从一直追溯到 `Object` 的继承链中的所有类* descended*（繁衍而来）。

---

继承的思想简单而强大：当你想要创建一个新类，而已经存在一个包含了你所需部分代码的类时，你就可以从那个已有类派生你的新类。这样一来，你就可以复用已有类的字段和方法，而无需自己编写（和调试！）它们。

子类继承其超类的所有*成员(member)*（字段、方法和嵌套类）。构造方法不是成员，因此不会被子类继承，但可以在子类中调用超类的构造方法。

## Java 平台的类层次结构

定义在 `java.lang` 包中的 [`Object`](https://docs.oracle.com/javase/8/docs/api/java/lang/Object.html) 类，定义并实现了所有类（包括你编写的类）所共有的行为。在 Java 平台中，许多类直接派生自 `Object`，另一些类又派生自其中某些类，依此类推，形成了一个类的层次结构。

![[inheritance-classes-object.gif]]

Java 平台中的所有类都是 Object 的后代

在该层次结构的顶端，`Object` 是所有类中最通用的。靠近层次结构底部的类提供更为专门化的行为。

## 继承示例

下面是在「类和对象」课程中出现的 `Bicycle` 类的一个可能实现的示例代码：

```java
public class Bicycle {
        
    // Bicycle 类有三个字段
    public int cadence;
    public int gear;
    public int speed;
        
    // Bicycle 类有一个构造方法
    public Bicycle(int startCadence, int startSpeed, int startGear) {
        gear = startGear;
        cadence = startCadence;
        speed = startSpeed;
    }
        
    // Bicycle 类有四个方法
    public void setCadence(int newValue) {
        cadence = newValue;
    }
        
    public void setGear(int newValue) {
        gear = newValue;
    }
        
    public void applyBrake(int decrement) {
        speed -= decrement;
    }
        
    public void speedUp(int increment) {
        speed += increment;
    }
        
}
```

一个作为 `Bicycle` 子类的 `MountainBike` 类的类声明可能如下所示：

```java
public class MountainBike extends Bicycle {
        
    // MountainBike 子类新增了一个字段
    public int seatHeight;

    // MountainBike 子类有一个构造方法
    public MountainBike(int startHeight,
                        int startCadence,
                        int startSpeed,
                        int startGear) {
        super(startCadence, startSpeed, startGear);
        seatHeight = startHeight;
    }   
        
    // MountainBike 子类新增了一个方法
    public void setHeight(int newValue) {
        seatHeight = newValue;
    }   
}
```

`MountainBike` 继承了 `Bicycle` 的所有字段和方法，并新增了字段 `seatHeight` 和一个用于设置它的方法。除构造方法外，这就如同你完全从零开始编写了一个拥有四个字段和五个方法的新 `MountainBike` 类。然而，你无需做所有这些工作。如果 `Bicycle` 类中的方法很复杂、且调试耗费了大量时间，那么这一点将尤为有价值。

## 在子类中可以做什么

子类继承其父类的所有*public*（公共）和*protected*（受保护）成员，无论子类位于哪个包中。如果子类与其父类位于同一个包中，它还会继承父类的*包私有(package-private)*成员。你可以原样使用继承来的成员，也可以替换它们、隐藏它们，或者用新成员加以补充：

- 继承来的字段可以直接使用，就像任何其他字段一样。
- 你可以在子类中声明一个与超类中同名的字段，从而*隐藏(hiding)*它（不推荐这样做）。
- 你可以在子类中声明超类中没有的新字段。
- 继承来的方法可以原样使用。
- 你可以在子类中编写一个与超类中签名相同的新的*实例*方法，从而*重写(overriding)*它。
- 你可以在子类中编写一个与超类中签名相同的新的*静态*方法，从而*隐藏(hiding)*它。
- 你可以在子类中声明超类中没有的新方法。
- 你可以编写一个子类构造方法，隐式地或通过使用关键字 `super` 来调用超类的构造方法。

本课程后续的章节将展开讨论这些主题。

## 超类中的私有成员

子类不继承其父类的 `private`（私有）成员。然而，如果超类提供了用于访问其私有字段的 public 或 protected 方法，那么子类也可以使用这些方法。

嵌套类可以访问其外部类的所有私有成员——包括字段和方法。因此，被子类继承的 public 或 protected 嵌套类可以间接地访问超类的所有私有成员。

## 对象的类型转换

我们已经看到，一个对象的类型就是用于实例化它的那个类的数据类型。例如，如果我们编写

```text
public MountainBike myBike = new MountainBike();
```

那么 `myBike` 的类型就是 `MountainBike`。

`MountainBike` 派生自 `Bicycle` 和 `Object`。因此，一个 `MountainBike` 是一个 `Bicycle`，也是一个 `Object`，它可以在任何需要 `Bicycle` 或 `Object` 对象的地方使用。

反过来则不一定成立：一个 `Bicycle` *可能*是一个 `MountainBike`，但不一定。同样地，一个 `Object` *可能*是一个 `Bicycle` 或 `MountainBike`，但不一定。

*类型转换(casting)*展示的是在继承和实现所允许的对象之间，用一种类型的对象代替另一种类型对象的使用方式。例如，如果我们编写

```text
Object obj = new MountainBike();
```

那么 `obj` 既是一个 `Object`，也是一个 `MountainBike`（直到 `obj` 被赋予另一个*不是* `MountainBike` 的对象为止）。这被称为*隐式转换(implicit casting)*。

另一方面，如果我们编写

```text
MountainBike myBike = obj;
```

则会得到一个编译时错误，因为编译器并不知道 `obj` 是一个 `MountainBike`。然而，我们可以通过*显式转换(explicit casting)*来*告诉*编译器，我们承诺将一个 `MountainBike` 赋给 `obj`：

```text
MountainBike myBike = (MountainBike)obj;
```

这个转换会插入一个运行时检查，验证赋给 `obj` 的是一个 `MountainBike`，这样编译器就能安全地假定 `obj` 是一个 `MountainBike`。如果在运行时 `obj` 不是一个 `MountainBike`，将会抛出异常。

---

**注意：** 你可以使用 `instanceof` 运算符对某个特定对象的类型进行逻辑判断。这可以避免你因不当的转换而引发运行时错误。例如：

```text
if (obj instanceof MountainBike) {
    MountainBike myBike = (MountainBike)obj;
}
```

这里 `instanceof` 运算符验证了 `obj` 引用的是一个 `MountainBike`，从而让我们可以在确定不会抛出运行时异常的情况下进行转换。
