## 定义方法

下面是一个典型方法声明的示例：

```text
public double calculateAnswer(double wingSpan, int numberOfEngines,
                              double length, double grossTons) {
    //do the calculation here
}
```

方法声明中唯一必需的组成部分是方法的返回类型、名称、一对圆括号 `()`，以及花括号 `{}` 之间的方法体。

更一般地说，方法声明有六个组成部分，顺序如下：

1. 修饰符——如 `public`、`private`，以及稍后您将学到的其它修饰符。
2. 返回类型——方法所返回值的数据类型；如果方法不返回值，则为 `void`。
3. 方法名——字段名的命名规则同样适用于方法名，但约定略有不同。
4. 圆括号中的参数列表——一个由逗号分隔的输入参数列表，每个参数前有其数据类型，整体由圆括号 `()` 包围。如果没有参数，则必须使用空的圆括号。
5. 异常列表——将在稍后讨论。
6. 花括号之间的方法体——方法的代码，包括局部变量的声明，都写在这里。

修饰符、返回类型和参数将在本课稍后讨论。异常将在后续课程中讨论。

---

**定义：** 方法声明中有两个组成部分构成了*方法签名(method signature)*——方法名和参数类型。

---

上面声明的方法的签名为：

```text
calculateAnswer(double, int, double, double)
```

## 为方法命名

尽管方法名可以是任何合法的标识符，但代码约定对方法名有所限制。按照约定，方法名应当是一个小写的动词，或者一个以小写动词开头、后接形容词、名词等的多词名称。在多词名称中，第二个及后续各词的首字母应当大写。下面是一些示例：

```text
run
runFast
getBackground
getFinalData
compareTo
setX
isEmpty
```

通常，一个方法在其所在的类中拥有唯一的名称。然而，由于*方法重载(method overloading)*，一个方法可能与其它方法同名。

## 重载方法

Java 编程语言支持*重载(overload)*方法，Java 能够区分具有不同*方法签名*的方法。这意味着，如果一个类中的各个方法拥有不同的参数列表，它们就可以同名（此规则有一些限定条件，将在题为「接口和继承」的课程中讨论）。

假设您有一个类，它能够使用书法绘制各种类型的数据（字符串、整数等等），并且为每种数据类型都包含一个绘制方法。为每个方法使用新的名称（例如 `drawString`、`drawInteger`、`drawFloat` 等等）会很繁琐。在 Java 编程语言中，您可以为所有绘制方法使用相同的名称，但为每个方法传递不同的参数列表。因此，这个数据绘制类可能会声明四个名为 `draw` 的方法，每个方法都有不同的参数列表。

```java
public class DataArtist {
    ...
    public void draw(String s) {
        ...
    }
    public void draw(int i) {
        ...
    }
    public void draw(double f) {
        ...
    }
    public void draw(int i, double f) {
        ...
    }
}
```

重载方法通过传入方法的参数的数量和类型来区分。在代码示例中，`draw(String s)` 和 `draw(int i)` 是不同且唯一的方法，因为它们要求不同的参数类型。

您不能声明多个具有相同名称、相同数量和类型参数的方法，因为编译器无法区分它们。

编译器在区分方法时不考虑返回类型，因此即使两个方法有不同的返回类型，您也不能声明两个具有相同签名的方法。

---

**注意：** 应当谨慎使用重载方法，因为它们可能使代码大大降低可读性。
