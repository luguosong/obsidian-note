## 简介

*可扩展应用程序*是指你可以在不修改其原始代码库的情况下扩展的应用程序。你可以使用新的插件或模块增强其功能。开发者、软件供应商和客户可以通过在应用程序类路径上或应用程序特定的扩展目录中添加新的 Java 归档(JAR) 文件来添加新功能或应用程序编程接口(API)。

本节介绍如何创建具有可扩展服务的应用程序，这些服务使你或其他人能够提供无需修改原始应用程序的服务实现。通过设计可扩展应用程序，你提供了一种在不更改核心应用程序的情况下升级或增强产品特定部分的方法。

可扩展应用程序的一个示例是允许最终用户添加新字典或拼写检查器的文字处理器。在此示例中，文字处理器提供字典或拼写功能，其他开发者甚至客户可以通过提供自己的功能实现来扩展。

以下是理解可扩展应用程序的重要术语和定义：

服务

一组编程接口和类，提供对某些特定应用程序功能或特性的访问。服务可以定义功能的接口和检索实现的方式。在文字处理器示例中，字典服务可以定义检索字典和单词定义的方式，但它不实现底层功能集。相反，它依赖于*服务提供者*来实现该功能。

服务提供者接口(SPI)

服务定义的公共接口和抽象类的集合。SPI 定义了应用程序可用的类和方法。

服务提供者

实现 SPI。具有可扩展服务的应用程序使你、供应商和客户能够在不修改原始应用程序的情况下添加服务提供者。

## 理解 DictionaryServiceDemo 示例

以下步骤向你展示如何重新创建文件 `DictionaryServiceDemo.zip` 的内容。这些步骤向你展示示例的工作原理以及如何运行它。

### 1\. 定义服务提供者接口

`DictionaryServiceDemo` 示例定义一个 SPI，即 [`` `Dictionary.java` ``](https://docs.oracle.com/javase/tutorial/ext/basics/examples/DictionaryServiceDemo/DictionaryServiceProvider/src/dictionary/spi/Dictionary.java) 接口。它只包含一个方法：

```java
package dictionary.spi;

public interface Dictionary {
    public String getDefinition(String word);
}
```

示例将编译的类文件存储在目录 `DictionaryServiceProvider/build` 中。

### 2\. 定义检索服务提供者实现的服务

[`` `DictionaryService.java` ``](https://docs.oracle.com/javase/tutorial/ext/basics/examples/DictionaryServiceDemo/DictionaryServiceProvider/src/dictionary/DictionaryService.java) 类代表字典服务客户端加载并访问可用的 `Dictionary` 服务提供者：

```java
package dictionary;

import dictionary.spi.Dictionary;
import java.util.Iterator;
import java.util.ServiceConfigurationError;
import java.util.ServiceLoader;

public class DictionaryService {

    private static DictionaryService service;
    private ServiceLoader<Dictionary> loader;

    private DictionaryService() {
        loader = ServiceLoader.load(Dictionary.class);
    }

    public static synchronized DictionaryService getInstance() {
        if (service == null) {
            service = new DictionaryService();
        }
        return service;
    }

    public String getDefinition(String word) {
        String definition = null;

        try {
            Iterator<Dictionary> dictionaries = loader.iterator();
            while (definition == null && dictionaries.hasNext()) {
                Dictionary d = dictionaries.next();
                definition = d.getDefinition(word);
            }
        } catch (ServiceConfigurationError serviceError) {
            definition = null;
            serviceError.printStackTrace();

        }
        return definition;
    }
}
```

示例将编译的类文件存储在目录 `DictionaryServiceProvider/build` 中。

`DictionaryService` 类实现了单例设计模式。这意味着只创建 `DictionaryService` 类的单个实例。有关更多信息，请参阅[单例设计模式](#singleton)节。

`DictionaryService` 类是字典服务客户端使用任何已安装 `Dictionary` 服务提供者的入口点。使用 [`ServiceLoader.load`](https://docs.oracle.com/javase/8/docs/api/java/util/ServiceLoader.html#load-java.lang.Class-) 方法检索私有静态成员 `DictionaryService.service`（单例服务入口点）。然后应用程序可以调用 `getDefinition` 方法，该方法遍历可用的 `Dictionary` 提供者，直到找到目标单词。如果没有 `Dictionary` 实例包含该单词的指定定义，`getDefinition` 方法返回 null。

字典服务使用 `ServiceLoader.load` 方法查找目标类。SPI 由接口 `dictionary.spi.Dictionary` 定义，因此示例使用此类作为 load 方法的参数。默认 load 方法使用默认类加载器搜索应用程序类路径。

但是，此方法的重载版本允许你根据需要指定自定义类加载器。这使你能够进行更复杂的类搜索。例如，一个特别热心的程序员可能创建一个 [`ClassLoader`](https://docs.oracle.com/javase/8/docs/api/java/lang/ClassLoader.html) 实例，可以在运行时添加的包含提供者 JAR 的应用程序特定子目录中搜索。结果是一个不需要重启即可访问新提供者类的应用程序。

此类加载器存在后，你可以使用其 iterator 方法访问并使用它找到的每个提供者。`getDefinition` 方法使用 `Dictionary` 迭代器遍历提供者，直到找到指定单词的定义。iterator 方法缓存 `Dictionary` 实例，因此连续调用几乎不需要额外的处理时间。如果自上次调用以来已将新提供者投入服务，则 iterator 方法将它们添加到列表中。

[`` `DictionaryDemo.java` ``](https://docs.oracle.com/javase/tutorial/ext/basics/examples/DictionaryServiceDemo/DictionaryDemo/src/dictionary/DictionaryDemo.java) 类使用此服务。要使用该服务，应用程序获取 `DictionaryService` 实例并调用 `getDefinition` 方法。如果定义可用，应用程序打印它。如果定义不可用，应用程序打印一条消息说明没有可用的字典包含该单词。

#### 单例设计模式

设计模式是软件设计中常见问题的一般解决方案。其思想是将解决方案转换为代码，并且该代码可以应用于问题出现的不同情况。单例模式描述了一种确保只创建类的单个实例的技术。本质上，该技术采用以下方法：不让类外的任何人创建对象的实例。

例如，[`` `DictionaryService` ``](https://docs.oracle.com/javase/tutorial/ext/basics/examples/DictionaryServiceDemo/DictionaryServiceProvider/src/dictionary/DictionaryService.java) 类如下实现单例模式：

- 将 `DictionaryService` 构造函数声明为 `private`，这阻止除 `DictionaryService` 之外的所有其他类创建其实例。
- 将 `DictionaryService` 成员变量 `service` 定义为 `static`，这确保只存在一个 `DictionaryService` 实例。
- 定义方法 `getInstance`，它使其他类能够受控地访问 `DictionaryService` 成员变量 `service`。

### 3\. 实现服务提供者

要提供此服务，你必须创建 [`` `Dictionary.java` ``](https://docs.oracle.com/javase/tutorial/ext/basics/examples/DictionaryServiceDemo/DictionaryServiceProvider/src/dictionary/spi/Dictionary.java) 实现。为简单起见，创建一个只定义几个单词的通用字典。你可以使用数据库、一组属性文件或任何其他技术实现字典。演示提供者模式的最简单方法是将所有单词和定义包含在单个文件中。

以下代码显示了 `Dictionary` SPI 的实现，即 [`` `GeneralDictionary.java` ``](https://docs.oracle.com/javase/tutorial/ext/basics/examples/DictionaryServiceDemo/GeneralDictionary/src/dictionary/GeneralDictionary.java) 类。注意它提供了无参数构造函数并实现了 SPI 定义的 `getDefinition` 方法。

```java
package dictionary;

import dictionary.spi.Dictionary;
import java.util.SortedMap;
import java.util.TreeMap;

public class GeneralDictionary implements Dictionary {

    private SortedMap<String, String> map;

    public GeneralDictionary() {
        map = new TreeMap<String, String>();
        map.put(
            "book",
            "a set of written or printed pages, usually bound with " +
                "a protective cover");
        map.put(
            "editor",
            "a person who edits");
    }

    @Override
    public String getDefinition(String word) {
        return map.get(word);
    }

}
```

示例将编译的类文件存储在目录 `GeneralDictionary/build` 中。**注意**：你必须在类 `GeneralDictionary` 之前编译类 `dictionary.DictionaryService` 和 `dictionary.spi.Dictionary`。

此示例的 `GeneralDictionary` 提供者只定义了两个单词：*book* 和 *editor*。显然，更有用的字典会提供更大量的常用词汇。

为了演示多个提供者如何实现相同的 SPI，以下代码显示了另一个可能的提供者。[`` `ExtendedDictionary.java` ``](https://docs.oracle.com/javase/tutorial/ext/basics/examples/DictionaryServiceDemo/ExtendedDictionary/src/dictionary/ExtendedDictionary.java) 服务提供者是一个扩展字典，包含大多数软件开发者熟悉的技术术语。

```java
import java.util.SortedMap;
import java.util.TreeMap;

public class ExtendedDictionary implements Dictionary {

                "and deleting data that attempts to use the common " +
                "vocabulary of the HTTP protocol; Representational State " +

```

很容易想象客户为其自身特殊需求使用一整套 `Dictionary` 提供者。服务加载器 API 使他们能够根据需求或偏好的变化向应用程序添加新字典。由于底层文字处理器应用程序是可扩展的，客户使用新提供者无需额外的编码。

### 4\. 注册服务提供者

要注册你的服务提供者，你创建一个提供者配置文件，该文件存储在服务提供者 JAR 文件的 `META-INF/services` 目录中。配置文件的名称是服务提供者的完全限定类名，其中名称的每个组件用句点(`.`) 分隔，嵌套类用美元符号(`$`) 分隔。

提供者配置文件包含服务提供者的完全限定类名，每行一个名称。文件必须是 UTF-8 编码的。此外，你可以通过以数字符号(`#`) 开始注释行在文件中包含注释。

例如，要注册服务提供者 `GeneralDictionary`，创建一个名为 [`` `dictionary.spi.Dictionary` ``](https://docs.oracle.com/javase/tutorial/ext/basics/examples/DictionaryServiceDemo/GeneralDictionary/src/META-INF/services/dictionary.spi.Dictionary) 的文本文件。此文件包含一行：

```text
dictionary.GeneralDictionary
```

类似地，要注册服务提供者 `ExtendedDictionary`，创建一个名为 [`` `dictionary.spi.Dictionary` ``](https://docs.oracle.com/javase/tutorial/ext/basics/examples/DictionaryServiceDemo/ExtendedDictionary/src/META-INF/services/dictionary.spi.Dictionary) 的文本文件。此文件包含一行：

```text

```

### 5\. 创建使用服务和服务提供者的客户端

由于开发完整的文字处理器应用程序是一项重大工作，本教程提供了一个使用 `DictionaryService` 和 `Dictionary` SPI 的更简单的应用程序。`DictionaryDemo` 示例从类路径上的任何 `Dictionary` 提供者搜索 *book*、*editor*、*xml* 和 *REST* 单词，并检索它们的定义。

以下是 [`` `DictionaryDemo` ``](https://docs.oracle.com/javase/tutorial/ext/basics/examples/DictionaryServiceDemo/DictionaryDemo/src/dictionary/DictionaryDemo.java) 示例。它从 `DictionaryService` 实例请求目标单词的定义，该实例将请求传递给其已知的 `Dictionary` 提供者。

```java
package dictionary;

import dictionary.DictionaryService;

public class DictionaryDemo {

  public static void main(String[] args) {

    DictionaryService dictionary = DictionaryService.getInstance();
    System.out.println(DictionaryDemo.lookup(dictionary, "book"));
    System.out.println(DictionaryDemo.lookup(dictionary, "editor"));
    System.out.println(DictionaryDemo.lookup(dictionary, "xml"));
    System.out.println(DictionaryDemo.lookup(dictionary, "REST"));
  }

  public static String lookup(DictionaryService dictionary, String word) {
    String outputString = word + ": ";
    String definition = dictionary.getDefinition(word);
    if (definition == null) {
      return outputString + "Cannot find definition for this word.";
    } else {
      return outputString + definition;
    }
  }
}
```

示例将编译的类文件存储在目录 `DictionaryDemo/build` 中。**注意**：你必须在类 `DictionaryDemo` 之前编译类 `dictionary.DictionaryService` 和 `dictionary.spi.Dictionary`。

## ServiceLoader 类

`java.util.ServiceLoader` 类帮助你查找、加载和使用服务提供者。它在应用程序的类路径或运行时环境的扩展目录中搜索服务提供者。它加载它们并使你的应用程序能够使用提供者的 API。如果你向类路径或运行时扩展目录添加新提供者，`ServiceLoader` 类会找到它们。如果你的应用程序知道提供者接口，它可以找到并使用该接口的不同实现。你可以使用该接口的第一个可加载实例或遍历所有可用接口。

`ServiceLoader` 类是 final 的，这意味着你不能将其设为子类或重写其加载算法。例如，你不能更改其算法以从不同位置搜索服务。

从 `ServiceLoader` 类的角度来看，所有服务都有单一类型，通常是单个接口或抽象类。提供者本身包含一个或多个具体类，这些类用特定于其目的的实现扩展服务类型。`ServiceLoader` 类要求单个公开的提供者类型具有默认构造函数，该构造函数不需要参数。这使 `ServiceLoader` 类能够轻松实例化它找到的服务提供者。

提供者按需定位和实例化。服务加载器维护已加载提供者的缓存。每次调用加载器的 `iterator` 方法都会返回一个迭代器，该迭代器首先按实例化顺序产生缓存的所有元素。然后服务加载器定位并实例化任何新提供者，依次将每个添加到缓存中。你可以使用 `reload` 方法清除提供者缓存。

要为特定类创建加载器，将该类本身提供给 `load` 或 `loadInstalled` 方法。你可以使用默认类加载器或提供你自己的 `ClassLoader` 子类。

`loadInstalled` 方法搜索已安装运行时提供者的运行时环境扩展目录。默认扩展位置是运行时环境的 `jre/lib/ext` 目录。你应该仅对已知的、受信任的提供者使用扩展位置，因为此位置成为所有应用程序的类路径的一部分。在本文中，提供者不使用扩展目录，而是依赖于应用程序特定的类路径。

## ServiceLoader API 的局限性

`ServiceLoader` API 很有用，但它有局限性。例如，无法从 `ServiceLoader` 类派生类，因此你无法修改其行为。你可以使用自定义 `ClassLoader` 子类更改类的查找方式，但 `ServiceLoader` 本身无法扩展。此外，当前的 `ServiceLoader` 类无法在运行时新提供者可用时通知你的应用程序。此外，你无法向加载器添加更改侦听器以了解是否将新提供者放入应用程序特定的扩展目录。

公共 `ServiceLoader` API 在 Java SE 6 中可用。虽然加载器服务早在 JDK 1.3 就存在，但该 API 是私有的，仅对内部 Java 运行时代码可用。

## 总结

可扩展应用程序提供服务点，可由服务提供者扩展。创建可扩展应用程序的最简单方法是使用 `ServiceLoader`，它适用于 Java SE 6 及更高版本。使用此类，你可以将提供者实现添加到应用程序类路径以提供新功能。`ServiceLoader` 类是 final 的，因此你无法修改其能力。
