## 学习路径：自定义网络

Java 平台之所以备受推崇，部分原因在于它非常适合编写使用互联网和万维网资源并与之交互的程序。事实上，兼容 Java 的浏览器将 Java 平台的这种能力发挥到极致，通过互联网传输并运行 applet。

本学习路径带您逐步了解编写可在互联网上使用的 Java 应用程序和 applet 的种种复杂之处。

[**网络概述（Overview of Networking）**](https://docs.oracle.com/javase/tutorial/networking/overview/index.html) 分为两部分。第一部分描述您可能已经在使用、却未意识到自己在使用网络的 Java 平台网络功能。第二部分提供网络的简要概述，让您熟悉在阅读如何使用 URL、套接字和数据报之前应当理解的术语和概念。

[**使用 URL（Working With URLs）**](https://docs.oracle.com/javase/tutorial/networking/urls/index.html) 讨论 Java 程序如何使用 URL 访问互联网上的信息。URL（统一资源定位符，Uniform Resource Locator）是互联网上某个资源的地址。Java 程序可以使用 URL 连接网络并检索信息。本课程提供 URL 的更完整定义，并展示如何创建和解析 URL、如何打开到 URL 的连接，以及如何对该连接进行读写。

[**关于套接字（All About Sockets）**](https://docs.oracle.com/javase/tutorial/networking/sockets/index.html) 解释如何使用套接字使您的程序能与网络上的其它程序通信。套接字是网络上两个运行程序之间双向通信链路的一个端点。本课程展示客户端如何连接到标准服务器（Echo 服务器）并通过套接字与之通信。然后，它带您逐步了解一个完整的客户端/服务器示例的细节，展示如何实现客户端/服务器对的客户端和服务端。

[**关于数据报（All About Datagrams）**](https://docs.oracle.com/javase/tutorial/networking/datagrams/index.html) 逐步带您了解一个使用数据报进行通信的简单客户端/服务器示例，然后挑战您改用多播套接字（multicast socket）重写该示例。

[**以编程方式访问网络参数（Programmatic Access to Network Parameters）**](https://docs.oracle.com/javase/tutorial/networking/nifs/index.html) 解释您可能需要访问网络接口参数的原因以及如何操作。它给出了如何列出分配给机器的所有 IP 地址及其它有用信息（如该接口是否正在运行）的示例。

[**使用 cookie（Working With Cookies）**](https://docs.oracle.com/javase/tutorial/networking/cookies/index.html) 讨论 cookie 如何用于在客户端和服务器之间创建会话，以及您如何在 HTTP URL 连接中利用 cookie。

**安全考量：**

请注意，网络上的通信受当前安全管理器的批准。[安全管理器（The Security Manager）](https://docs.oracle.com/javase/tutorial/essential/environment/security.html) 描述了什么是安全管理器以及它如何影响您的应用程序。如需了解 JDK 提供的安全特性的总体信息，请参阅 [Java SE 的安全特性（Security Features in Java SE）](https://docs.oracle.com/javase/tutorial/security/index.html)。

以下涵盖 URL、套接字和数据报的课程中，示例程序都是独立应用程序，默认情况下没有安全管理器。如果您将这些应用程序转换为 applet，它们可能无法通过网络进行通信，具体取决于运行它们的浏览器或查看器。如需了解对 applet 施加的安全限制，请参见 [applet 能做什么和不能做什么（What Applets Can and Cannot Do）](https://docs.oracle.com/javase/tutorial/deployment/applet/security.html)。
