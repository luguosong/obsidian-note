## 嵌套类

Java 编程语言允许您在一个类中定义另一个类。这样的类被称为*嵌套类(nested class)*，其形式如下所示：

```text
class OuterClass {
    ...
    class NestedClass {
        ...
    }
}
```

---

**术语：** 嵌套类分为两类：非静态(non-static)的和静态(static)的。非静态嵌套类被称为*内部类(inner class)*。声明为 `static` 的嵌套类被称为*静态嵌套类(static nested class)*。

---

```text
class OuterClass {
    ...
    class InnerClass {
        ...
    }
    static class StaticNestedClass {
        ...
    }
}
```

嵌套类是其外部类( enclosing class)的成员。非静态嵌套类（内部类）可以访问外部类的其它成员，即使那些成员被声明为 private。静态嵌套类不能访问外部类的其它成员。作为 `OuterClass` 的成员，嵌套类可以被声明为 `private`、`public`、`protected` 或*包级私有(package private)*。（回想一下，外部类只能被声明为 `public` 或*包级私有*。）

## 为什么要使用嵌套类？

使用嵌套类的令人信服的理由包括以下几点：

- **它是一种对只在一处使用的类进行逻辑分组的方法**：如果一个类只对另一个类有用，那么将其嵌入到那个类中并把两者放在一起就是合乎逻辑的。嵌套这样的「辅助类(helper class)」可以使它们所在的包更加精简。
- **它增强了封装性**：考虑两个顶层类 A 和 B，其中 B 需要访问 A 的成员，而这些成员原本会被声明为 `private`。通过将类 B 隐藏在类 A 内部，A 的成员可以被声明为 private，而 B 仍然可以访问它们。此外，B 本身也可以对外部世界隐藏起来。
- **它可以带来更易读、更易维护的代码**：将小类嵌套在顶层类中，可以把代码放到离使用它的地方更近的位置。

## 内部类

与实例方法和实例变量一样，内部类与其外部类的实例相关联，并可以直接访问该对象的方法和字段。而且，由于内部类与一个实例相关联，它本身不能定义任何静态成员。

作为内部类实例的对象存在于外部类实例*之内*。考虑下面的类：

```text
class OuterClass {
    ...
    class InnerClass {
        ...
    }
}
```

`InnerClass` 的实例只能存在于 `OuterClass` 的实例之内，并且可以直接访问其外部实例的方法和字段。

要实例化一个内部类，您必须首先实例化外部类。然后，使用以下语法在外部对象内创建内部对象：

```text
OuterClass outerObject = new OuterClass();
OuterClass.InnerClass innerObject = outerObject.new InnerClass();
```

内部类有两种特殊的类别：局部类(local class)和匿名类(anonymous class)。

## 静态嵌套类

与类方法和类变量一样，静态嵌套类与其外部类相关联。与静态类方法类似，静态嵌套类不能直接引用其外部类中定义的实例变量或方法：它只能通过对象引用来使用它们。[内部类与静态嵌套类示例](#内部类与静态嵌套类示例)演示了这一点。

---

**注意：** 静态嵌套类与其外部类的实例成员（及其它类）交互的方式，与任何其它顶层类完全相同。实际上，静态嵌套类在行为上就是一个为了打包方便而嵌套在另一个顶层类中的顶层类。[内部类与静态嵌套类示例](#内部类与静态嵌套类示例)也演示了这一点。

---

实例化静态嵌套类的方式与顶层类相同：

```text
StaticNestedClass staticNestedObject = new StaticNestedClass();
```

## 内部类与静态嵌套类示例

下面的示例 [`OuterClass`](https://docs.oracle.com/javase/tutorial/java/javaOO/examples/OuterClass.java) 以及 [`TopLevelClass`](https://docs.oracle.com/javase/tutorial/java/javaOO/examples/TopLevelClass.java) 演示了 `OuterClass` 的哪些类成员可以被内部类（`InnerClass`）、静态嵌套类（`StaticNestedClass`）和顶层类（`TopLevelClass`）访问：

### OuterClass.java

```java
public class OuterClass {

    String outerField = "Outer field";
    static String staticOuterField = "Static outer field";

    class InnerClass {
        void accessMembers() {
            System.out.println(outerField);
            System.out.println(staticOuterField);
        }
    }

    static class StaticNestedClass {
        void accessMembers(OuterClass outer) {
            // Compiler error: Cannot make a static reference to the non-static
            //     field outerField
            // System.out.println(outerField);
            System.out.println(outer.outerField);
            System.out.println(staticOuterField);
        }
    }

    public static void main(String[] args) {
        System.out.println("Inner class:");
        System.out.println("------------");
        OuterClass outerObject = new OuterClass();
        OuterClass.InnerClass innerObject = outerObject.new InnerClass();
        innerObject.accessMembers();

        System.out.println("\nStatic nested class:");
        System.out.println("--------------------");
        StaticNestedClass staticNestedObject = new StaticNestedClass();        
        staticNestedObject.accessMembers(outerObject);
        
        System.out.println("\nTop-level class:");
        System.out.println("--------------------");
        TopLevelClass topLevelObject = new TopLevelClass();        
        topLevelObject.accessMembers(outerObject);                
    }
}
```

### TopLevelClass.java

```java
public class TopLevelClass {

    void accessMembers(OuterClass outer) {     
        // Compiler error: Cannot make a static reference to the non-static
        //     field OuterClass.outerField
        // System.out.println(OuterClass.outerField);
        System.out.println(outer.outerField);
        System.out.println(OuterClass.staticOuterField);
    }  
}
```

此示例打印如下输出：

```text
Inner class:
------------
Outer field
Static outer field

Static nested class:
--------------------
Outer field
Static outer field

Top-level class:
--------------------
Outer field
Static outer field
```

注意，静态嵌套类与其外部类的实例成员交互的方式与任何其它顶层类完全相同。静态嵌套类 `StaticNestedClass` 无法直接访问 `outerField`，因为它是外部类 `OuterClass` 的实例变量。Java 编译器在突出显示的语句处会生成错误：

```java
static class StaticNestedClass {
    void accessMembers(OuterClass outer) {
       // Compiler error: Cannot make a static reference to the non-static
       //     field outerField
       System.out.println(outerField);
    }
}
```

要修复此错误，请通过对象引用访问 `outerField`：

```java
System.out.println(outer.outerField);
```

同样，顶层类 `TopLevelClass` 也无法直接访问 `outerField`。

## 遮蔽

如果某个特定作用域（例如内部类或方法定义）中的一种声明（如成员变量或参数名）与外部作用域中的另一个声明同名，那么该声明就会*遮蔽(shadow)*外部作用域中的声明。此时，您不能仅凭名称来引用被遮蔽的声明。下面的示例 [`ShadowTest`](https://docs.oracle.com/javase/tutorial/java/javaOO/examples/ShadowTest.java) 演示了这一点：

```java
public class ShadowTest {

    public int x = 0;

    class FirstLevel {

        public int x = 1;

        void methodInFirstLevel(int x) {
            System.out.println("x = " + x);
            System.out.println("this.x = " + this.x);
            System.out.println("ShadowTest.this.x = " + ShadowTest.this.x);
        }
    }

    public static void main(String... args) {
        ShadowTest st = new ShadowTest();
        ShadowTest.FirstLevel fl = st.new FirstLevel();
        fl.methodInFirstLevel(23);
    }
}
```

下面是这个示例的输出：

```properties
x = 23
this.x = 1
ShadowTest.this.x = 0
```

此示例定义了三个名为 `x` 的变量：`ShadowTest` 类的成员变量、内部类 `FirstLevel` 的成员变量，以及 `methodInFirstLevel` 方法中的参数。作为 `methodInFirstLevel` 方法定义的参数 `x` 遮蔽了内部类 `FirstLevel` 的变量。因此，当您在方法 `methodInFirstLevel` 中使用变量 `x` 时，它指向的是方法参数。要引用内部类 `FirstLevel` 的成员变量，请使用关键字 `this` 来代表其所在的作用域：

```java
System.out.println("this.x = " + this.x);
```

对于外层更大作用域中的成员变量，通过它们所属的类名来引用。例如，下面的语句从方法 `methodInFirstLevel` 中访问 `ShadowTest` 类的成员变量：

```java
System.out.println("ShadowTest.this.x = " + ShadowTest.this.x);
```

## 序列化

强烈不建议对内部类（包括局部类和匿名类）进行[序列化(Serialization)](https://docs.oracle.com/javase/tutorial/jndi/objects/serial.html)。当 Java 编译器编译某些构造（例如内部类）时，它会创建*合成构造(synthetic construct)*；这些是指在源代码中没有对应构造的类、方法、字段和其他构造。合成构造使得 Java 编译器能够在不更改 JVM 的情况下实现新的 Java 语言特性。然而，合成构造在不同的 Java 编译器实现中可能有所不同，这意味着 `.class` 文件在不同实现之间也可能不同。因此，如果您序列化一个内部类，然后用不同的 JRE 实现来反序列化它，可能会遇到兼容性问题。有关编译内部类时生成的合成构造的更多信息，请参见 [获取方法参数的名称（Obtaining Names of Method Parameters）](https://docs.oracle.com/javase/tutorial/reflect/member/methodparameterreflection.html)一节中的 [隐式参数与合成参数（Implicit and Synthetic Parameters）](https://docs.oracle.com/javase/tutorial/reflect/member/methodparameterreflection.html#implcit_and_synthetic)部分。
