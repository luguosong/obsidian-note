## 创建对象

如你所知，类为对象提供了蓝图；您从类创建对象。下面这些取自 [`CreateObjectDemo`](https://docs.oracle.com/javase/tutorial/java/javaOO/examples/CreateObjectDemo.java) 程序的语句，每一条都创建一个对象并将其赋值给一个变量：

```text
Point originOne = new Point(23, 94);
Rectangle rectOne = new Rectangle(originOne, 100, 200);
Rectangle rectTwo = new Rectangle(50, 100);
```

第一行创建了一个 [`Point`](https://docs.oracle.com/javase/tutorial/java/javaOO/examples/Point.java) 类的对象，第二行和第三行各创建了一个 [`Rectangle`](https://docs.oracle.com/javase/tutorial/java/javaOO/examples/Rectangle.java) 类的对象。

这些语句中的每一条都包含三个部分（下文详细讨论）：

1. **声明**：以**粗体**显示的代码都是变量声明，它们将一个变量名与一个对象类型关联起来。
2. **实例化**：`new` 关键字是一个 Java 运算符，用于创建对象。
3. **初始化**：`new` 运算符之后跟着对一个构造方法的调用，由该构造方法初始化新对象。

## 声明一个引用对象的变量

前面您已学到，要声明一个变量，您编写：

```text
type name;
```

这会通知编译器，您将使用 *name* 来引用其类型为 *type* 的数据。对于基本类型变量，此声明还会为该变量预留适当大小的内存。

您也可以单独在一行声明一个引用变量。例如：

```text
Point originOne;
```

如果您像这样声明 `originOne`，它的值将是不确定的，直到实际创建一个对象并将其赋给它。仅仅声明一个引用变量并不会创建对象。为此，您需要使用 `new` 运算符，如下一节所述。在代码中使用 `originOne` 之前，您必须为其赋一个对象。否则，您会得到一个编译器错误。

处于这种状态（当前未引用任何对象）的变量，可以用下图说明（变量名 `originOne`，加上一个指向空无一物的引用）：

## 实例化类

`new` 运算符通过为新对象分配内存并返回指向该内存的引用来实例化一个类。`new` 运算符还会调用对象的构造方法。

---

**注意：** 「实例化一个类(instantiating a class)」这个短语与「创建一个对象(creating an object)」含义相同。当您创建一个对象时，您就在创建该类的一个「实例(instance)」，因此也就是在「实例化」一个类。

---

`new` 运算符需要一个单一的后缀参数：对一个构造方法的调用。构造方法的名称提供了要实例化的类的名称。

`new` 运算符返回一个指向它所创建对象的引用。这个引用通常被赋给一个适当类型的变量，例如：

```text
Point originOne = new Point(23, 94);
```

`new` 运算符返回的引用不必赋给一个变量。它也可以在表达式中直接使用。例如：

```java
int height = new Rectangle().height;
```

这条语句将在下一节讨论。

## 初始化对象

下面是 Point 类的代码：

```java
public class Point {
    public int x = 0;
    public int y = 0;
    //constructor
    public Point(int a, int b) {
        x = a;
        y = b;
    }
}
```

这个类包含一个构造方法。您可以通过其声明使用与类相同的名称且没有返回类型来识别构造方法。Point 类中的构造方法接受两个整数参数，正如代码 `(int a, int b)` 所声明的那样。下面的语句为这两个参数提供 23 和 94 作为值：

```text
Point originOne = new Point(23, 94);
```

执行此语句的结果可以用下图说明：

![[objectcreation-objects-oneRef.gif]]

下面是 Rectangle 类的代码，它包含四个构造方法：

```java
public class Rectangle {
    public int width = 0;
    public int height = 0;
    public Point origin;

    // four constructors
    public Rectangle() {
        origin = new Point(0, 0);
    }
    public Rectangle(Point p) {
        origin = p;
    }
    public Rectangle(int w, int h) {
        origin = new Point(0, 0);
        width = w;
        height = h;
    }
    public Rectangle(Point p, int w, int h) {
        origin = p;
        width = w;
        height = h;
    }

    // a method for moving the rectangle
    public void move(int x, int y) {
        origin.x = x;
        origin.y = y;
    }

    // a method for computing the area of the rectangle
    public int getArea() {
        return width * height;
    }
}
```

每个构造方法都允许您使用基本类型和引用类型，为矩形的原点、宽度和高度提供初始值。如果一个类有多个构造方法，它们必须具有不同的签名。Java 编译器根据参数的数量和类型来区分这些构造方法。当 Java 编译器遇到下面的代码时，它知道要调用 Rectangle 类中那个需要一个 Point 参数后跟两个整数参数的构造方法：

```text
Rectangle rectOne = new Rectangle(originOne, 100, 200);
```

这调用了 `Rectangle` 的某个构造方法，将 `origin` 初始化为 `originOne`。此外，该构造方法将 `width` 设置为 100，将 `height` 设置为 200。现在有两个引用指向同一个 Point 对象——一个对象可以被多个引用指向，如下图所示：

![[objectcreation-objects-multipleRefs.gif]]

下面这行代码调用了需要两个整数参数的 `Rectangle` 构造方法，这两个参数为宽度和高度提供初始值。如果您查看构造方法内部的代码，会看到它创建了一个新的 Point 对象，其 x 和 y 值被初始化为 0：

```text
Rectangle rectTwo = new Rectangle(50, 100);
```

下面这条语句中使用的 Rectangle 构造方法不接受任何参数，因此它被称为*无参构造方法(no-argument constructor)*：

```text
Rectangle rect = new Rectangle();
```

所有类都至少有一个构造方法。如果一个类没有显式声明任何构造方法，Java 编译器会自动提供一个无参构造方法，称为*默认构造方法(default constructor)*。这个默认构造方法会调用类父类的无参构造方法；如果该类没有其它父类，则调用 `Object` 的构造方法。如果父类没有构造方法（`Object` 确实有一个），编译器将拒绝该程序。
