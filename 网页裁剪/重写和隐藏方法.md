## 重写和隐藏方法

## 实例方法

子类中的实例方法，如果与超类中的实例方法具有相同的签名（方法名，加上参数的数量和类型）和返回类型，就会*重写（override）*超类的该方法。

子类能够重写方法的能力，使得一个类可以从某个行为"足够接近"的超类继承，然后根据需要修改其行为。重写的方法与其所重写的方法具有相同的名称、参数数量与类型，以及返回类型。重写的方法还可以返回被重写方法所返回类型的某个子类型。这种子类型被称为*协变返回类型（covariant return type）*。

在重写方法时，你可能希望使用 `@Override` 注解，它告知编译器你打算重写超类中的某个方法。如果由于某种原因，编译器检测到该方法在超类中并不存在，则会生成一个错误。有关 `@Override` 的更多信息，请参见[`注解（Annotations）`](https://docs.oracle.com/javase/tutorial/java/annotations/index.html)。

## 静态方法

如果子类定义了一个与超类中静态方法具有相同签名的静态方法，那么子类中的该方法会*隐藏（hide）*超类中的该方法。

隐藏静态方法与重写实例方法之间的区别具有重要的含义：

- 被调用的、被重写的实例方法版本是子类中的那个版本。
- 被调用的、被隐藏的静态方法版本取决于它是从超类还是从子类中被调用。

考虑一个包含两个类的示例。第一个是 `Animal`，它包含一个实例方法和一个静态方法：

```java
public class Animal {
    public static void testClassMethod() {
        System.out.println("The static method in Animal");
    }
    public void testInstanceMethod() {
        System.out.println("The instance method in Animal");
    }
}
```

第二个类是 `Animal` 的子类，名为 `Cat`：

```java
public class Cat extends Animal {
    public static void testClassMethod() {
        System.out.println("The static method in Cat");
    }
    public void testInstanceMethod() {
        System.out.println("The instance method in Cat");
    }

    public static void main(String[] args) {
        Cat myCat = new Cat();
        Animal myAnimal = myCat;
        Animal.testClassMethod();
        myAnimal.testInstanceMethod();
    }
}
```

`Cat` 类重写了 `Animal` 中的实例方法，并隐藏了 `Animal` 中的静态方法。该类的 `main` 方法创建了 `Cat` 的一个实例，并在类上调用 `testClassMethod()`，在实例上调用 `testInstanceMethod()`。

该程序的输出如下：

```text
The static method in Animal
The instance method in Cat
```

正如所言，被调用的、被隐藏的静态方法版本是超类中的那个版本，而被调用的、被重写的实例方法版本是子类中的那个版本。

## 接口方法

接口中的[默认方法（default method）](https://docs.oracle.com/javase/tutorial/java/IandI/defaultmethods.html)和[抽象方法（abstract method）](https://docs.oracle.com/javase/tutorial/java/IandI/abstract.html)会像实例方法一样被继承。然而，当一个类或接口的超类型提供了多个具有相同签名的默认方法时，Java 编译器会遵循继承规则来解决名称冲突。这些规则由以下两条原则驱动：

- 实例方法优先于接口的默认方法。
	考虑下面的类和接口：
	```java
	public class Horse {
	    public String identifyMyself() {
	        return "I am a horse.";
	    }
	}
	```
	```java
	public interface Flyer {
	    default public String identifyMyself() {
	        return "I am able to fly.";
	    }
	}
	```
	```java
	public interface Mythical {
	    default public String identifyMyself() {
	        return "I am a mythical creature.";
	    }
	}
	```
	```java
	public class Pegasus extends Horse implements Flyer, Mythical {
	    public static void main(String... args) {
	        Pegasus myApp = new Pegasus();
	        System.out.println(myApp.identifyMyself());
	    }
	}
	```
	方法 `Pegasus.identifyMyself` 返回字符串 `I am a horse.`
- 已经被其他候选者重写的方法将被忽略。当超类型共享一个共同祖先时，就会出现这种情况。
	考虑下面的接口和类：
	```java
	public interface Animal {
	    default public String identifyMyself() {
	        return "I am an animal.";
	    }
	}
	```
	```java
	public interface EggLayer extends Animal {
	    default public String identifyMyself() {
	        return "I am able to lay eggs.";
	    }
	}
	```
	```java
	public interface FireBreather extends Animal { }
	```
	```java
	public class Dragon implements EggLayer, FireBreather {
	    public static void main (String... args) {
	        Dragon myApp = new Dragon();
	        System.out.println(myApp.identifyMyself());
	    }
	}
	```
	方法 `Dragon.identifyMyself` 返回字符串 `I am able to lay eggs.`

如果两个或更多独立定义的默认方法发生冲突，或者一个默认方法与一个抽象方法发生冲突，那么 Java 编译器会产生一个编译器错误。你必须显式地重写超类型的方法。

考虑那个关于现在能够飞行的、计算机控制的汽车的示例。你有两个接口（`OperateCar` 和 `FlyCar`）为同一个方法（`startEngine`）提供了默认实现：

```java
public interface OperateCar {
    // ...
    default public int startEngine(EncryptedKey key) {
        // 实现
    }
}
```
```java
public interface FlyCar {
    // ...
    default public int startEngine(EncryptedKey key) {
        // 实现
    }
}
```

一个同时实现 `OperateCar` 和 `FlyCar` 的类必须重写 `startEngine` 方法。你可以使用 `super` 关键字调用任意一个默认实现。

```java
public class FlyingCar implements OperateCar, FlyCar {
    // ...
    public int startEngine(EncryptedKey key) {
        FlyCar.super.startEngine(key);
        OperateCar.super.startEngine(key);
    }
}
```

`super` 前面的名称（在此示例中是 `FlyCar` 或 `OperateCar`）必须引用一个直接超接口，该超接口定义或继承了被调用方法的默认实现。这种形式的方法调用并不局限于区分包含相同签名默认方法的多个已实现接口。你可以使用 `super` 关键字在类和接口中调用默认方法。

从类继承的实例方法可以重写抽象的接口方法。考虑下面的接口和类：

```java
public interface Mammal {
    String identifyMyself();
}
```

```java
public class Horse {
    public String identifyMyself() {
        return "I am a horse.";
    }
}
```

```java
public class Mustang extends Horse implements Mammal {
    public static void main(String... args) {
        Mustang myApp = new Mustang();
        System.out.println(myApp.identifyMyself());
    }
}
```

方法 `Mustang.identifyMyself` 返回字符串 `I am a horse.`。类 `Mustang` 从类 `Horse` 继承了方法 `identifyMyself`，后者重写了接口 `Mammal` 中同名的抽象方法。

**注意**：接口中的静态方法永远不会被继承。

## 修饰符

重写方法的访问修饰符可以允许比被重写方法更多、但不能更少的访问权限。例如，超类中一个受保护的（protected）实例方法在子类中可以变为公共的（public），但不能变为私有的（private）。

如果你试图将超类中的实例方法改为子类中的静态方法，或反之，将会得到一个编译时错误。

## 小结

下表总结了当你定义一个与超类中方法具有相同签名的方法时会发生什么。

|  | 超类实例方法 | 超类静态方法 |
| --- | --- | --- |
| 子类实例方法 | 重写 | 生成编译时错误 |
| 子类静态方法 | 生成编译时错误 | 隐藏 |

---

**注意：** 在子类中，你可以重载（overload）从超类继承来的方法。这种重载的方法既不隐藏、也不重写超类的实例方法——它们是子类独有的新方法。
