## RMI 应用程序概述

RMI 应用程序通常由两个独立的程序组成：一个服务器和一个客户端。典型的服务器程序创建一些远程对象，使对这些对象的引用可访问，并等待客户端在这些对象上调用方法。典型的客户端程序获取对服务器上一个或多个远程对象的远程引用，然后在这些对象上调用方法。RMI 提供服务器和客户端之间通信和来回传递信息的机制。这样的应用程序有时被称为*分布式对象应用程序(distributed object application)*。

分布式对象应用程序需要执行以下操作：

- **定位远程对象。** 应用程序可以使用各种机制获取对远程对象的引用。例如，应用程序可以将其远程对象注册到 RMI 的简单命名设施——RMI 注册表(registry)中。或者，应用程序可以传递和返回远程对象引用作为其他远程调用的一部分。
- **与远程对象通信。** 远程对象之间的通信细节由 RMI 处理。对于程序员来说，远程通信看起来就像常规的 Java 方法调用。
- **加载被传递对象的类定义。** 因为 RMI 允许对象来回传递，所以它提供了加载对象类定义以及传输对象数据的机制。

## 动态代码加载的优势

RMI 的核心且独特的特性之一是它能够在接收者的 Java 虚拟机中未定义对象类时下载该对象类的定义。对象的所有类型和行为（以前仅在单个 Java 虚拟机中可用）可以传输到另一个（可能是远程的）Java 虚拟机。RMI 按对象的实际类传递对象，因此对象的行为在发送到另一个 Java 虚拟机时不会改变。此功能使新的类型和行为能够被引入到远程 Java 虚拟机中，从而动态扩展应用程序的行为。本路线中的计算引擎示例使用此功能向分布式程序引入新行为。

## 远程接口、对象和方法

与任何其他 Java 应用程序一样，使用 Java RMI 构建的分布式应用程序由接口和类组成。接口声明方法。类实现接口中声明的方法，可能还会声明额外的方法。在分布式应用程序中，某些实现可能存在于某些 Java 虚拟机中而不存在于其他虚拟机中。具有可跨 Java 虚拟机调用的方法的对象称为*远程对象(remote objects)*。

对象通过实现*远程接口*而成为远程对象，该接口具有以下特征：

- 远程接口继承接口 `java.rmi.Remote`。
- 该接口的每个方法在其 `throws` 子句中声明 `java.rmi.RemoteException`（除任何应用程序特定的异常之外）。

当对象从一个 Java 虚拟机传递到另一个 Java 虚拟机时，RMI 对远程对象的处理与非远程对象不同。RMI 不是在接收 Java 虚拟机中制作实现对象的副本，而是为远程对象传递一个远程*存根(stub)*。存根充当远程对象的本地代表或代理，对于客户端来说基本上就是远程引用。客户端在本地存根上调用方法，存根负责在远程对象上执行方法调用。

远程对象的存根实现与远程对象实现的相同一组远程接口。此属性使存根能被强制转换为远程对象实现的任何接口。但是，*只有*远程接口中定义的方法才可从接收 Java 虚拟机调用。

## 使用 RMI 创建分布式应用程序

使用 RMI 开发分布式应用程序涉及以下一般步骤：

1. 设计并实现分布式应用程序的组件。
2. 编译源代码。
3. 使类可网络访问。
4. 启动应用程序。

### 设计并实现应用程序组件

首先，确定你的应用程序架构，包括哪些组件是本地对象，哪些组件是可远程访问的。此步骤包括：

- **定义远程接口。** 远程接口指定客户端可以远程调用的方法。客户端针对远程接口编程，而不是这些接口的实现类。此类接口的设计包括确定将用作这些方法的参数和返回值的对象类型。如果这些接口或类中的任何一个尚不存在，你也需要定义它们。
- **实现远程对象。** 远程对象必须实现一个或多个远程接口。远程对象类可能包括仅在本地可用的其他接口和方法的实现。如果任何本地类要用作这些方法中任何一个的参数或返回值，也必须实现它们。
- **实现客户端。** 使用远程对象的客户端可以在定义远程接口后的任何时候实现，包括在部署远程对象之后。

### 编译源代码

与任何 Java 程序一样，你使用 `javac` 编译器编译源文件。源文件包含远程接口的声明、它们的实现、任何其他服务器类以及客户端类。

---

**注意：** 对于 Java 平台标准版 5.0 之前的版本，需要额外的步骤使用 `rmic` 编译器构建存根类。但是，此步骤已不再需要。

### 使类可网络访问

在此步骤中，你使某些类定义可网络访问，例如远程接口及其关联类型的定义，以及需要下载到客户端或服务器的类定义。类定义通常通过 Web 服务器实现网络可访问。

### 启动应用程序

启动应用程序包括运行 RMI 远程对象注册表、服务器和客户端。

## 构建通用计算引擎

本路线关注一个简单而强大的分布式应用程序，称为*计算引擎(compute engine)*。计算引擎是服务器上的一个远程对象，它从客户端接收任务、运行任务并返回任何结果。任务在服务器运行的机器上运行。这种类型的分布式应用程序可以使许多客户端机器利用一台特别强大的机器或具有专用硬件的机器。

计算引擎的新颖之处在于它运行的任务不需要在计算引擎编写或启动时定义。可以在任何时候创建新类型的任务，然后交给计算引擎运行。任务的唯一要求是其类实现特定接口。完成任务所需的代码可以由 RMI 系统下载到计算引擎。然后，计算引擎使用运行它的机器上的资源运行任务。

执行任意任务的能力由 Java 平台的动态特性实现，RMI 将此特性扩展到网络。RMI 动态地将任务代码加载到计算引擎的 Java 虚拟机中，并在没有事先了解实现任务的类的情况下运行任务。这种能够动态下载代码的应用程序通常被称为*基于行为的应用程序(behavior-based application)*。此类应用程序通常需要完整的支持代理的基础设施。使用 RMI，此类应用程序是 Java 平台上分布式计算基本机制的一部分。
