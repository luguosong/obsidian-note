## 学习路径：集合

本节描述 Java 集合框架（Java Collections Framework）。在此您将了解什么是集合，以及它们如何让您的工作更轻松、程序更优秀。您将学习构成 Java 集合框架的核心要素——接口、实现、聚合操作和算法。

[**简介（Introduction）**](https://docs.oracle.com/javase/tutorial/collections/intro/index.html) 告诉您什么是集合，以及它们如何让您的工作更轻松、程序更优秀。您将了解构成集合框架的核心要素：*接口（interfaces）*、*实现（implementations）* 和 *算法（algorithms）*。

[**接口（Interfaces）**](https://docs.oracle.com/javase/tutorial/collections/interfaces/index.html) 描述*核心集合接口*，它们是 Java 集合框架的核心与灵魂。您将学习有效使用这些接口的一般准则，包括何时使用哪个接口。您还将学习每个接口的惯用写法，帮助您充分发挥接口的作用。

[**聚合操作（Aggregate Operations）**](https://docs.oracle.com/javase/tutorial/collections/streams/index.html) 代替您遍历集合，使您能够编写更简洁高效的代码来处理存储在集合中的元素。

[**实现（Implementations）**](https://docs.oracle.com/javase/tutorial/collections/implementations/index.html) 描述 JDK 的*通用集合实现*，并告诉您何时使用哪种实现。您还将学习*包装器实现（wrapper implementations）*，它们为通用实现添加功能。

[**算法（Algorithms）**](https://docs.oracle.com/javase/tutorial/collections/algorithms/index.html) 描述 JDK 提供的用于操作集合的*多态算法（polymorphic algorithms）*。运气好的话，您再也不必自己编写排序程序了！

[**自定义实现（Custom Implementations）**](https://docs.oracle.com/javase/tutorial/collections/custom-implementations/index.html) 告诉您为什么可能需要编写自己的集合实现（而不是使用 JDK 提供的通用实现），以及如何着手去做。借助 JDK 的*抽象集合实现（abstract collection implementations）*，这很简单！

[**互操作性（Interoperability）**](https://docs.oracle.com/javase/tutorial/collections/interoperability/index.html) 告诉您集合框架如何与在集合引入 Java 之前就已存在的旧 API 互操作。此外，它还告诉您如何设计新的 API，使其能与其他新 API 无缝互操作。
