## 类型擦除

泛型被引入 Java 语言是为了在编译时提供更严格的类型检查，并支持泛型编程。为了实现泛型，Java 编译器应用类型擦除（type erasure）来：

- 将泛型类型中的所有类型参数替换为它们的边界，如果类型参数是无界的则替换为 Object。因此，生成的字节码只包含普通的类、接口和方法。
- 在必要时插入类型转换以保持类型安全。
- 生成桥接方法以在扩展的泛型类型中保持多态。

类型擦除确保不会为参数化类型创建新的类；因此，泛型不会带来运行时开销。
