# 创建普通笔记

创建一篇普通的内容笔记，放在某个已有分类目录下。基础模版为 `模版/普通笔记模版.md`。

## 与 doc-create-category 的区别

- 本 skill 创建**普通笔记**（写内容的知识点笔记）。
- 若要新建一个分类（目录 + 文件夹笔记），用 `doc-create-category`。
- 一篇普通笔记的 `分类` 指向其所在目录的文件夹笔记，而**不**是它自己——不要混淆。

## 铁律：必须先拿到明确位置

- 开始写文件前，必须明确两件事：
  1. **笔记名**
  2. **相对位置**：放在哪个分类目录下，以相对 `编程笔记/` 的路径表达，如「`编程笔记/03-Java/02-服务端开发/Spring/` 下」。
- 如果用户只给了笔记名、没说放哪，**停下来询问**，不要根据笔记主题猜测它该归到哪个分类。用 AskUserQuestion 列出可选分类目录，或直接请用户指定。

> 为什么：这篇笔记归入哪个分类是用户的决策。猜错分类会导致 `分类` 双链、目录归属都错，而且笔记一旦放错位置，后续会被"位置锁死"。

## 工作流

1. **确认笔记名与位置**（缺一不可，缺则询问）。笔记名用中文，简短有意义，无日期前缀。
2. **推导 `分类` 属性**：指向**所在目录的文件夹笔记**（= 所在目录名）。例：放在 `编程笔记/03-Java/02-服务端开发/Spring/` 下，`分类` 写 `- "[[Spring]]"`。
3. **填充 frontmatter**：`分类` 填好；其余字段留空但保留（见下方模版）。`关联笔记` 可在了解用户明确意图后补充相关节点，但**不要自动臆造**关联。
4. **写正文**：首行为与文件名一致的一级标题。用 Obsidian 风格语法：
   - 内部链接用 wikilink：`[[笔记名]]`、`[[笔记名|显示文本]]`
   - 外部链接用标准 Markdown：`[文本](https://...)`
   - 高亮用 `==文本==`
   - 代码块标注语言：` ```java `、` ```js ` 等
5. **创建文件**到指定位置。

## 普通笔记模版

````markdown
---
分类:          # 指向所在目录的文件夹笔记，如 - "[[Spring]]"
关联笔记:
描述:
排序:
分组:
创建时间: 2026-06-25 16:00
修改时间: 星期四 25日 六月 2026 16:00:00
---
# 笔记名

````

- 时间填**当前时间**，不要回填历史。格式对齐仓库现有笔记：创建时间 `YYYY-MM-DD HH:mm`，修改时间 `星期X X日 X月 YYYY HH:mm:ss`。
- frontmatter 字段未用到可留空，但**不要删除字段**，保持结构一致。

完整字段语义见项目根 `AGENTS.md` 的「Frontmatter」一节。注意 `分类`（唯一物理归属，单一值）与 `关联笔记`（多值关系网）职责不同，跨主题关联一律走 `关联笔记`，不要破坏 `分类` 的唯一性。
