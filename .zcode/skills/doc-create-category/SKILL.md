# 创建分类（目录 + 文件夹笔记）

本仓库的 `编程笔记/` 采用「目录即分类 + 文件夹笔记」的组织方式（配合 Notebook Navigator 插件）。**创建一个分类 = 同时创建三样东西**：

1. 一个目录
2. 该目录**内部**一个与目录同名的 `.md` 文件夹笔记（且文件夹笔记内嵌通用 base `![[分类索引.base]]`，作为该分类的动态索引页）
3. 在 Notebook Navigator 配置里登记该目录的**图标**与**颜色**（写入 `.obsidian/plugins/notebook-navigator/data.json`），让新分类一建好就在导航栏有辨识度

## 铁律：必须先拿到明确位置

这是本 skill 最核心的约束。创建分类会**改变目录树结构**，位置一旦错了，后续所有子笔记的 `分类` 双链、文件夹笔记层级都会跟着错，回溯成本高。因此：

开始任何写操作之前，必须先明确两件事：

1. **分类名**（= 目录名 = 文件夹笔记文件名）
2. **相对位置**：新分类作为哪个目录的子目录。位置以**相对于 `编程笔记/` 的路径**表达，如「`编程笔记/03-Java/` 下」。

如果用户只给了分类名、没给明确位置，**停下来询问**，不要根据分类主题猜测它该归到哪个顶层分类下。用 AskUserQuestion 列出可选父目录，或直接请用户指定。

> 为什么：新分类挂在哪一支下面，是用户的分类决策，不是 agent 能推断的。这一步省事的代价是后面一连串的错误归属。

## 工作流

1. **确认分类名与位置**（缺一不可，缺则询问）。
   - 分类名用中文；目录是否带编号前缀由用户决定（如 `01-基础` 或 `并发`），不要擅自加编号。
2. **校验深度**：从 `编程笔记/` 算起，目录深度不得超过 3 层（即路径最多 `编程笔记/二级/三级/三级.md`）。若新建会让深度超过 3 层，向用户说明并建议改用 `关联笔记` 双链或标签表达细分主题，而非再开目录。
3. **推导文件夹笔记的 `分类` 属性**：
   - 父目录是 `编程笔记/` 本身（即顶层分类，如 `编程笔记/01-基础知识/`）→ `分类` **留空**（它就是分类树的根）。
   - 否则 → `分类` 指向**父目录的文件夹笔记**（= 父目录名）。例：在 `编程笔记/03-Java/` 下创建 `并发`，文件夹笔记的 `分类` 写 `- "[[03-Java]]"`。
4. **创建目录**：`编程笔记/<父路径>/<分类名>/`
5. **在目录内部创建文件夹笔记**：`<新目录>/<分类名>.md`，内容用下方模版（含 `![[分类索引.base]]` 嵌入，使该分类成为动态索引页）。
6. **配置 Notebook Navigator 图标与颜色**：按下方「配置 Notebook Navigator 图标与颜色」章节确定图标和颜色，用 Edit 把新条目追加进 `.obsidian/plugins/notebook-navigator/data.json` 的 `folderIcons` 和 `folderColors` 两个 map。
7. **提示用户重载插件**：配置写完后告诉用户在 Obsidian 里重载 Notebook Navigator（设置 → 第三方插件 → 关闭再开启，或 `Ctrl+P` → `Reload app without saving`）才可见效。

## 配置 Notebook Navigator 图标与颜色

Notebook Navigator 用 `.obsidian/plugins/notebook-navigator/data.json` 里的 `folderIcons` 和 `folderColors` 两个 map 控制文件夹外观。创建分类后必须同步登记，否则新分类在导航栏无图标无颜色。

### key 格式（重要）

map 的 **key = 新分类在仓库根的完整相对路径，无尾部斜杠**。例：在 `编程笔记/03-Java/` 下建 `并发`，key 是 `编程笔记/03-Java/并发`（不是 `并发`，也不是 `编程笔记/03-Java/并发/`）。

### 颜色：继承父分类（同专题同色）

仓库约定**同一专题子树统一颜色**，颜色表达「专题归属」。因此新分类的颜色**默认从父级继承**：

1. 读取 `folderColors`，把新分类路径**逐级向上**剥，找**最近的已配色祖先目录**，取它的颜色。
   - 例：新分类 `编程笔记/03-Java/并发` → 祖先依次是 `编程笔记/03-Java`（已配 `#ed8b00`）、`编程笔记`（已配 `#3b82f6`）→ 取最近的 `编程笔记/03-Java` 的 `#ed8b00` 橙。
2. **例外——新建顶层分类**（父目录是 `编程笔记/` 本身）：无祖先可继承，从下方调色板挑一个**尚未被使用的颜色**；拿不准时问用户。
3. 用户明确指定颜色时，以用户为准。

### 写入方式

用 Edit 把一行新条目追加进对应 map，**2 空格缩进**，保持 JSON 合法。**不要整文件重写**，只动两个 map 里对应位置。

追加示例（在 `编程笔记/03-Java/` 下建 `并发`，继承 Java 橙）：

在 `folderColors` 里 `编程笔记/03-Java/02-服务端开发/Spring` 那一行**之后**插入：
```json
    "编程笔记/03-Java/并发": "#ed8b00",
```
在 `folderIcons` 里对应位置插入：
```json
    "编程笔记/03-Java/并发": "git-branch",
```

> 写入顺序无所谓功能，但按字母/路径顺序插入更易维护。写完后用 `node -e "JSON.parse(...)"` 校验 JSON 合法性，避免缩进或逗号错误。

### 校验

写完两个条目后，必须跑一次 JSON 校验：
```bash
node -e "JSON.parse(require('fs').readFileSync('.obsidian/plugins/notebook-navigator/data.json','utf8')); console.log('JSON 格式正确')"
```
校验失败则修正后再继续。

## 文件夹笔记必须放在目录内部

⚠️ 文件夹笔记的路径必须是 `目录/目录名.md`（在目录**内部**）。

- ✅ 正确：`编程笔记/03-Java/01-基础/01-基础.md`
- ❌ 错误：`编程笔记/03-Java/01-基础.md`（放在父目录里）

放在父目录里会被 Notebook Navigator 当成普通笔记，无法绑定文件夹——这是最常犯的错误。

## 文件夹笔记模版

````markdown
---
分类:          # 按工作流步骤 3 推导；顶层分类留空
关联笔记:
描述:
排序:
分组:
创建时间: 2026年06月25日
修改时间: 2026年06月25日
---
# 分类名

## 本分类下的笔记

![[分类索引.base]]

````

- **嵌入 `分类索引.base` 是创建分类的标配**：仓库有一个通用 base `数据库/分类索引.base`，它用 `this` 关键字自动筛选「`分类` 属性指向当前文件夹笔记」的所有笔记。每个文件夹笔记都通过 `![[分类索引.base]]` 嵌入它，从而成为该分类的动态索引页——新增子笔记只要 `分类` 双链写对，就会自动出现在索引里，无需逐个分类维护。
- **索引只显示普通笔记，自动排除子分类页面**：base 通过「文件名 == 所在目录名」的公式判据，自动过滤掉子分类的文件夹笔记，只列出真正的普通笔记。因此新建子分类后，它**不会**出现在父分类的索引里——这是预期行为（子分类该在它自己的索引页里聚合它的笔记，而非挂在父级）。
- 文件名、目录名、一级标题三者完全一致。
- 时间填**当前时间**，不要回填历史。格式统一为中文：创建时间 / 修改时间均写 `YYYY年MM月DD日`（如 `2026年06月25日`）。
- frontmatter 字段未用到可留空，但**不要删除字段**，保持结构一致。

完整字段语义与分类机制见项目根 `AGENTS.md` 的「笔记分类机制」「Frontmatter」两节。
