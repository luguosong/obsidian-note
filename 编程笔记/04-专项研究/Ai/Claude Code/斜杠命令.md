# 斜杠命令

## 内置斜杠命令

### 会话管理

| 命令 | 作用 |
|---|---|
| `/clear` | 清空上下文，开始新对话（旧会话保留在 `/resume`） |
| `/compact` | 压缩对话历史，释放上下文（可传聚焦指令） |
| `/context` | 可视化上下文占用，给出优化建议 |
| `/resume` | 按名称 / ID 恢复历史会话 |
| `/branch` | 在当前对话点创建分支 |
| `/export` | 导出对话为纯文本 |
| `/recap` | 生成当前会话的单行摘要 |

### 配置与设置

| 命令 | 作用 |
|---|---|
| `/config` | 调整主题、模型、输出样式等（v2.1.181+ 可直接传 `key=value`） |
| `/model` | 切换模型 |
| `/effort` | 设置推理力度 |
| `/fast` | 切换快速模式 |
| `/permissions` | 管理工具权限规则 |
| `/status` | 显示版本、模型、账户、连接状态 |
| `/usage` (`/cost`) | 显示成本与额度 |

### 项目与记忆

| 命令 | 作用 |
|---|---|
| `/init` | 初始化项目并生成 `CLAUDE.md` |
| `/memory` | 编辑 `CLAUDE.md`，管理自动记忆 |
| `/add-dir` | 为会话添加额外工作目录 |
| `/cd` | 切换工作目录（v2.1.169+） |

### 开发与审查

| 命令 | 作用 |
|---|---|
| `/plan` | 进入计划模式 |
| `/code-review` | 审查 diff 的正确性与可清理项（ultra 走云端深度审查） |
| `/review` | 审查 GitHub PR |
| `/security-review` | 安全漏洞分析 |
| `/simplify` | 审查并应用简化修复 |
| `/diff` | 交互式查看未提交变更 |

### 代理与并行工作

| `/tasks` | 查看后台任务 |
| `/workflows` | 工作流进度视图 |
| `/goal` | 设置目标条件，跨轮次工作 |

### 几个要点

> [!tip] 别名
> 很多命令有别名：`/reset` → `/clear`、`/quit` → `/exit`、`/bug` → `/feedback`、`/app` → `/desktop`。

> [!warning] 非交互模式（`claude -p`）
> 仅 `/help`、`/compact`、`/resume`（经 flag）等少数命令可用，绝大多数斜杠命令需要交互式会话。

> [!note] 平台 / 计划限制
> - `/desktop` 需 macOS / Windows + 订阅。
> - `/setup-bedrock`、`/setup-vertex` 需对应环境变量。
> - 部分命令需 Pro / Max。

> [!note] 版本演进
> - 部分命令需特定最低版本（如 `/cd` v2.1.169、`/reload-skills` v2.1.152）。
> - 旧的 `/vim`、`/pr-comments` 已移除。
