# 斜杠命令

## 定义

> [!note] 定义
> 斜杠命令是你在 Claude 的交互式会话中用来控制行为的快捷方式。

## 分类

| 类型 | 说明 | 示例 |
|---|---|---|
| 内置命令 | Claude Code 自带 | `/help`、`/clear`、`/model` |
| Skills | 基于自定义 `SKILL.md` 文件 | `/optimize`、`/pr` |
| 插件命令 | 来自已安装的插件 | `/frontend-design:frontend-design` |
| MCP prompts | 来自 MCP server | `/mcp__github__list_prs` |

> [!note] 自定义命令与 Skills 的关系
> 自定义 slash command 已经合并进 Skills。`.claude/commands/` 仍然可用，但现在更推荐使用 `.claude/skills/`。两者都会创建 `/command-name` 形式的快捷命令。完整参考请见 [Skills 指南](https://github.com/luongnv89/claude-howto/blob/main/zh/03-skills/README.md)。

## 自定义斜杠命令

在项目或用户目录下放置 `.md` 文件即可注册自定义斜杠命令：

```bash
# 项目级：仅当前项目可用
<project>/.claude/commands/<name>.md

# 用户级：所有项目可用
~/.claude/commands/<name>.md
```

> [!tip] 同名冲突
> 如果 skill 和 command 同名，**skill 优先**。

## 内置斜杠命令

### 会话管理

| 命令 | 作用 |
|---|---|
| `/clear [name]` | 清空上下文，开始新对话（旧会话留在 `/resume`；别名 `/reset`、`/new`） |
| `/compact [instructions]` | 压缩对话历史，释放上下文（可传聚焦指令） |
| `/context [all]` | 用彩色网格可视化上下文占用，给出优化建议 |
| `/resume [session]` | 按 ID / 名称恢复历史会话（别名 `/continue`） |
| `/branch [name]` | 在当前对话点创建分支，原对话可经 `/resume` 返回 |
| `/fork <directive>` | 派生后台子代理处理任务，结果回传主会话（区别于切换对话的 `/branch`） |
| `/rewind` | 回退对话和/或代码到之前某点（别名 `/checkpoint`、`/undo`） |
| `/rename [name]` | 重命名当前会话 |
| `/recap` | 生成当前会话的单行摘要 |
| `/export [filename]` | 导出对话为纯文本或文件 |
| `/copy [N]` | 复制最近第 N 条回复到剪贴板，`w` 写入文件 |
| `/btw <question>` | 问一个不入对话历史的临时问题 |
| `/exit` | 退出 CLI（别名 `/quit`；在附加的后台会话中则脱离，会话继续运行） |

### 配置与设置

| 命令 | 作用 |
|---|---|
| `/config [key=value]` | 打开设置；v2.1.181+ 可直接传 `key=value`（如 `/config theme=dark`，别名 `/settings`） |
| `/model [model]` | 切换模型，可用左右箭头调 effort；选择器显示可读标签（如 "Sonnet 4.6"） |
| `/effort [level\|auto]` | 设置推理强度：`low`/`medium`/`high`/`xhigh`/`max`/`ultracode`（视模型而定） |
| `/fast [on\|off]` | 切换快速模式 |
| `/permissions` | 管理工具权限的 allow/ask/deny 规则（别名 `/allowed-tools`） |
| `/sandbox` | 切换沙盒模式 |
| `/usage-credits` | 配置用量额度以应对速率限制（原名 `/extra-usage`） |
| `/usage` | 显示成本、额度与活动统计（别名 `/cost`、`/stats`） |
| `/status` | 显示版本、模型、账号、连接状态 |
| `/color [color\|default]` | 设置本次会话的提示栏颜色 |
| `/theme` | 更改颜色主题 |
| `/statusline` | 配置状态栏 |
| `/keybindings` | 打开快捷键配置文件 |
| `/terminal-setup` | 配置终端快捷键（如 Shift+Enter） |
| `/privacy-settings` | 隐私设置（仅 Pro / Max） |
| `/voice [hold\|tap\|off]` | 切换按住说话语音听写 |


### 项目与记忆

| 命令 | 作用 |
|---|---|
| `/init` | 初始化项目并生成 `CLAUDE.md` |
| `/memory` | 编辑 `CLAUDE.md`，管理自动记忆 |
| `/add-dir <path>` | 为会话添加额外工作目录 |
| `/cd <path>` | 切换会话工作目录，保留提示缓存（v2.1.169+） |

### 开发与审查

| 命令 | 作用 |
|---|---|
| `/plan [description]` | 进入计划模式（可带描述直接开始） |
| `/code-review [...]` | （Skill）审查 diff 的正确性与可清理项，`--fix` 应用、`ultra` 走云端深度审查 |
| `/review [PR]` | 按 PR 号审查 GitHub PR（与 `/code-review` 同引擎；无参则列出可选 PR） |
| `/security-review` | 分析当前分支待提交变更的安全漏洞 |
| `/simplify [target]` | （Skill）审查改动代码的清理机会并应用修复（不含 bug 查找） |
| `/diff` | 交互式查看未提交变更与逐轮 diff |
| `/run` | （Skill）启动并驱动应用，在运行中的应用里查看变更效果（v2.1.145+） |
| `/verify` | （Skill）构建、运行应用并观察结果以验证变更（v2.1.145+） |
| `/run-skill-generator` | （Skill）教 `/run`、`/verify` 如何构建/启动/驱动本项目应用 |
| `/claude-api [migrate]` | （Skill）加载 Claude API 参考资料；`migrate` 升级旧代码到新模型 |
| `/deep-research <question>` | （Workflow）多源联网调研并生成带引用的报告 |

### 几个要点

> [!tip] 别名速查
> - `/reset`、`/new` → `/clear`；`/continue` → `/resume`
> - `/quit` → `/exit`；`/bug`、`/share` → `/feedback`
> - `/settings` → `/config`；`/allowed-tools` → `/permissions`
> - `/app` → `/desktop`；`/ios`、`/android` → `/mobile`
> - `/checkpoint`、`/undo` → `/rewind`；`/bashes` → `/tasks`
> - `/rc` → `/remote-control`；`/routines` → `/schedule`；`/proactive` → `/loop`

> [!warning] 非交互模式（`claude -p`）
> 仅 `/help`、`/compact`、`/resume`（经 flag）等少数命令可用，绝大多数斜杠命令需要交互式会话。

> [!note] 版本演进与变更
> - `/fork`：v2.1.161 前是 `/branch` 的别名，之后成为独立的「派生子代理」命令。
> - `/usage-credits`：由原 `/extra-usage` 更名而来。
> - `/pr-comments`：v2.1.91 移除，改为直接让 Claude 查看 PR 评论。
> - `/vim`：v2.1.92 移除，改用 `/config` → Editor mode。
> - `/cd` v2.1.169、`/reload-skills` v2.1.152、`/run`·`/verify` v2.1.145+ 才可用。

> [!note] 平台 / 计划限制
> - `/desktop` 需 macOS / Windows + 订阅；`/mobile` 需对应扫码环境。
> - `/setup-bedrock`、`/setup-vertex` 需对应环境变量。
> - `/privacy-settings` 需 Pro / Max；部分命令需特定套餐。
