# 概述

Java 既是一种编程语言，也是一个平台。

## 核心特性

| 特性 | 定位 | 关键机制 |
|------|------|----------|
| **简单易学** | 针对 C/C++ 的痛点设计，降低出错率 | 无指针运算、自动内存管理、`String` 内置类型、独立 `boolean` 类型 |
| **面向对象** | 除基本类型外一切皆对象，禁止多重继承和全局函数 | 封装（访问修饰符）、继承（单继承）、多态（运行时动态绑定） |
| **平台无关性** | Write Once, Run Anywhere | 源码编译为与平台无关的字节码（`.class`），由 JVM 解释执行 |
| **安全性** | 四层防护机制 | 字节码校验器、沙箱模型、无指针运算、数组边界检查 |
| **高性能（JIT）** | 热点代码即时编译为原生机器码，运行越久越快 | HotSpot 编译器、内联优化、逃逸分析、循环展开 |
| **多线程** | 语言级内置，Java 21+ 支持虚拟线程 | `Thread`/`Runnable`、`synchronized`、`java.util.concurrent`、虚拟线程 |
| **自动垃圾回收** | GC 自动追踪对象引用并回收，消除内存泄漏 | 无需手动 `malloc`/`free`，JVM 在适当时机自动回收不可达对象 |

## Java 技术体系

Java 平台由两个组件构成：

- **Java 虚拟机（JVM）**：负责加载、校验并执行 `.class` 字节码，是 Java 跨平台能力的基础。同一份 `.class` 文件可在 Windows、Linux、Solaris、Mac OS 等不同操作系统上运行——因为每种平台都有各自的 JVM 实现。
- **Java 应用编程接口（API）**：一个庞大的现成软件组件集合，被组织成相关类和接口的库；这些库被称为==包（Package）==。

![[Java平台组件.png]]

源代码首先以 `.java` 纯文本编写，由 `javac` 编译成平台无关的字节码（`.class`），再由 JVM 解释执行；HotSpot 等虚拟机会在运行时把热点代码即时编译为本地机器码以提升性能。

![[Java编译运行流程.png]]

![[JVM跨平台.png]]

> [!note] Java 既是编译型语言又是解释型语言
> 源码先由 `javac` `编译`成平台无关的字节码（`.class`），再由 JVM `解释执行`，同时 JIT 会将热点字节码 `编译`成本地机器码。

## 发展史

Java 由 Sun Microsystems 于 1990 年代初启动，1996 年 JDK 1.0 正式发布；2010 年 Oracle 收购 Sun 后接手维护。自 Java 9 起改为每 6 个月发布一个新版本，每 2 年推出一个长期支持（LTS）版本。

> [!info]- 完整时间线
> - `1991 年 6 月`：James Gosling、Mike Sheridan、Patrick Naughton 在 Sun 启动 Java 项目
>   - 最初面向交互式电视；语言最初取名 _Oak_（源自 Gosling 办公室窗外的橡树），后改名 _Green_，最终定名 _Java_（来自印度尼西亚的 Java 咖啡）
>   - 语法沿用 C/C++ 风格，让系统与应用程序员易于上手
> - `1995 年 5 月 23 日`：Sun 在 SunWorld 大会正式发布 Java，Netscape 同日宣布浏览器支持 Java
> - `1996 年 1 月 23 日`：**JDK 1.0** 发布，Java 语言正式诞生
>   - 承诺「Write Once, Run Anywhere」；主流浏览器随即支持运行 Java applet，Java 迅速流行
> - `1997 年 2 月 19 日`：**JDK 1.1** 发布（内部类、JavaBeans、JDBC）
>   - 同年首届 JavaOne 大会逾万人参会；Sun 曾谋求 ISO/IEC、Ecma 标准化但退出，Java 至今仍是 JCP 主导的==事实标准==
> - `1998 年 12 月 8 日`：**J2SE 1.2** 发布（集合框架、`反射`、Swing、JIT 编译器），Java 2 时代开启
>   - 版本按平台拆分：J2EE（企业）、J2ME（移动）、J2SE（桌面）
> - `2000 年 5 月 8 日`：**J2SE 1.3** 发布，`HotSpot` 成为 Sun 默认 JVM
> - `2002 年 2 月 6 日`：**J2SE 1.4** 发布（正则表达式、异常链、NIO、XML 解析器）
> - `2004 年 9 月 30 日`：**J2SE 5.0** 发布（`泛型`、for-each、自动装箱、`枚举`、可变参数），首次去掉「1.」前缀
> - `2006 年`：开源与更名双管齐下
>   - `11 月 13 日`：Sun 将大部分 JVM 以 GPL-2.0-only 许可开源；`12 月 11 日`发布 **Java SE 6**（脚本语言支持 JSR 223、可插拔注解）
>   - 同年 J2EE/J2ME/J2SE 正式更名为 Java EE / Java ME / Java SE
> - `2007 年 5 月 8 日`：Sun 完成 JVM 核心代码的开源
> - `2010 年`：Oracle 收购 Sun，接手 Java 维护
>   - `1 月 27 日`Oracle 收购 Sun；`4 月 2 日`James Gosling 离职 Oracle；11 月 Apache 威胁退出 JCP
> - `2011 年 7 月 28 日`：**Java SE 7** 发布（try-with-resources、switch 字符串、`fork/join 框架`）；自 Java SE 7 起 OpenJDK 成为官方参考实现
> - `2014 年 3 月 18 日`：**Java SE 8 LTS** 发布（`Lambda 表达式`、Stream API、`Optional`、新日期时间 API）
> - `2017 年 9 月 21 日`：**Java SE 9** 发布（模块系统 Jigsaw）；自 Java 9 起改为 6 个月发布节奏，浏览器插件逐步弃用
> - `2018 年 9 月 25 日`：**Java SE 11 LTS** 发布（HTTP Client、Flight Recorder、移除 Java EE 模块）
>   - Oracle JDK 转为商业许可，OpenJDK 成为免费开源替代（Adoptium 等继续提供免费 LTS 构建）
> - `2021 年 9 月 14 日`：**Java SE 17 LTS** 发布（密封类、模式匹配预览、移除实验性 AOT）
> - `2023 年 9 月 19 日`：**Java SE 21 LTS** 发布（`虚拟线程`、模式匹配 switch、记录模式）
> - `2024 年 3 月 19 日`：**Java SE 22** 发布（`Foreign Function & Memory API` 正式版、多文件源码直接运行预览）
> - `2024 年 9 月 17 日`：**Java SE 23** 发布（ZGC 分代模式默认启用、Markdown 文档注释）
> - `2025 年 3 月 18 日`：**Java SE 24** 发布（`永久禁用 Security Manager`、抗量子加密模块、分代 Shenandoah GC）
> - `2025 年 9 月 16 日`：**Java SE 25 LTS** 发布（`Scoped Values` 正式版、`结构化并发`第五次预览、紧凑源文件、灵活构造器）
>   - Java 25 引入==紧凑源文件与实例 main 方法==，Hello World 可简写为：
>   ```java
>   void main() {
>       IO.println("Hello World!");
>   }
>   ```
> - `2026 年 3 月 17 日`：**Java SE 26** 发布，最新非 LTS 版本；下一个 LTS 计划为 Java 29（2027-09）

> [!note]- 版本命名变迁
> Java 的版本命名经历过几次调整。早期 `JDK 1.0 ~ 1.1` 直接叫 JDK。`1998` 年进入 Java 2 时代，对应版本为 `J2SE 1.2 ~ 5.0`；`1999 年 6 月` Sun 将 Java 划分为标准版（J2SE）、企业版（J2EE）和微型版（J2ME）三个版本。`2004` 年，`J2SE 1.5` 为强调版本重要性首次去掉「1.」前缀，改称 `Java SE 5.0`。`2005 年 6 月` JavaOne 大会上，Sun 宣布统一去掉命名中的「2」，各版本分别更名为 Java SE、Java EE 和 Java ME。自 `Java 9 起`，改为每 6 个月发布一个新版本的快速发布节奏，每 2 年推出一个长期支持（LTS）版本。

> [!tip]- 选择哪个版本学习？
> 当前推荐以 `Java 25 LTS` 作为学习和开发基准。它是最新长期支持版本（2025-09 发布），`虚拟线程` 已全面成熟，`Scoped Values` 正式落地。`结构化并发` 仍在预览阶段（API 持续演进中），主流框架已跟进支持。下一个 LTS 将是 2027 年 9 月的 Java 29。

## 加载与执行原理

### CLASSPATH 加载机制

JVM 启动时，ClassLoader 按以下顺序加载 `.class` 文件：

```text
Bootstrap ClassLoader   → JDK 核心库（rt.jar, $JAVA_HOME/lib）
        ↓
Extension ClassLoader   → 扩展库（$JAVA_HOME/lib/ext）
        ↓
Application ClassLoader → CLASSPATH 指定路径
```

> [!note] 当前目录 `.`
> - 未设置 `CLASSPATH` 环境变量时，默认 `CLASSPATH = .`（当前目录）
> - 一旦手动设置了 `CLASSPATH`，`.` 就不再自动包含，需显式加上：
>
> ```bash
> # 手动设置时必须带上 .，否则当前目录失效
> CLASSPATH=.;C:\libs\foo.jar
> ```

## 从零编写第一个 Java 程序

### JDK 下载与环境配置

Java 开发工具包（JDK）包含了编译器、运行时环境和各种开发工具。你可以从 Oracle 官方网站或 OpenJDK 社区下载适合你操作系统的版本。

![[Pasted image 20260626211304.png]]

安装目录说明：

```text
$JAVA_HOME/
├── bin/        # 可执行工具目录（java/javac/jstack 等命令均在此）
├── conf/       # 运行时配置文件（安全策略/网络/日志等）
├── include/    # C/C++ 头文件（JNI/JVMTI 开发时 #include 此处）
├── jmods/      # 模块化 JDK 包（.jmod 文件，jlink 构建自定义 JRE 的原料）
├── legal/      # 各模块开源协议声明（LICENSE/NOTICE，发布合规用）
├── lib/        # JVM 核心库（JDK 类库）与内部数据（非用户 classpath，勿混淆）
├── man/        # 命令手册页（man java 等，部分发行版/平台才有）
└── release     # 纯文本版本描述文件（JAVA_VERSION/OS_ARCH 等 KV 信息）
```

将 `bin` 目录配置到环境变量：

- Windows：系统属性 → 高级系统设置 → 环境变量 → Path → 新建 `%JAVA_HOME%\bin`

> [!warning] JAVA_HOME 与 PATH 的区别
> `JAVA_HOME` 是给 Maven/Tomcat 等工具定位 JDK 根目录用的，PATH 只是让 shell 能执行 `java` 命令，两者解决不同层次的问题。

### Hello World

编写 `HelloWorld` 源码：

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

编译源码：

```shell
# 在命令行中，进入 HelloWorld.java 所在目录，执行：
javac HelloWorld.java
```

编译后会生成一个 `HelloWorld.class` 文件，包含 Java 字节码。执行以下命令运行程序：

```shell
java HelloWorld
```

> [!warning] 类名而非文件名
> `java` 命令后面跟的不是文件名，而是==类名==。

```text
# 命令行输出
Hello, World!
```

### public class 与 class 的区别

一个 Java 源文件中可以定义多个 `class`。编译之后，一个 `class` 就会对应生成一个 `class` 字节码文件。

如果一个类是 `public` 的，类名必须和源文件名保持一致。`public` 的类可以没有；如果有的话，也只能有一个。

## Java 注释类型

```java
// 单行注释

/*
多行注释
多行注释
 */

/**
 * 文档注释（JavaDoc）
 * 用于生成 API 文档，支持 HTML 标签和特殊标记（@param/@return 等）
 * javadoc -d doc/ HelloWorld.java
 */
```

> [!tip] JavaDoc 的实际价值
> 文档注释不仅用于 `javadoc` 命令生成 HTML API 文档，主流 IDE（IntelliJ IDEA / Eclipse / VS Code）中鼠标悬停在类、方法、字段上看到的说明提示，同样读取的是 JavaDoc——写好 JavaDoc 等于同时为生成文档和 IDE 智能提示提供内容。
